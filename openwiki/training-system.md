# 训练系统与知识模型

本页是理解后台 Agent 的核心页面：它如何建模、选材、编排任务、更新状态，以及如何处理外部词汇记忆。

## 后台 Agent 的职责

文档一致把后台 Agent 定义为以下角色：

1. 建立并维护学习者模型；
2. 将考试目标分解为三阶段目标；
3. 根据水平、时间和训练量选择内容；
4. 将零散知识组织成完整任务；
5. 从作答和文字作品中生成待验证错误假设；
6. 协调外部词汇复习、语法补强、迁移与延迟复习；
7. 组织阅读—翻译—写作的统一闭环，并按未辅助作答、修订、换材料迁移和延迟复测更新模型；
8. 评估阶段内进步与下一阶段准备度；
9. 后期按英语一/英语二及题型切换训练策略。

来源主要是 `docs/05-训练系统与教学规范.md`、`docs/04-MVP-PRD.md` 和 `docs/06-数据模型与效果评估.md`。

## 学习者模型必须包含什么

### 明确输入

- 考试类型、日期、目标分；
- 每日/每周学习时间与可用时段；
- 语言背景（四六级、学习经历等）；
- 显式偏好（任务长度、反馈密度、纸屏习惯等）；
- 用户主动声明的困难与优势。

### 动态推断

- 词汇、词组、语法、阅读、翻译、写作状态；
- 不同题型的速度、正确率和稳定性；
- 反复出现的错误假设及其证据、置信度和状态；
- 当前阶段与进入下一阶段的准备度。

### 关键原则

- 明确输入与系统推断必须分开展示；
- 错误假设不是永久标签；
- 模型必须能被用户查看、纠正和删除；
- 模型要同时表达“会什么”和“不确定什么”。

## 局部知识状态

仓库不是要做一个粗糙的总分熟悉度，而是要维护局部知识状态：

| 对象 | 主要维度 |
|---|---|
| 词汇 | 最近学习、目标词义、阅读识别、翻译处理、主动表达、迁移、保持率 |
| 词组 | 含义、结构、阅读识别、翻译/写作使用 |
| 语法 | 识别、拆解、解释、改写、迁移、稳定性 |
| 阅读 | 各题型正确率、证据链、耗时、难度、稳定性 |
| 翻译 | 理解、完整、中文表达、速度 |
| 写作 | 审题、内容、结构、语言、语域、模板迁移 |
| 读写闭环 | 读懂、译出、受控表达、独立写作、修订采纳、换材料迁移 |
| 考试执行 | 模块用时、做题顺序、检查、整套稳定性 |

状态更新时，不应只记录一个分数，还要保留证据数量、新鲜度、难度、独立程度、是否见过材料和置信度。

## 反复错误假设

错误假设是可变的诊断对象，不是永久标签。最小字段包括：

- 假设描述；
- 触发证据；
- 相关知识节点；
- 影响题型；
- 置信度；
- 支持/反对证据；
- 验证任务；
- 当前状态：待验证、确认、改善、推翻、复查。

这类假设适合驱动“为什么连续错同一类题”的分析与改题编排。

## 完整任务应该长什么样

每个任务至少包含：

- 阶段目标；
- 关联知识节点；
- 选择原因；
- 预计时长；
- 输入材料；
- 主动处理；
- 反馈；
- 补强；
- 迁移；
- 完成标准；
- 复习安排。

读写研究稿进一步要求把阅读、翻译和写作放进同一个闭环：未辅助作答后，根据证据位置和错因给出当前所需脚手架，再通过修订、重做、换材料迁移和延迟复测来确认是否真正掌握。

任务执行顺序通常是：

1. 激活前置知识；
2. 处理完整材料；
3. 独立作答或输出；
4. 查看分层反馈；
5. 确认知识缺口或错误假设；
6. 做补强；
7. 用新句、新段或新题迁移；
8. 生成复习计划。

## 三阶段编排原则

### 前期熟悉阶段

目标是减少模型不确定性，而不是提分最大化。前期评估可以调用短 CAT/MST 或锚定任务，但这些只服务于测量定位，不直接决定日常训练内容。

### 中期任务阶段

目标是补足局部知识并建立读写迁移。一个周周期应至少包含：

- 主路径任务；
- 最近词汇的阅读再遇、翻译/表达迁移和到期语法复习；
- 至少一次文字表达；
- 至少一次新材料迁移；
- 至少一个高优先错误假设验证；
- 周末阶段内进步检查。

### 后期考试冲刺阶段

目标是把能力组织成题型表现和考场流程。这里的题型特化应建立在前期可解释测量与影子标定之上，不把 IRT/CAT 直接当作教学编排器。

## 词汇迁移：本系统与外部工具的分工

这是本仓库最有辨识度的训练设计之一：

- 外部工具负责高效率记忆、词书和复习调度；
- 本系统不自建闪卡，不把“已学”直接当作掌握；
- 本系统记住用户最近学过的具体词汇与目标词义，并在完整任务中让它们被读懂、译出或恰当使用；
- 词汇状态必须通过语境验证、翻译处理、受控表达和延迟迁移来更新。

### 推荐迁移梯度

1. 语境确认；
2. 篇章再遇；
3. 跨语境理解；
4. 翻译处理；
5. 受控表达；
6. 自由表达；
7. 延迟迁移。

### 不要犯的错误

- 把第三方“已学”直接写成“已掌握”；
- 把 App 使用时长当作有效学习时长；
- 为了覆盖词表强行堆砌生硬材料；
- 把所有词都硬塞进作文。

## 训练强度与难度控制

读写研究稿强调，难度不能只看一个总分，至少要分别建模词汇覆盖、目标词义、句法、篇章结构、推理跨度、主题背景、输出长度和限时压力。

推荐的初始配比是假设值，不是硬规则：

- 20% 巩固：大概率能独立完成；
- 60% 主训练：需要努力但能完成；
- 20% 挑战/诊断：暴露下一层问题。

难度调整关注：词汇覆盖率、句法层级、篇章抽象度、题型复杂度、输出长度、限时压力和材料熟悉度。
