# BinnAgent 考研英语版 Wiki 指令

本仓库当前处于产品发现与研发前置阶段，尚未进入业务代码实现。`docs/` 下的中文 Markdown 文档是当前产品需求与领域规则的权威来源；后续代码出现后，再同时以实际代码、测试和运行配置为依据。

生成和维护 Wiki 时优先覆盖：

1. 仓库快速上手与目录导航；
2. 最新双实验室定位：语境实验室负责读懂与吸收，表达实验室负责想清楚并独立写出；
3. 固定三阶段学习周期：前期熟悉、中期任务、后期考试冲刺；三阶段是周期，不是并列产品专区；
4. 学习者模型、局部知识状态、帮助依赖与反复错误假设；
5. 多维知识负荷匹配、训练任务编排、逐级撤架与阶段内进步验证；
6. 词汇迁移记忆：外部闪卡积累、最近词汇接入、阅读再遇、翻译处理、主动表达和延迟迁移；
7. 英语一/英语二共同读写能力底座和后期题型特化；
8. 数据、效果评估、隐私、版权与合规边界；
9. 后续代码架构、运行、测试、部署和外部集成点。

写作要求：

- 使用简体中文；
- 所有结论可追溯到仓库文件；
- 明确区分“当前已有”“产品要求”“计划实现”，不得把规划描述成已实现功能；
- 优先提供工程师可执行的导航、流程、边界和检查清单；
- 文档间避免重复，使用链接组织关联内容；
- 发现产品文档冲突时列出冲突与来源，不自行静默裁决；
- 不读取、记录或输出 API Key、访问令牌及其他凭据。
