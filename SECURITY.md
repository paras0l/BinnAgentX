# Security Policy

## 当前支持范围

本仓库目前仅支持本地和隔离 CI 中的技术 Spike，不接收真人学习记录、第三方账号、未授权内容或生产凭据。尚不存在公开生产服务或安全奖励计划。

## 报告问题

不要在公开 Issue 中提交 API Key、Cookie、真实作文、个人信息或可直接利用的生产漏洞细节。项目进入封闭试点前必须补充专用安全联系人和响应 SLA；在此之前，请通过仓库所有者提供的私下渠道联系。

## 必须立即停止的情况

- 凭据、完整作文、完整提示或模型原始响应进入普通日志；
- 用户端获得控制 API、内部角色或数据库能力；
- 合成身份适配器可以在生产环境启动；
- 重试产生重复远程扣费或覆盖用户原始作品；
- 未达到权利状态的内容可以呈现；
- IRT/CAT、自动总分或预计提分被意外开启。

发现以上情况后应停止扩展功能，保留审计证据，撤销暴露的凭据，并在继续前增加回归测试。
