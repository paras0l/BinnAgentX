# 受约束模型网关与确定性降级实现记录

> 状态：受约束模型网关 v0.1
>
> 日期：2026-07-16
>
> 证据边界：证明网关能拒绝超预算、超时、结构错误和证据错引，并可审计地降级；不证明任何远程模型已上线或反馈具有教学效度。

## 1. 本增量的目的

模型不能直接接管表达反馈。它只能在一个有限、可验证、可降级的通道中提供候选结果：

`V1 不可变版本 → 预算门 → 超时门 → JSON Schema 门 → 证据子串门 → 发放或确定性降级`

任何门失败都发放已审核的本地单项反馈，不向学习者暴露内部错误，不允许模型代写。

## 2. 输出契约

机器契约位于 `contracts/model/priority-feedback/v1/`，只接受：

- 固定版本 `schema_version=1.0.0`；
- 唯一优先方向 `claim | logic | expression`；
- 20–500 字符的单项反馈；
- 必须在用户 V1 中连续出现的 `evidence_quote`；
- 必须为 `null` 的 `replacement_text`。

Schema 禁止额外字段。即使 JSON 结构合法，证据不是 V1 的精确子串也会被拒绝。

## 3. 预算、超时与降级

单次运行预算继续使用 `model_call_count` 和 `cost_usd`，开发默认是最多 3 次、最多 0.20 美元。网关在调用前评估估算成本，并对已发起但超时或异常的远程调用保守记账。

降级结果分为：远程调用禁止、预算不足、超时、适配器异常、输出结构无效和证据错引。每类都有稳定的 `outcome` 和 `rejection_code`。

## 4. 安全审计与回放

`model_invocations` 保存调用用途、适配器、提示版本、结果、费用、延迟、输出哈希、证据偏移/哈希和拒绝码。它不保存完整提示词、学习者 V1 或证据引文。

学习端仍只看到一条可理解的反馈。控制舱回放可查看安全审计元数据，用于区分通过、拒绝和降级，但不复制学习者作品。相同幂等键不会生成第二条调用记录。

## 5. 当前执行边界

同步学习 API 只注册 `deterministic_fixture`。`BINNAGENT_ENABLE_REMOTE_MODEL_CALLS` 默认为 `false`，仓库中没有供应商凭据或真实远程适配器。远程分支只通过单元测试替身验证超时、超预算、非法输出和证据错引。

这意味着本增量给出了“模型可以进入的边界”，但没有让远程模型进入用户主路径。

## 6. 自动化证据

已覆盖：

- JSON Schema 有效样例；
- 本地夹具通过与证据哈希；
- 远程开关与预算的调用前拒绝；
- 超时后保守记账和确定性降级；
- 代写字段和证据错引拒绝；
- 表达任务端到端 API 幂等、审计和 V2 闭环回归。

## 7. 下一增量

实现 Worker/Outbox 模型调用预约协议：在事务内保留预算和输入版本，在 Worker 中执行可重试调用，再以幂等命令提交结果。在此之前，不应将真实远程适配器注册到同步学习 API。作文总分、预测提分和自动阶段切换仍保持禁用。
