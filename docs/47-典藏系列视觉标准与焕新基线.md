# 典藏系列视觉标准与焕新基线

> 状态：视觉方向已冻结——概念 1 为白昼模式、概念 3 为月夜模式；昼夜双模式为典藏皮肤独有能力。  
> 范围：学习端既有皮肤系统中的 `collector` 等级，不新建商城、货币、抽取、奖励或平行工作区。  
> 关联规范：[皮肤系统优化](38-皮肤系统优化.md)、[皮肤系统统一规范](39-皮肤系统统一规范.md)。

## 1. 目标

典藏皮肤不是“更亮的色板 + 更多装饰”，而是同一学习产品中的完整收藏级表现层。它必须同时做到：

1. 用户在首屏 3 秒内感知到明确的主题世界、最高等级和稀缺工艺。
2. 首页、资产、画像、偏好、训练与完成页保持同一视觉叙事，但不改变任何学习规则、业务语义和任务顺序。
3. 主题资产不仅被陈列，还要与下一次训练、空状态、阶段完成和图鉴等既有节点建立可理解的场景关系。
4. 典藏效果可被关闭、降级和恢复；资源失败时仍是完整可用的学习界面。
5. 付费价值来自完整幻想、跨场景一致性、专属时刻与可回看的收藏内容，不来自虚假倒计时、随机抽取或干扰学习的促销。

## 2. 外部参考转化

本规范只吸收公开产品原则，不复制其他产品的视觉资产或商业机制。

- Riot 将高阶外观描述为围绕角色幻想推动设计与技术能力的“luxury good”，并通过多形态、VFX、语音和 HUD 等多层体验表达价值。这支持“典藏必须是完整体验，而不是单张背景”的判断。来源：[Exalted Skins, the Mythic Shop, and Nexus Finishers](https://www.leagueoflegends.com/en-gb/news/game-updates/dev-exalted-skins-the-mythic-shop-and-nexus-finishers/)。
- VALORANT 的高级外观同时覆盖模型、动画、特效、音频、终结表现和变体，但坚持不影响核心玩法。这支持“主题能力可深入表现层，不能改变学习任务”的边界。来源：[VALORANT Store and Cosmetic Content](https://playvalorant.com/en-us/news/dev/valorant-store-and-cosmetic-content/)。
- VALORANT 的制作复盘强调先建立完整幻想，再通过测试移除妨碍核心操作的动画。这支持“先沉浸、再以任务可用性做硬门槛”的流程。来源：[The Craft and Fantasy of VALORANT Weapon Skins](https://playvalorant.com/en-us/news/dev/the-craft-and-fantasy-of-valorant-weapon-skins/)、[True to Form: The story of Reaver](https://playvalorant.com/en-us/news/dev/true-to-form-the-story-of-reaver/)。
- W3C 要求正文对比度至少 `4.5:1`、大字号至少 `3:1`、必要控件与图形至少 `3:1`；交互触发的非必要动画应可关闭。来源：[WCAG 2.2](https://www.w3.org/TR/WCAG22/)、[Animation from Interactions](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions)。
- Apple 的动效指导要求动效服务于反馈和状态、避免外围持续运动，并让用户可以选择减少动态。这支持粒子标题、指针轨迹和角色漂浮全部服从减少动态设置。来源：[Motion](https://developer.apple.com/design/human-interface-guidelines/motion)、[Accessibility](https://developer.apple.com/design/human-interface-guidelines/accessibility)。

## 3. 典藏的六层价值模型

| 层级 | 用户感知 | 必须证据 |
| --- | --- | --- |
| 世界 | 进入一个完整主题，而非换色 | 首屏主场景、环境材质、统一色光 |
| 工艺 | 最高等级有独特材质与细节 | 专属边框、徽记、按钮、图标底座与细节动效 |
| 陪伴 | 角色理解当前学习场景 | 至少首页、训练、完成三个语义明确的独立角色场景 |
| 拥有 | 用户能感到“这是我的典藏版本” | 既有图鉴入口、章节或典藏标识；不引入货币和数值炫耀 |
| 记忆 | 每次关键节点都有专属时刻 | 开始训练、空状态、阶段完成的差异化表现 |
| 克制 | 长时间学习仍然舒适 | 高对比正文、稳定布局、减少动态、资源失败降级 |

任意一层缺失，典藏都不能仅靠徽章或背景图通过验收。

## 4. 首屏标准

### 4.1 信息顺序

必须复用当前学习首页顺序：

`左侧导航 → 页面位置/账号 → 页面标题 → 下一次训练主视觉 → 训练任务队列 → 进度与证据`

皮肤允许改变主视觉构图、角色安全区、按钮外观、表面层级与装饰位置；不得改变入口名称、任务队列职责、主操作顺序和证据含义。

白昼与月夜必须复用完全相同的组件树、交互入口、学习状态和业务数据。昼夜之间只允许改变样式、资产、空间位置与视觉层级；任何现有组件都不得因模式切换而隐藏、删除或缺省。

### 4.2 视觉层级

- 下一次训练是唯一最高视觉权重区域。
- 一个首屏只允许一个高细节主场景、一个主操作和一个弱化的图鉴/典藏入口。
- 角色脸部、主姿态和关键主题标志不得被文案、按钮或裁切遮挡。
- 正文区域不叠加高频纹理、粒子或复杂插画。
- 典藏徽记必须表达版本身份，不伪装成奖励、任务状态或学习成就。
- 不使用“全黑 + 满屏金色”作为默认高级感，不用稀缺倒计时、发光币值或抽取概率表达价值。

### 4.3 版式下限

- 桌面设计基准：`1440 × 1024`。
- 桌面主视觉建议高度：`440–560px`；必须在 `1024px` 高度内露出训练任务队列标题与首个任务。
- 正文基准：`14–16px`，行高不低于 `1.5`。
- 主要点击目标最小高度：`40px`；键盘焦点环不可被发光、边框或背景吞没。
- 产品保持电脑浏览器专属边界；`1024px` 是最小布局宽度。`1024px`、`1100px`、`1440px` 与 125% 缩放必须保持任务顺序，不横向溢出。

## 5. 组件级标准

典藏必须至少覆盖以下稳定锚点：

| 锚点/组件 | 典藏要求 | 禁止 |
| --- | --- | --- |
| `navigation` | 专属选中态、图标底座和清晰焦点 | 只靠颜色区分 |
| `hero` | 收藏级场景、文字安全区、主操作 | 多个主按钮、角色遮挡 |
| `theme-artbook-trigger` | 低干扰但可识别的典藏入口 | 固定占据主任务视线 |
| `primary-actions` / 任务队列 | 同主题表面、图标和状态语言 | 把每一行做成独立宣传卡 |
| `card` / `summary` | 统一边角、材质和阴影尺度 | 每类卡片随意换装饰 |
| `empty-state` | 场景匹配的独立角色资产 | 从图集裁切、重复主视觉 |
| `settings-toolbar` / 控件 | 专属按钮、开关、选择器状态 | 改变控件语义或可访问名称 |
| 完成/阶段节点 | 独立里程碑画面与静态降级 | 以动效替代结果文本 |

### 5.1 本轮组件工艺落地

- `session-brief` 保留原有三个信息项，外层取消厚重卡片感；每个信息项使用独立 `1px` 轻边框、半透明珠光底和一致圆角。
- 典藏昼夜开关只显示“白昼 / 月夜”，不再重复显示“典藏昼夜”说明文字；可访问组名继续保留。
- 标题纹章用于页面标题分隔，不与操作按钮争夺空间；卡片只在少量关键节点使用原 UI 库蓝、粉、黄语言的轻量贝壳/海星角饰，不把完整花饰裁进通用面板。
- 白昼与月夜的按钮、输入、筛选、标签、空状态、任务卡和训练面板只改变样式与位置，不删减任何原有组件。
- 粒子场景建立后，原生 `h1` 只保留语义和可访问名称，视觉字形隐藏并由粒子画布替代；关闭动态、脚本失败、打印、强制色或画布不可用时，原生标题必须恢复可见。

## 6. 动效标准

### 6.1 动效分级

| 级别 | 用途 | 时长/频率 |
| --- | --- | --- |
| 微反馈 | Hover、Pressed、选中、焦点 | `120–180ms` |
| 组件转场 | 图鉴展开、主题切换、任务状态 | `180–260ms` |
| 场景呼吸 | 水光、角色轻微漂浮、低频环境 | 单循环不短于 `2.4s`，位移不超过 `6px` |
| 典藏时刻 | 首次进入、阶段完成 | 一次性，建议 `600–1200ms` |

### 6.2 硬约束

- `data-motion="reduced"` 与 `prefers-reduced-motion: reduce` 都必须关闭位移、缩放、粒子、指针轨迹和持续漂浮，仅保留必要的瞬时状态或淡入淡出。
- 指针轨迹不得在输入、选择文本和训练作答时生成高密度元素。
- 标题粒子失败、脚本未加载或画布不可用时，原生标题必须立即可见。
- 页面滚动不得触发视差背景或大面积外围运动。
- 所有主题动效不得改变布局尺寸，不得使主要控件位移。

## 7. 典藏昼夜双模式

- 白昼模式采用象牙白、珍珠母贝、潟湖青与香槟金，复用既有夏日海岛主视觉。
- 月夜模式采用深海蓝、月光青、珠光白与克制金色，使用独立月夜潟湖主视觉。
- 昼夜切换是 `collector` 等级专属能力，其他等级不展示模式入口。
- 模式切换不得刷新页面、重置当前页面、训练进度或草稿；账号偏好保存后跨页面与刷新恢复。
- 昼夜使用同一信息架构和组件数量；装饰可换位，但功能控件、入口、提示、状态与证据必须一一对应。
- 月夜模式设置原生 `color-scheme: dark`，但成功、警告、危险和焦点语义保持不变。

## 8. 色彩与材质标准

- “海豹夏日乐园”品牌三主色固定为海洋蓝 `#319EE8`、珊瑚粉 `#FF8FA7`、阳光黄 `#FFD166`。主题可以使用深浅派生色满足文字与边界对比度，但不得用青绿、香槟金或其他色相替换三主色的品牌职责。
- 白昼以蓝色承担画布、导航和结构，粉色承担主操作与活动强调，黄色承担选中、推荐和阳光点缀；月夜在深海底色上保留三色的发光映射，不改变语义状态色。
- 主题色板至少包含画布、导航轨、表面、正文、弱正文、主强调、边界、焦点、成功、警告和危险语义；状态色不得被主题色替代。
- 正文与背景目标对比度 `≥ 4.5:1`；大字号 `≥ 3:1`；必要控件轮廓与图形 `≥ 3:1`。
- 图片上的文字必须拥有独立安全底、稳定暗角或足够纯净的负空间，不能依赖某个偶然裁切。
- 玻璃、珠光、金属、纸张、水光等材质最多选择两类作为主材质，一类作为点缀。
- 高光与发光只服务于主操作、选中态或典藏徽记，不扩散到所有边界。

## 10. 图鉴标准

继续使用现有 `theme-artbook-trigger` 和图鉴模态框，不建立新路由或新工作区。

图鉴至少包含：

1. **主题序章**：一句完整世界观与主视觉。
2. **角色设定**：角色动作、表情和场景职责，可放大阅读。
3. **组件工艺**：按钮、徽章、分隔线、纹样和状态件，不再只展示整张 UI 总表。
4. **素材档案**：概念稿与来源说明；AI 生成资产记录用途与约束。
5. **动态偏好**：说明完整动态与减少动态的差异。

图鉴是收藏内容，不承担训练、奖励、购买或任务完成职责。

## 11. 资产生产标准

- 位图用于主场景、角色和复杂材质；SVG 用于徽记、边框、分隔线、按钮纹样、波浪路径和可缩放装饰。
- 不把概念大图、角色图集或 UI 总表裁成运行时角色。
- 新生成角色使用独立完整轮廓资产；透明边缘必须检查绿溢色、白边、阴影与 2× 清晰度。
- SVG 必须有明确 `viewBox`，不内嵌不可追溯位图，不包含业务文字；装饰使用 `aria-hidden="true"`。
- 所有资产在注册表声明尺寸、类型、裁切与加载策略；只有一张真正影响 LCP 的主视觉可以预加载。
- 资源加载失败时回退到语义色板与通用组件，不显示破图。

## 12. 已识别工程阻塞项

浏览器取证发现：学习端同源代理 `apps/learner-web/app/api/learner/[...path]/route.ts` 目前只导出 `GET` 与 `POST`，偏好保存使用 `PUT /v1/preferences`，因此界面保存皮肤会返回 `405`。这会导致预览中的典藏主题在离开设置页后恢复，属于典藏用户旅程的 P0 验收阻塞项。

本轮实现必须：

1. 在既有代理中复用同一 `forward` 处理器导出 `PUT`，不增加旁路 API。
2. 增加代理方法覆盖或浏览器测试，证明皮肤可以保存、切换页面、刷新后恢复。
3. 验证 `paper → seal-summer → paper → seal-summer` 不残留资产变量。

本轮已在既有代理导出 `PUT`，并通过前端自动化、API 集成测试和真实浏览器保存/切换验证；该 P0 阻塞已解除。

## 13. 验收清单

### 13.1 视觉

- 首屏明确达到典藏等级，且主任务比装饰更突出。
- 首页在 `1440 × 1024` 首屏可见主操作和任务队列首项。
- 资产、画像、偏好和真实训练工作区均完成昼夜模式同屏验收，不以首页截图替代跨界面验收。
- 角色无裁脸、遮字、拉伸、低清或透明边缘问题。
- 图鉴不再是四张不可读缩略图的简单平铺。
- 位图、SVG、字体、间距、色彩与文案均通过同屏对照 QA。

### 13.2 交互

- 皮肤选择、即时预览、保存、跨页和刷新恢复全部成立。
- 典藏昼夜切换前后组件数量与功能入口保持一致。
- 图鉴可键盘打开、关闭、切换章节并恢复焦点。
- 主操作、任务队列、导航、表单、开关和选择器仍使用原业务链路。
- 完整动态与减少动态都可用；粒子与指针轨迹不拦截输入。

### 13.3 响应式与无障碍
