# 产品发现与研发路线图

> 原则：先验证双实验室闭环、学习者模型和迁移效果，再进入技术立项。本路线图暂不包含编码任务。最新定位见[双实验室产品定位与竞争力分析](14-双实验室产品定位与竞争力分析.md)。

## 2. 阶段 1：问题发现

### 样本

- 12–18 名考生；
- 英一、英二各不少于 5 人；
- 覆盖基础薄弱、中等、高分目标；
- 覆盖应届、在职、二战；
- 至少一半同时使用两种以上工具；
- 另访谈 5 名考研英语教师/助教。

### 考生访谈

- 最近一周的学习内容如何安排；
- 单词、语法、阅读、网课和真题怎样衔接；
- 当前使用哪种词汇工具，能否导出或分享哪些学习记录；
- 带来一个“卡片上认识、文章中却失败”的真实例子；
- 复盘最近一次不敢下笔、无法展开或依赖范文的写作；
- 能否低成本取得最近学习的具体词汇，而不只是总词量；
- 愿意以 API 授权、文件、截图还是手动方式提交词汇记录；
- 哪些学习行为感觉碎片化；
- 最近一次任务过难或过易的具体经历；
- 如何判断自己本阶段是否进步；
- 哪类错误反复出现，如何处理；
- 每日/每周希望自主设置多少训练量；
- 会如何在高题量与复习、写作之间取舍；
- 何时开始真题和题型方法；
- 作文模板如何背、是否会换题使用；
- 希望系统记住哪些背景和偏好；
- 哪些系统推断必须允许自己纠正。
- 只看“语境实验室/读懂实验室”和“表达实验室”的名称时，分别理解成什么；
- 每天愿意连续完成多长的阅读—短写作任务。

### 教师访谈

- 如何从作答判断局部知识状态；
- 反复错误应如何形成假设而非标签；
- 词汇、词组和语法地图的合理粒度；
- 不同基础适合什么材料难度；
- 前期评估、中期任务、后期冲刺的进阶条件；
- 英一/英二各题型需要何种特化；
- 作文模板从记忆到迁移的判定；
- 哪些自动反馈必须人工复核。

## 3. 阶段 2：学习者模型共创

### 目标

验证以下模型是否足以支持训练，又不过度收集：

- 目标；
- 学习时间；
- 语言背景；
- 显式偏好；
- 词汇/词组/语法局部状态；
- 阅读、翻译、写作和考试执行；
- 反复错误假设；
- 阶段准备度。

### 方法

- 教师独立分析同一批匿名作答；
- 比较教师之间的知识状态与错误假设；
- 找出无法稳定判断的字段；
- 设计支持证据、反证和置信度；
- 让用户查看模型并指出误判；
- 删除无法改变训练决策的字段。

### 退出门槛

- 教师能理解并基本同意字段含义；
- 用户知道哪些是自己填写、哪些是系统推断；
- 每个动态状态都能指向训练决策；
- 错误假设可被验证、推翻和归档。

## 4. 阶段 3：三阶段原型

### 必测页面

1. 阶段首页；
2. 学习者模型；
3. 训练量定制器；
4. 完整任务；
5. 语法地图；
6. 词汇迁移记忆与最近词表导入；
7. 文字表达工作区；
8. 冲刺题型页；
9. 阶段报告。

### 必测任务

- 确认英一/英二、目标和时间；
- 完成一段前期评估并纠正画像；
- 设置标准/加强/自定义训练量；
- 完成阅读—语法—翻译联动任务；
- 查看某语法节点为何未点亮；
- 选择现有词汇工具、提交一份学习记录并理解其证据等级；
- 查看并质疑一个错误假设；
- 将中期路径切换到冲刺阶段；
- 完成一次作文模板换题迁移；
- 看懂阶段报告中的真实进步证据。

### 指标

- 无主持人帮助的任务成功率；
- 用户能否复述三阶段区别；
- 能否理解训练量提高后的负荷；
- 是否认为任务完整而非零散；
- 是否理解知识状态和错误假设可变；
- 是否分得清真题重做与迁移进步。

## 5. 阶段 4：14 天人工训练实验

### 设计

- 20–30 名用户；
- 第 1–3 天完成评估和校准；
- 第 4–14 天进入中期任务；
- 用户自选低/中/高训练量；
- 团队人工编排完整任务和更新学习者模型；
- 保留用户原有背词工具，把最近具体词汇编入匹配阅读、翻译和文字表达；
- 与用户实验前一周原有学习方式比较。

### 观察

- 任务预计时长与实际时长；
- 不同训练量的完成率和保持率；
- 内容过难/过易率；
- 外部最近词表能否顺利提交，目标词是否能自然回到匹配阅读；
- 用户能否在跨主题阅读、翻译或适当表达中再次使用最近词；
- 语法地图是否帮助理解路径；
- 错误假设是否准确并可修复；
- 文字输出是否持续发生；
- 用户能否说出阶段内具体进步；
- 人工编排每用户耗时，确定自动化价值。

### 建议通过门槛

- 多数用户认为训练比原方式更连贯；
- 完整任务完成率达到约定阈值；
- 新材料或延迟验证出现正向信号；
- 最近词汇从外部记忆到篇章理解的迁移出现正向信号；
- 训练量提高没有显著推高无效重复；
- 至少一项反复错误得到多证据验证与改善；
- 用户愿意继续使用同一阶段路径。

## 6. 内容与评测准备

### 最小内容集

- 合法可用的英一/英二材料；
- 词汇/词组状态样例；
- 最近词汇进入分级阅读、翻译和表达的迁移样本；
- 语法地图第一版节点、依赖和点亮任务；
- 分级阅读与长难句；
- 句子、翻译、段落和作文表达任务；
- 各题型方法与真题多轮任务；
- 作文模板的记忆和迁移样本；
- 多教师评分的错误假设与文字金标。

### 质量要求

- 内容难度有标定依据；
- 答案和解析双人审核；
- 语法节点和前置关系可解释；
- 写作/翻译至少双评；
- 真题多轮不把答案记忆当能力；
- 内容来源和授权可追溯；
- 版本可回滚。

## 7. 进入技术立项的门槛

- 三阶段结构得到用户理解；
- 学习者模型字段得到教师与用户验证；
- 完整任务明显优于随机碎片；
- 训练量定制不会破坏复习和表达；
- 语法地图点亮规则可执行；
- 错误假设具有可接受一致性；
- 英一/英二后期配置明确；
- 内容权利路径明确；
- 指标和评测集已具备代表性样本；
- 预估成本支持足量训练；
- 外部词汇数据接入有不依赖单一厂商的降级方案。

## 8. 技术立项阶段必须回答

- 规则、知识图谱、内容检索、模型与任务引擎的边界；
- 如何维护高粒度学习者模型而不产生伪精确；
- 如何按用户训练量扩展任务而不随机生成低质题；
- 如何对内容难度进行标定与更新；
- 如何保存文字版本和延迟验证；
- 如何避免真题答案泄露与重复记忆污染；
- 如何进行内容、评分、知识地图和规则版本回归；
- 模型失败时确定性任务是否可继续；
- 单个有效进步单元的成本与延迟；
- 数据最小化、删除、授权和生成内容标识如何落地；
- 第三方词汇工具采用合作 API、用户导入、分享还是手动记录，各自的稳定性和合规边界。
- 后台 Agent 的确定性节点、模型决策节点、工具权限、暂停恢复和人工复核如何划分；
- 如何分别评测 Agent 的单节点、执行轨迹、最终任务质量和真实学习效果；
- IRT 能力量尺对应什么构念和产品决策，哪些训练证据有资格进入测量；
- 考研阅读题组如何处理局部依赖，采用 MST、testlet 还是 bifactor 模型；
- CAT 的标定、等值、DIF、题目曝光、停止规则和固定测验回退如何验证；
- 自动翻译/写作评分误差是否稳定到足以进入开放题 IRT，如何显式建模。

在真实题库、标定数据、量表拟合和影子运行达到门槛前，IRT/CAT 只作为研究与后台评估，不进入 MVP 的用户可见能力值或日常训练推荐。详见[后台 Agent 与 IRT/CAT 技术路线](13-Agent与IRT-CAT技术路线.md)。

## 9. MVP 后里程碑

### A. 熟悉阶段可信

评估和校准能形成准确、可纠正的初始模型。

### B. 中期知识补足有效

外部词汇积累、系统词汇验证、语法地图和表达任务能形成延迟与迁移进步。

### C. 训练量可规模化

不同强度下仍保持内容匹配、任务连贯和成本可控。

### D. 后期题型特化稳定

英一/英二各题型、真题多轮和模板迁移形成可验证结果。

### E. 长周期效果验证

通过 8–12 周试点观察阶段内进步、模考表现和最终稳定性。

## 10. 下一步动作

1. 招募考生和教师；
2. 执行碎片化、水平匹配和训练量访谈；
3. 共创学习者模型与错误假设格式；
4. 制作三阶段信息架构；
5. 制作训练量定制器原型；
6. 制作阅读—语法—翻译完整任务；
7. 定义语法地图第一版；
8. 测试外部词汇工具的 API、导出、截图和手动记录路径；
9. 开展 14 天人工训练实验；
10. 根据结果修订 PRD，再做技术选型。
