# 考研英语 App 市场与竞品调研

> 调研时间：2026-07-15  
> 范围：中国大陆、面向英语（一）/英语（二）初试备考的移动与 Web 产品。  
> 方法：教育部/研招网公开信息、App Store 产品页与版本说明、产品官网、公开研究。评分数是触达与存量的粗略代理，不代表活跃用户或市场份额。

## 1. 市场背景

教育部公布的全国研考报名人数为：2023 年 474 万、2024 年 438 万、2025 年 388 万、2026 年 343 万。2026 年仍有 343 万报名者，但市场已从高速扩张转向存量竞争。来源：[2023](https://www.moe.gov.cn/jyb_xwfb/s5147/202301/t20230111_1038965.html)、[2024](https://www.moe.gov.cn/jyb_xwfb/s5147/202312/t20231225_1096034.html)、[2025](https://www.moe.gov.cn/jyb_xwfb/gzdt_gzdt/moe_1485/202411/t20241121_1163982.html)、[2026](https://www.moe.gov.cn/jyb_xwfb/gzdt_gzdt/moe_1485/202511/t20251124_1421370.html)。

公开资料没有可靠披露其中英语一、英语二各自人数以及英语科目 App 的真实月活、付费率和市场份额。因此本报告不做伪精确 TAM/SAM 估算，后续应通过渠道数据、投放测试和用户调查补足。

### 对产品的含义

- 用户意图强、目标明确、备考周期长，适合订阅和长期个性化；
- 需求强季节性，寒假/春季打基础、暑期强化、秋季真题与写作、考前冲刺；
- 用户普遍同时使用纸质真题、单词 App、网课和社媒资料，切换成本与信息碎片化明显；
- 头部产品已建立内容、品牌、师资和算法壁垒，新产品应避开“词库更多、课程更多”的正面竞争；
- 报名人数下降会放大获客成本压力，留存与口碑比纯下载量更重要。

## 3. 功能同质化地图

| 能力 | 市场成熟度 | 新产品策略 |
|---|---:|---|
| 通用考研词库 | 很高 | 不自建“更多词书”卖点，聚焦真题考义和个人弱词 |
| 艾宾浩斯/间隔重复 | 很高 | 作为底层基础能力，不作为品牌差异 |
| 真题在线刷题/打印 | 高 | 必须有，但应强调作答过程证据与复盘 |
| 直播/录播课程 | 很高 | MVP 不做；可后续接入精选短讲 |
| 错题本 | 高 | 从收藏夹升级为错因、相似错误和复习队列 |
| AI 作文批改 | 中高且上升 | 采用考试量表、证据标注、改写训练和多次稿，而非只给分数与范文 |
| AI 计划 | 中高且上升 | 计划必须绑定时间、能力证据、任务结果和自动重排 |
| 能力雷达 | 中 | 展示必须可解释、可追溯，避免伪精确 |
| 固定阶段周期 | 低 | 重点机会：熟悉—任务—冲刺的明确进阶结构 |
| 训练量自由定制 | 低 | 重点机会：按时间和目标扩展连贯任务，而非固定题量 |
| 语法地图与局部知识状态 | 低到中 | 重点机会：具体到词、词组和结构，并用证据点亮 |
| 反复错误假设 | 低 | 重点机会：把重复错误作为待验证假设，驱动针对训练 |
| 读写联动 | 低 | 重点机会：把阅读中的词组、句法迁移到翻译和写作 |
| 跨题型迁移诊断 | 低 | 重点机会：同一词汇/句法问题如何同时影响阅读、翻译和写作 |

## 4. 公开用户反馈中的需求信号

App Store 公开评论只能作为方向性材料，存在陈旧、选择偏差和营销内容，不能替代访谈。仍可观察到：

- 用户重视简洁、可持续和明确的每日任务；
- 仅靠通知并不能促成行动，用户需要更小的启动成本与可感知回报；
- 真题在线查词、全文翻译、打印与平板标注能显著降低工具切换；
- 用户会比较“看见单词”与“能输出/能在题目中识别”的差异；
- 课程和资料丰富不等于能形成计划，人工督学仍被当作高价值服务；
- 错题自动分类、薄弱点练习和周测正在成为主流预期。

## 5. 用户未被充分满足的问题

### 5.1 计划与执行脱节

传统计划基于日历，缺少对完成质量、耗时、连续中断和能力变化的响应。用户落后后通常选择废弃整个计划。

### 5.2 错题记录与真正诊断脱节

“选错了 B”可能来自不认识词、句法断错、没有找到证据、错误推断、题干误读或时间压力。大多数错题本只保存题目，无法决定下一步训练。

### 5.3 单项工具与总分目标脱节

词汇、长难句、阅读、翻译和写作分别存在优秀工具，但缺少统一的能力状态和资源分配逻辑。

### 5.4 碎片学习与阶段进步脱节

每日单词、每日长难句、随机小题和视频打卡都能产生完成感，却不一定属于同一学习路径。用户很难知道本阶段补足了哪些知识、是否达到了进入真题训练的条件。机会在于固定阶段、完整任务和迁移验证。

### 5.5 自动反馈与可信度脱节

作文打分、能力雷达和自动计划容易显得精确，却可能缺少量表、证据和置信度。考生需要知道“为什么这样判断，以及如何验证”。

## 6. 竞争策略

### 应建立的壁垒

1. **结构化真题与知识图谱**：题目—句子—词义—句法—考点—错误类型关联；
2. **高粒度学习者模型**：目标、时间、语言背景、偏好、局部知识状态和错误假设；
3. **阶段化任务引擎**：固定周期、可定制训练量和完整任务路径；
4. **语法地图与读写迁移**：从识别、拆解到改写和新语境使用；
5. **考研英语教研评测集**：水平匹配、错误假设、翻译与写作的专家金标；
6. **结果证据**：同难度进步、延迟保持、迁移和模考改善。

### 不应竞争的方向

- 自建背单词引擎、词书数量和记忆算法；
- 名师数量；
- 全科和院校信息覆盖；
- 用生成题数量替代内容质量；
- 用连续打卡天数替代学习效果。

## 7. 待一手验证的假设

| 编号 | 假设 | 验证方式 |
|---|---|---|
| H1 | 用户比起更多资料，更需要固定阶段、匹配水平的连续训练 | 问题访谈 + 14 天训练实验 |
| H2 | 用户愿意主动定制训练量，并接受系统给出的负荷风险 | 训练量原型 + 行为测试 |
| H3 | 用户愿意维护目标、时间、语言背景和显式偏好，以换取更准任务 | 原型行为观察 |
| H4 | 语法地图和错误假设能让阶段进步比正确率更可理解 | 教师共创 + 用户测试 |
| H5 | 英一/英二可以共用能力底座，仅在题型、材料难度和评分上配置 | 教师共创 + 真题标注一致性测试 |
| H6 | 中期持续文字表达能提升后期翻译、写作和模板迁移 | 纵向训练实验 |
| H7 | 用户愿意继续使用原有词汇工具，并用低摩擦方式提交学习记录 | 工具链日记研究 + 接入原型 |

## 8. 调研限制

- 未获得第三方移动数据平台的真实 MAU、留存、收入与投放数据；
- 未登录并完整走查所有付费功能；
- 商店评分数量不能直接比较跨平台用户规模；
- 开发者文案属于自述，功能质量与效果仍需实测；
- 尚未区分应届、在职、二战及不同基础用户的实际使用权重；
- 尚未完成教师对竞品解析与评分准确性的盲评。
