# 研究结论与立项建议

> 调研时间：2026-07-15  
> 研究类型：公开资料桌面调研；尚未完成一手访谈、可用性测试和付费验证。

## 1. 结论先行

建议有条件立项，产品方向收敛为：**考研英语读写迁移 Agent**。

Agent 作为后台训练编排系统，不提供通用聊天入口。产品前台只围绕**语境实验室**与**表达实验室**展开：前者解决“学过但没有在篇章中吸收”，后者解决“有知识但不敢写、不会独立表达”。详细定位、竞品判断与 MVP 边界见[双实验室产品定位与竞争力分析](14-双实验室产品定位与竞争力分析.md)。

2026 年全国硕士研究生招生考试报名人数为 343 万，市场规模仍大，但连续回落意味着竞争已经转向留存、训练质量和结果可信度。[教育部数据](https://www.moe.gov.cn/jyb_xwfb/gzdt_gzdt/moe_1485/202511/t20251124_1421370.html)

现有产品已经分别占据单词、课程、综合题库、真题和自动作文批改入口；按词汇水平生成阅读材料也已快速商品化。产品机会不在增加一个入口，而在建立“近期积累—语境吸收—自主表达—延迟迁移”的连续训练和证据链。

## 2. 核心产品命题

### 2.1 关注阶段内进步

每个阶段先确定能力起点和可验证目标，再配置训练。系统不只统计完成量，而要回答：

- 本阶段哪些知识状态被补足；
- 哪些反复错误假设被证实或推翻；
- 阅读、翻译和写作表现是否在同难度或新材料上改善；
- 用户是否达到进入下一阶段的门槛。

### 2.2 训练量可自由定制

用户可以按每天/每周可用时间、目标强度和题量偏好自由设置训练量，不受固定内容额度限制。系统负责：

- 将训练量换算为可执行任务；
- 预测完成压力并提示过载；
- 保证增加的是同一学习路径上的有效练习，而非随机堆题；
- 根据正确率、耗时、保持率和疲劳信号调节难度与密度。

### 2.3 避免碎片化

所有学习内容必须归属一个阶段目标、能力节点和完整任务。外部背词记录、语法点、句子和小题不能成为孤立指标，而要回到任务后的补强与迁移验证。

例如：

`阅读任务 → 发现让步结构误读 → 语法地图定位 → 结构微课 → 句子改写 → 新材料验证 → 回到阅读`

词汇采用“外部高效记忆、本系统迁移使用”：

`外部闪卡新学词 → 进入最近词汇记忆 → 匹配阅读中再遇见 → 翻译处理 → 适用词进入表达 → 延迟新语境验证`

### 2.4 水平匹配

内容选择同时考虑：

- 当前局部知识状态；
- 材料难度；
- 用户能够投入的时间；
- 是否需要提示；
- 最近同类任务表现；
- 目标分数和所处阶段。

系统应把大多数训练控制在“需要努力但可以完成”的区间，并保留少量巩固与挑战任务。

### 2.5 强化文字表达

考研英语不仅是识别答案。中期开始持续安排：

- 句子结构复述和改写；
- 关键词组主动使用；
- 英译汉；
- 中文提纲到英文句子；
- 段落展开；
- 作文模板的默写、变体和题目迁移。

最终目标是能读、能译、能写，而不是只会认单词和选答案。

## 3. 固定三阶段周期

### 阶段一：前期熟悉阶段

目标：系统形成初步用户画像，用户理解训练方式。

核心内容：

- 确认英一/英二、目标分和考试日期；
- 收集目标、学习时间、语言背景和显式偏好；
- 完成词汇、词组、语法、阅读和文字表达评估；
- 形成初始知识地图；
- 建立首批反复错误假设；
- 通过若干校准任务验证评估是否准确。

### 阶段二：中期任务阶段

目标：以完整任务持续补足局部知识状态。

核心内容：

- 词汇和词组掌握；
- 使用有道词典、不背单词等用户自选工具完成词汇积累；
- 通过授权导入、文件、分享、截图或用户确认取得最近学习的具体词表；
- 在符合用户水平的阅读材料中安排自然、反复的语境再遇；
- 对适合主动使用的词组和表达安排翻译、改写和作文输出；
- 在本系统中分别验证能认、会读、会译、会用和延迟保持；
- 语法地图逐点点亮；
- 长难句、段落与篇章训练；
- 阅读到翻译、改写和段落表达的联动；
- 反复错误假设的验证与针对性任务；
- 周期性阶段评估和任务重排。

### 阶段三：后期考试冲刺阶段

目标：把基础能力迁移到实际试卷，形成稳定应试表现。

核心内容：

- 按英语一/英语二切换试卷配置；
- 完形、阅读 A、新题型、翻译、小作文和大作文分别特化；
- 真题多轮训练，每一轮目标不同；
- 考试技巧、时间分配和选项方法论；
- 作文模板记忆、默写、变体和套题应用；
- 整套模拟与考场流程；
- 最终薄弱题型补强。

阶段顺序固定，但时长和训练量自适应。临近考试才加入的用户可压缩前两阶段，但不能完全跳过评估和局部补足。

## 4. 学习者模型最低要求

### 明确输入

- 考试类型、日期和目标分；
- 每日/每周学习时间与可用时段；
- 语言背景：四六级、既往考试、学习经历、母语/其他语言影响；
- 显式偏好：任务长度、纸屏习惯、反馈密度、题量、提醒方式；
- 用户主动声明的困难和优势。

### 动态推断

- 具体词汇和词组掌握状态；
- 具体语法结构掌握状态；
- 阅读、翻译和写作能力状态；
- 不同题型的速度、正确率和稳定性；
- 反复出现的错误假设及其证据、置信度和状态；
- 不同任务难度下的学习效率和保持率；
- 当前阶段与进入下一阶段的准备度。

## 5. MVP 建议

> 本节已按双实验室方向收敛；旧版 PRD 中的并列功能必须重新基线化，不能直接进入开发。

### P0

- 学习者档案与英一/英二确认；
- 前期多维评估和校准任务；
- 双实验室首页与三阶段状态；
- 可自由配置时间与训练量的任务引擎；
- 最近词汇接入、词义归一和迁移队列；
- 多维知识负荷匹配的语境任务、最小必要讲解与延迟验证；
- 表达 Skill、短段落、逐级撤架、分层反馈、修订和新题迁移；
- 语法与篇章技能图谱作为后台状态，不做独立课程入口；
- 语境到表达、表达缺口返回语境的双向联动；
- 反复错误假设的生成、验证与撤销；
- 阶段内进步报告；
- 后期至少覆盖阅读与写作的英一/英二差异化训练。

### 延后

- 全科、择校、调剂和复试；
- 直播课程和教师商城；
- 公共社区、排行榜和强社交；
- 复杂虚拟形象；
- 无审核的大规模自动生成模拟题。

## 6. 立项成立条件

- 用户确认“训练碎片化、内容不匹配、进步不可见”是高频问题；
- 用户能看懂三阶段结构并知道自己为何处于当前阶段；
- 14 天实验中，完整任务完成率和延迟保持率优于原有学习方式；
- 学习者模型能用真实证据解释任务选择；
- 反复错误假设能被教师理解、验证和纠正；
- 用户愿意主动设置更高或更低训练量，而系统能保持任务连贯；
- 真题、解析、词典和范文的权利路径可落地。
