# 编码前决策登记与 ADR 清单

> 状态：Sprint 0 决策台账 v0.1
> 日期：2026-07-15
> 目的：区分已冻结、仅供 Spike、必须通过 Spike 决定和正式产品前仍待输入的事项。

## 1. 状态定义

| 状态 | 含义 | 是否允许直接实现 |
|---|---|---|
| `frozen` | 产品硬边界，除非重新立项不得偏离 | 可以 |
| `spike_baseline` | 为技术实验设置的可逆默认值 | 可以，但必须隔离并记录替换成本 |
| `experiment` | 必须实现至少两个方案或明确对照后决定 | 不得直接宣称生产选型 |
| `shadow_only` | 只保存数据或离线运行，不影响用户高影响状态 | 只能影子实现 |
| `deferred` | 当前不建设 | 不可以 |
| `blocked_for_product` | 缺少真人、专家、权利或合规证据 | 不阻塞技术 Spike，阻塞正式产品 |

## 4. Spike 前临时回答

为避免编码被非关键厂商选择阻塞，首轮采用以下临时回答：

- 只在本地和隔离 CI 使用合成身份与合成学习记录；
- 不选择正式认证厂商，使用 `identity` 端口和开发适配器；
- 不选择生产模型厂商，至少实现两个网关适配器；
- 不选择生产队列，先以 PostgreSQL Outbox + Worker 验证语义；
- 不选择生产云平台，所有服务必须容器可运行、配置外置；
- 控制舱只有 `developer_reviewer`，但权限模型不能写死成布尔 `is_admin`；
- 内容只读取版本化本地内容包，内容供应商协议后续通过相同端口接入；
- 真实个人数据、第三方账号和未授权内容不得进入环境。

## 5. Spike 期间必须得到的数据

- 每个模型节点的 p50/p95 延迟、token、成本、格式失败和拒判；
- 每个状态节点的执行次数、重试、重复副作用和恢复耗时；
- 编辑器在 5,000 词、重叠标记、中文输入法、版本迁移时的正确性和性能；
- 桌面视口、200% 缩放和键盘路径下的可用性缺陷；
- 两套前端构建产物、会话、API client 和权限测试结果；
- 删除演练覆盖的存储、事件、追踪和外部引用；
- 内容权利门、版本门和下架后的行为；
- 正常、边界、恶意和故障场景通过率。

## 6. 正式 MVP 前仍需用户输入或外部证据

- 首发地区、经营主体、正式认证方式和部署地区；
- 团队实际技术配置和运维能力；
- 可接受的商业成本、响应时间和训练量策略；
- 首批正式支持的外部词汇工具；
- 内容供应商、真题年份、授权范围和下架 SLA；
- 写作/翻译人工金标和教师复核资源；
- 真实用户的桌面交互、材料难度和迁移结果；
- 隐私、生成式 AI、算法和内容经营相关专项法律结论。

## 7. 文档术语裁决

为消除“进入实现”歧义，后续统一使用：

- **技术 Spike**：可在缺少真人/教师条件时启动，只验证工程命题；
- **正式产品实现**：必须满足认知交互、内容、评分和真人证据门槛；
- **封闭试点**：允许目标用户使用，但仍不得宣称提分；
- **正式发布**：必须通过合规、内容权利、安全、效果表述和运维门槛。

任何文档中的“进入开发/实现”如未注明层级，优先按更严格的“正式产品实现”解释。
