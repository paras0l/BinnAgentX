# 合规、版权与风险清单

> 版本日期：2026-07-15。本文是产品前置清单，不构成法律意见；上线前应由熟悉中国互联网、教育、数据与知识产权的律师复核。

## 1. 产品合规定位

BinnAgent 是阶段化学习教育与自适应训练系统。Agent 位于后台，负责学习者建模、内容匹配、任务编排和反馈。

最高优先级要求：

1. 学习者模型、作答、作文和错误假设均可能构成个人信息，必须最小必要、透明、可纠正、可删除；
2. AI 生成的反馈、范例、改写、图片或音频按适用规则标识；
3. 真题、外刊、词典、例句、解析、范文和模板不能因教育用途默认可商用；
4. 面向公众提供生成式 AI 能力前，根据部署方式判断备案、登记、安全评估和算法义务；
5. 自动难度匹配、阶段切换和评分必须可解释并允许申诉/纠正；
6. 禁止“保证提分”“保证上岸”及无证据的效果宣传。

## 2. 个人信息保护

《个人信息保护法》要求处理目的明确、合理、直接相关，并采取对个人权益影响最小的方式；收集应限于最小范围。[中国人大网正式文本](https://www.npc.gov.cn/npc/c2/c30834/202108/t20210820_313088.html)

2026 年个人信息保护专项行动将教育领域、App 和 SDK 列为重点，明确治理过度收集学校、学籍、位置等信息、第三方提供不透明、缺少注销及超范围权限等问题。[中国网信网公告](https://www.cac.gov.cn/2026-04/02/c_1776867645836849.htm)

### 2.1 学习者模型数据分级

| 数据 | 建议 | 用途/风险 |
|---|---|---|
| 英一/英二、考试日期、目标分 | 必要 | 阶段和试卷配置 |
| 学习时间与训练量 | 必要 | 任务编排和负荷计算 |
| 语言背景 | 自愿、分项说明 | 水平匹配；不得用于不合理差别待遇 |
| 显式偏好 | 可选 | 呈现和任务长度 |
| 作答、作文、耗时、提示 | 必要但可管理 | 知识状态和反馈 |
| 错误假设 | 高谨慎推断 | 必须有证据、置信度和纠正机制 |
| 目标院校/专业 | 可选 | 只用于科目确认 |
| 手机号/邮箱 | 选择最小登录方式 | 账号和同步 |
| 位置、通讯录、短信 | 禁止默认收集 | 与训练无关 |
| 身份证、人脸、声纹 | MVP 不收集 | 敏感且非必要 |
| 健康、心理、人格、家庭状态 | 不推断、不建标签 | 超出产品目的 |

### 2.2 系统推断要求

- 用户明确输入与系统推断分开显示；
- 错误假设不作为永久标签；
- 每个推断显示来源、时间、置信度和用途；
- 用户可查看、纠正、推翻和删除；
- 被纠正或删除的推断不得继续影响任务；
- 不用基础薄弱、院校、语言背景等标签实施不合理价格或服务差异。

### 2.3 用户权利

- 清晰、分层的隐私告知；
- 账号注销；
- 查看、更正、导出和删除学习者模型；
- 撤回非必要同意不影响基础训练；
- 管理个性化匹配与提醒；
- 查看第三方处理者类别、目的和范围；
- 反馈、申诉和人工复核渠道；
- 数据泄露应急通知流程。

### 2.4 第三方模型与跨境

- 明确模型、OCR、分析、推送、支付等供应商的处理角色；
- 合同约束用途、留存、再训练、访问和删除；
- 默认关闭供应商使用用户内容训练；
- 无法关闭时不得传输作文、画像和错误假设等内容；
- 数据出境前进行专项法律评估；
- 传输前字段最小化和去标识化；
- 保留供应商变更、故障和事件响应台账。

### 2.5 外部词汇工具数据

- 不向用户索取第三方词汇工具账号密码，不模拟用户登录；
- 官方接口必须使用用户可见、可撤销的授权，并限制到完成词汇训练所需范围；
- 文件、分享内容、截图和手动记录分别说明来源、用途、保存期限与可信等级；
- 学习报告截图默认裁剪或遮蔽昵称、头像、好友和其他无关信息；
- OCR 解析结果必须允许用户确认和更正；
- 优先接收词形、用户学习时间和学习状态，不批量复制第三方词典释义、例句、音频或助记内容；
- 断开工具后停止新增数据接收，历史数据可导出和删除；
- 第三方“已学”“已复习”和 App 使用时长不能直接推断为已掌握。

## 3. 未成年人

大多数考研用户是成年人，但仍可能存在低龄大学生或提前备考者。不满 14 周岁未成年人个人信息属于敏感个人信息，处理需取得监护人同意；还需遵守未成年人网络保护要求。[未成年人网络保护条例](https://www.moe.gov.cn/jyb_xxgk/moe_1777/moe_1778/202310/t20231025_1087333.html)

MVP 建议：

- 明确适用年龄与年龄确认；
- 未完成专门流程前，封闭测试限定 18 岁以上；
- 不以 App 商店年龄分级替代法律合规；
- 对未成年人不做付费诱导、排名压力或深夜提醒；
- 如未来开放，建立监护人同意、专门规则和合规审计。

## 4. 生成式 AI 与算法

《生成式人工智能服务管理暂行办法》要求提供者承担内容、数据和用户权益等义务；具体备案/登记责任与主体、模型、部署和开放方式有关。[中国政府网](https://www.gov.cn/zhengce/zhengceku/202307/content_6891752.htm)

上线前确认：

- 使用已备案模型 API 后，应用自身是否还需登记或备案；
- 是否构成面向境内公众的生成式 AI 服务；
- 自适应推荐是否涉及算法备案义务；
- ICP、公安联网备案、App 备案、等保等基础义务；
- 内容审核、投诉、日志和应急要求；
- 服务地区与供应商资质。

### 自动化决策要求

- 解释内容难度、任务量和阶段切换的主要依据；
- 用户可修改目标、时间、偏好和训练量；
- 对高影响错误假设和自动评分提供复核；
- 评估不同基础、英一/英二、应届/在职用户的偏差；
- 不用用户弱点进行操纵性营销；
- 不把付费档位与必要的水平匹配、公平评分绑定。

参考：[互联网信息服务算法推荐管理规定](https://www.cac.gov.cn/2022-01/04/c_1642894606364259.htm)。

## 5. AI 生成内容标识

《人工智能生成合成内容标识办法》自 2025 年 9 月 1 日起施行，覆盖文本、图片、音频、视频和虚拟场景，并规定显式与隐式标识。[中国网信网](https://www.cac.gov.cn/2025-03/14/c_1743654684782215.htm)

产品要求：

- 自动生成的反馈、参考表达、范例、改写、图片或音频有适用标识；
- 下载/导出内容保留所需显式和元数据标识；
- 用户协议说明标识方式和传播责任；
- 不提供恶意移除、篡改或伪造标识的功能；
- 保存必要生成记录、内容版本和来源；
- 确定性官方规则、人工审核内容和 AI 生成内容在界面上可区分。

## 6. 内容与版权

### 高风险内容

- 历年真题及其引用的外刊文章；
- 出版社、培训机构或教师的解析；
- 词典释义、例句、音频和词书编排；
- 作文范文、模板、讲义、图片、字体；
- 竞品知识地图、标签体系和数据库；
- 用户上传的付费资料。

国家版权局公开信息提醒，翻译型、注释型、习题集类教辅是易发侵权类别，数字化使用同样可能侵权。[国家版权局案例说明](https://www.ncac.gov.cn/xxfb/yjdt/201906/t20190626_49544.html)

### 内容策略

1. 每个内容单元记录来源、权利人、授权范围、期限、地区和媒介；
2. 真题能否商用逐项法律判断，不把“官方考试”自动等同公共领域；
3. 自研解析、语法地图和作文模板不得改写或拼接竞品内容；
4. 外刊、词典、音频、图片和字体优先正规授权；
5. 用户上传资料仅为本人提供必要处理，不默认进入公共题库或训练集；
6. 建立投诉、下架、证据保全和替换流程；
7. 检查生成内容对输入材料的近似复制风险；
8. 产品名与视觉资产上线前进行商标和版权检索。

## 7. 教育效果与营销

禁止或审慎使用：

- “保证上岸”“必提 20 分”“无限刷题必然提分”；
- 用单个用户故事推断普遍效果；
- 把相关性包装成因果；
- 隐瞒自动评分和错误假设的不确定性；
- 用虚构教师、学员、评价或录取结果；
- 宣称替代官方大纲或官方评分。

“无限定制训练量”应解释为用户可自由设置、系统可持续提供匹配路径，不得宣传为无节制刷题或无限生成低质内容。

## 8. 内容安全与考试诚信

- 拒绝考试作弊、实时答题和监考规避；
- 用户输入和上传内容进行安全处理；
- 题目、网页和上传文档中的指令视为不可信数据；
- 自动生成材料不得冒充真题；
- 保存真题安全边界，考试期间处理疑似泄题；
- 公开分享功能如上线，需有举报、审核和下架机制；
- 参考答案默认在用户独立作答后出现。
