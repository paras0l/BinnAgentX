# 核心 Agent 辅助与材料编排实现记录

更新日期：2026-07-19

## 目标

核心 Agent 的职责不是维持“看起来安全但帮不上忙”的状态，而是在用户已经独立尝试、仍然明确需要帮助时，尽可能把问题拆开并推进到下一次用户输出。同时保留三条边界：不代替用户交最终答案、不把内部降级状态当作正常体验文案、不把一次模型判断冒充能力提升证据。

## 用户真正需要帮助的点

| 时机 | Agent 应帮助什么 | 当前实现 | 后续用户动作 |
|---|---|---|---|
| 阅读前与独立作答前 | 明确任务、建立篇章检查顺序 | 工作区给出思考框架，不先揭示答案 | 保存独立 V1 |
| 用户选中词、句或证据后 | 解释语境义、句法、逻辑或证据边界 | 受约束模型分析选区；结果绑定原文区间 | 自动增加一项讲解后的临时迁移任务 |
| 阅读 V1 后仍卡住 | 从方向提醒逐级增加到局部线索、候选比较和最小示范 | H1 任务重述、H2 局部提示、H3 候选比较、H4 最小示范；必须逐级领取 | 每一级后都要亲自保存新版本；H3/H4 自动增加临时迁移任务 |
| 表达 V1 后不知道先改什么 | 只找最值得先改的一项，并引用 V1 | 远程模型优先反馈，受预算、超时和结构校验约束 | 用户亲自形成 V2 并保存修订引用 |
| 表达完成后想看更地道的组织 | 比较不同表达风格，而不是给唯一标准答案 | 表达实验室生成多风格对照与差异解释 | 自动增加风格讲解迁移任务，可把差异保存为学习资产 |
| 模型暂时失败 | 保住草稿与训练流程，仍给审核过的最小帮助 | 内部使用确定性降级并记录审计信息 | 页面不展示“模型暂不可用”等常驻标签，可稍后重试模型增强能力 |

临时任务与主任务分离：它接住讲解后立即迁移的需要，但不改变主任务版本、完成门或匹配证据。相同讲解来源只创建一次临时任务，避免重复堆积。

## 文章与表达材料生成

内容生成不再把旧 fixture 改名后冒充 AI 结果。正常运行必须调用已配置模型，生成失败或内容对象不合格时明确返回 `generation_failed`；旧材料重标只保留为命令行显式开启的离线测试选项。

阅读材料生成不是一次问答，而是一次完成以下内容资产：

- 全新的英文标题与正文，段落数对齐目标难度；
- 4–6 道可复用题目及各自答案、解释和 H1–H4；
- `foundation`、`standard`、`advanced` 三档题目，覆盖语境词义、语法填空、细节理解、主旨、推断、写作目的、句子插入、篇章逻辑和证据推理；
- 能在新文章中逐字定位的证据区间与哈希；
- 至少两项语法挑战，正确片段必须能在新文章中逐字定位；
- 平行重构任务和可迁移表达。

表达材料生成一次完成情境、受众、目的、论证动作、禁用方式、V1 最低要求、优先反馈检查、V2 成功标准和平行迁移任务。模型返回在进入内容池前经过 Pydantic 结构校验、JSON Schema 校验、内容哈希和整包完整性检查。超长字段与多余列表项可确定性收敛；证据或语法原文无法定位时不伪造，单项最多执行三轮“生成 → 审核 → 带意见修订”，仍不合格就失败。

复杂任务使用 LongCat thinking：推理位于独立 `reasoning_content`，系统只解析 `message.content` 中的最终 JSON。文章生成、内容审核、长句/段落分析和表达多风格复盘保留推理；短小的优先反馈关闭推理，以控制学习现场延迟。

## 审核 Agent

候选材料生成后由独立 Content Review Agent 负责主审核。审核维度包括事实自洽、答案唯一性、证据一致性、难度对齐、题型多样性、H1–H4 梯度和语言质量。所有分数至少为 4/5 且不存在 high/critical issue 才能标记为 `agent_reviewed`；`revise` 会把字段级修改要求交回下一轮生成，`reject` 或三轮仍不合格会终止该材料。

正式开发内容目录接受 `agent_reviewed` 与 `developer_reviewed`。人工审核不再逐条承担第一遍机械审核，而负责抽样复核、高风险争议、事实与权利升级，以及 `eligible_pilot` / `eligible_release` 发布门。

2026-07-18 使用已配置 LongCat 真实调用完成两级验证：

1. 单篇阅读探针生成 4 段、312 词、4 级提示、3 项语法挑战和 4 项可迁移表达，并通过内容 Schema。
2. 完整材料包生成 6 项内容：2 篇校准阅读、2 篇匹配阅读、2 项微型表达；整包验证错误为 0。

没有配置审核 Agent 时，生成结果仍标记为 `agent_generated_unreviewed`，不会进入正式内容目录。2026-07-19 的真实双 Agent 探针中，审核 Agent 连续识别并拒绝了源材料复用问题，证明退回门有效；随后生成输入已改为只接收题材标签和难度元数据，并增加 12 词连续片段去重门。修订后的真实生成探针成功产出 372 词新文章与 6 道题，覆盖语境词义、细节理解、语法填空、主旨、推断和证据推理三档难度，全部证据区间和 Schema 校验通过。LLM 审核仍不能替代教师抽检、版权复核、难度标定或真人效度实验。

## 表达实验室材料选取

微型表达内容现在显式声明 `source_reading_content_ids`。阅读任务结束后，`ExpressionMaterialMatcher` 不再按文件名或固定顺序配对，而是执行：

1. 过滤不支持当前英语一/英语二方向的候选；
2. 沿生成内容的 `_ai_` 谱系还原来源阅读 ID；
3. 优先选择与刚完成阅读显式关联的表达任务；
4. 结合剩余训练时间、词汇/句法负荷和本篇阅读最高提示等级排序；
5. 保存候选版本、最终选择、策略版本和原因码，进入运行回放。

真实生成包验证中，系统从 2 个表达候选里选中了与新生成 `matched_reading_01` 同源的表达任务；当阅读最高帮助达到 H3 时，决策同时记录 `source_reading_linked` 和 `lighter_expression_load_after_high_support`。

如果历史内容尚未声明来源关联，选材器允许在同考试方向候选中保守回退，并记录 `source_link_unavailable_fallback`，便于后续内容治理发现和补齐关联，而不会悄悄假装已完成精准迁移。

## 尚未关闭的证据门槛

- 审核 Agent 的抽样人工一致性评估、事实核验、权利状态升级和难度标定；
- 模型调用转入 Worker/Outbox 后的异步恢复、重试观测和成本报警；
- 账号级跨浏览器学习历史与长期学习者记忆；
- H1–H4、表达反馈与材料匹配对真人学习效果的对照验证。
