# 桌面学习工作区实现记录

> 状态：首个纵向切片 H0 桌面主路径 v0.4
>
> 日期：2026-07-16
>
> 证据边界：证明开发内容与本地 PostgreSQL 下的桌面交互、恢复和 API 主路径可运行；不证明真人可用性、材料匹配效度或学习效果。

## 1. 本次交付结果

学习端从静态工程入口升级为可操作的电脑浏览器工作区，打通：

`首次背景 → 校准 A → 校准 B → 匹配阅读与精确标记 → 微型表达 V1 → 难度反馈 → 下一任务占位 → 完成`

界面没有聊天入口。后台编排通过运行和任务 API 驱动，用户前台只看到当前学习目标、材料、自己的认知动作、保存状态和可理解的进步证据。

本增量只完成 H0 无帮助路径。任务聚合已经支持分级介入、V2 和修订引用，但学习端尚无自动提示/反馈生成通道，因此界面没有伪造 AI 反馈或 V1/V2 差异。

## 2. 安全工作区读模型

新增：

`GET /learner/v1/runs/{workflow_run_id}/workspace`

它在一个响应中返回：

- 当前 `VerticalSliceRun` 投影；
- 当前 `LearningTask` 投影；
- 通过内容权利、Manifest 和哈希验证后允许呈现的内容字段。

阅读任务只返回标题、段落、可选语义类型、题干和选项；不会返回：

- `correct_answer`；
- `hints`；
- `public_explanation`；
- 最小证据金标；
- 常见错误候选。

微型表达只返回情境、受众、目的、论证动作、可选局部表达、禁止机械使用规则、字数和 V1 最低要求；不会返回自动反馈内部规则和拒判配置。

内容仍由服务端权利门控制，浏览器不能直接读取仓库内容文件。工作区快照避免浏览器依次请求运行、任务和内容造成的网络瀑布，也减少三份状态不同步的风险。

## 3. 首次背景

首次背景只收集会改变本次任务负荷的字段：

- 英语一或英语二；
- 目标分数；
- 每周可用时间和单次训练时间；
- 自报阅读起点；
- 是否见过完整考研英语试卷。

可选细分偏好继续使用保守默认值。页面不展示 Agent、工作流、置信区间、模型或能力分，不产生词汇量、预测提分和“已掌握”结论。

## 4. 阅读与语义标记工作区

桌面工作区采用稳定双栏：左侧完整材料，右侧任务、选择、解释和标记编辑器。顶部固定展示阶段、当前保存事实和剩余步骤，底部展示当前标记数和最高帮助等级。

标记流程：

1. 用户先用鼠标或键盘选择同一段落内的原文；
2. 浏览器根据真实 DOM Range 计算段落 ID 和精确字符偏移；
3. 用户选择观点、证据、逻辑、不确定点或可迁移表达；
4. 用户必须写下自己的解释；
5. 浏览器计算选区 SHA-256；
6. 服务端再次验证内容版本、偏移、片段和哈希后追加事实。

界面不奖励标记数量。保存成功只提示“本次把判断绑定到精确原文”，不写成理解已稳定或能力已掌握。

匹配阅读完成前至少需要一条带解释的语义标记和一次独立作答。校准阶段允许标记，但不强制用数量填满流程。

### 4.1 首轮体验修正

根据电脑浏览器实机体验，阅读工作区补充了五项基础交互：

- 用户选中原文后，观点、证据、逻辑和“这句看不懂”动作直接出现在选区旁；右栏自动回到顶部，完整标记编辑器不再藏在输出区下方；
- 右栏常驻“完成本步还需要”，区分未完成、已完成和“遇到困难时”的可选动作；
- 校准阶段也明确提供“看不懂”入口，并让用户区分词义、长句、指代、逻辑和背景卡点；
- 阅读与表达均提供可展开的通用思考起点和无关题目方法示例，明确不对应当前题目、不给当前答案；
- 普通提交、保存和选择动作改用描边或浅底按钮，只在首次开始、最终确认等少数高承诺动作保留实色强调。

这里的“思考起点”是静态 H0 支架，不读取学习者模型、不算一次分级提示，也不提高任务的 `highest_hint_level`。个性化 H1–H4 仍属于后续增量，不能用这组静态文案冒充已实现的自适应帮助。

### 4.2 标记回看

保存标记后不再只显示数量：

- 工作区任务读模型返回当前内容版本的标记 ID、类型、精确片段、用户解释和创建时间；不返回旧材料版本的标记；
- 原文保留轻量、按类型区分的标记痕迹，点击标记可定位右栏记录；
- 右栏常驻“已保存标记”入口，保存后自动展开，刷新恢复时也可直接回看；
- 同一段重叠标记按字符边界分段呈现，避免为了展示而改变服务端保存的精确坐标。

本增量只提供只读回看。编辑和删除会改变学习证据，需要单独定义追加事实、审计和撤销语义后再实现。

## 5. 输出体验

阅读任务要求用户先选择判断，再用自己的话解释原文与选择之间的关系。系统不会预先给完整解析。

微型表达工作区同时呈现：

- 写作情境、受众和目的；
- 需要完成的论证动作；
- 最多一条可选局部表达资源；
- V1 最低内容要求和实时英文词数；
- 用户自己的连续文本编辑区。

允许从关键词、中英混合或不完整句起步，但提交前必须达到内容包规定的英文词数范围。前端不会一键翻译或生成成品段落。

## 6. 保存、恢复与冲突

浏览器只使用两个版本化本地键：

- `binnagent:learner-resume:v1`：只保存当前运行 ID；
- `binnagent:learner-draft:v1:{task_id}`：只保存任务 ID、内容版本、选择、当前文字和更新时间。

不把完整服务端响应、画像、答案金标、模型信息或控制权限写入 `localStorage`。本地存储被禁用时，界面明确显示草稿只在当前页面，不伪称已保存。

草稿绑定内容版本；材料替换后旧草稿不会自动套用到新正文。服务端保存成功后清理对应本地草稿。刷新时只用运行 ID 重新读取权威工作区快照。

所有写请求携带独立 `Idempotency-Key` 和 `expected_version`。版本冲突返回公开 `SESSION_CONFLICT`，不会用本地旧状态覆盖服务端事实。一次操作在“作答已保存、完成或推进失败”的中间点中断时，重试会识别已有服务端事实并从未完成步骤继续。

## 7. 键盘、输入法和无障碍

已实现：

- `Ctrl/Cmd + S` 保存本地草稿；
- `Alt/Option + 1/2` 在材料和输出间切换焦点；
- `Alt/Option + M` 处理当前选区；
- `Alt/Option + U` 标记当前选区为不确定；
- 非输入状态下 `?` 打开或关闭快捷键帮助；
- 中文输入法组合态期间不触发保存、提交或快捷键；
- 状态变化使用文字和 `aria-live`，不只依赖颜色；
- 首次背景和工作区均通过 axe Chromium 扫描。

当前只把电脑浏览器作为产品终端，不为窄屏移动端重新排版核心工作区。

## 8. 测试证据

自动化覆盖：

- 安全工作区快照与真实 PostgreSQL 内容读取；
- 阅读响应不包含正确答案、提示和解析；
- 表达响应不包含自动反馈内部规则；
- 运行指针和草稿的版本化、最小化存储；
- 内容版本变化后拒绝恢复旧草稿；
- 首次背景可操作且无聊天入口；
- 从首次背景进入双栏阅读工作区；
- 页面级 Chromium axe 无障碍扫描；
- 原有任务、运行、幂等、迁移、控制回放和内容契约回归。

浏览器 E2E 使用裁剪后的 API 固定响应验证前端状态；真实 API/数据库语义由 Python 集成测试验证。二者组合能定位前端与领域问题，但不等同于真人端到端研究。
