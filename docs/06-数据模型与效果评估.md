# 数据模型与效果评估

> 本文描述学习者模型、阶段证据和评估口径，不是数据库或技术架构设计。

## 1. 数据原则

1. 原始学习证据、用户显式输入和系统推断分开；
2. 所有知识状态与错误假设可追溯、可纠正；
3. 以阶段内进步和迁移为主要结果，不以完成量替代效果；
4. 只收集完成训练所必需的数据；
5. 模型同时表达“会什么”和“不确定什么”；
6. 三阶段历史保留，不能用新状态覆盖成长过程。

## 2. 学习者模型结构

### 2.1 目标模型

- exam_type：英语一/英语二；
- exam_date：考试日期；
- target_score：目标分；
- target_profile：保底/标准/冲刺；
- stage_goal：当前阶段目标；
- priority_modules：用户或系统确认的优先模块。

### 2.2 学习时间模型

- weekly_minutes；
- available_windows；
- preferred_session_length；
- unavailable_dates；
- user_selected_volume；
- completion_capacity：基于历史估计，不覆盖用户选择；
- review_load：未来复习压力。

### 2.3 语言背景

- 四六级或既往考研成绩；
- 英语学习年限与中断时间；
- 阅读/写作经验；
- 母语和其他语言影响（用户自愿提供）；
- 使用过的教材、词书和课程；
- 是否见过某批评估/真题材料。

背景只用于训练匹配，不能直接代替能力证据。

### 2.4 显式偏好

- 阅读栏宽、字号与行距；
- 是否偏好快捷键或鼠标操作；
- 是否计时；
- 任务长度；
- 反馈密度；
- 使用的外部词汇工具及数据提交偏好；
- 训练量单位；
- 提醒频率；
- 用户主动设置的重点/回避项。

显式偏好可影响呈现和编排，但不能关闭必要复习、独立作答或考试规则。

### 2.5 局部知识状态

| 对象 | 主要维度 |
|---|---|
| 词汇 | 外部最近学习、目标词义、阅读识别、篇章理解、翻译处理、主动表达、语境迁移、保持率 |
| 词组 | 含义、结构、阅读识别、翻译/写作使用 |
| 语法 | 识别、拆解、解释、改写、迁移、稳定性 |
| 阅读 | 各题型正确率、证据链、耗时、难度、稳定性 |
| 翻译 | 理解、完整、中文表达、速度 |
| 写作 | 审题、内容、结构、语言、语域、模板迁移 |
| 考试执行 | 模块用时、做题顺序、检查、整套稳定性 |

每个状态保存：证据数量、新鲜度、难度、独立程度、材料是否见过和置信度。

阅读材料匹配使用独立的动态画像快照，至少区分：语境词义、句法处理、证据定位、推理、阅读速度、提示依赖和作答信心校准。快照同时保存生成时间、证据窗口、各维度证据量与不确定性；它是可重算的匹配依据，不是对用户的永久等级标签。

词汇不能只保存一个熟悉度分数。建议按目标词义维护独立状态：

`外部新近学习 → 待语境验证 → 句中识别 → 篇章读懂 → 翻译处理 → 受控表达 → 主动表达 → 延迟稳定`

其中“翻译处理”和“主动表达”不是所有词的必经状态。每个词义另存 `use_target`：

- `receptive`：以阅读识别和篇章理解为目标；
- `translation`：需要在英译汉中准确处理；
- `productive`：值得进入改写和作文主动表达。

同时记录 `prompt_level`，区分看到目标词提醒后使用与无提示自然使用。

### 2.6 反复错误假设

最小字段：

- hypothesis_id；
- 可读描述；
- 触发证据；
- 相关知识节点；
- 影响题型；
- 频率与影响；
- 当前置信度；
- 支持证据与反对证据；
- 验证任务；
- 状态：待验证/确认/改善/推翻/复查；
- 最近验证时间。

错误假设是可变的诊断对象，不能当作用户永久标签。

### 2.7 阶段状态

- current_stage；
- stage_started_at；
- planned_stage_end；
- entry_evidence；
- stage_targets；
- progress_evidence；
- exit_criteria；
- readiness；
- manual_adjustments 与原因。

## 3. 核心业务对象

| 对象 | 关键内容 | 用途 |
|---|---|---|
| 内容单元 | 真题/句子/词汇/语法/作文题、难度、权利、版本 | 训练材料 |
| 知识节点 | 前置关系、例句、任务形式、点亮规则 | 知识地图 |
| 训练周期 | 周期目标、训练量、任务路径 | 连续编排 |
| 完整任务 | 阶段、节点、材料、处理、输出、迁移 | 主要学习单位 |
| 作答尝试 | 答案/文字、耗时、提示、信心、版本 | 原始证据 |
| 反馈 | 规则结果、自动反馈、教师纠正、版本 | 学习与质控 |
| 外部工具连接 | 工具、连接方式、授权范围、同步状态 | 词汇积累协调 |
| 外部学习记录 | 来源、时间、词量/时长、原始文件、证据等级 | 投入与计划校准 |
| 最近词汇记忆 | 规范词形、目标词义、来源、学习时间、适用目标、迁移队列 | 承接外部记忆 |
| 词汇验证 | 目标词义/词组、语境、作答或输出、延迟结果 | 掌握度证据 |
| 复习项 | 原因、方式、到期、完成和保持 | 延迟巩固 |
| 错误假设 | 假设、证据、验证和状态 | 针对性训练 |
| 阶段报告 | 起点、当前、迁移、稳定项、风险项 | 进步展示 |
| 校准会话 | 背景输入、短材料、作答、证据跨度、耗时、提示、信心、熟题与难度反馈 | 建立和修正初始画像 |
| 材料难度画像 | 考试类型、主题、篇幅、词汇、句法、推理、题型、时间压力、权利与版本 | 多维选材 |
| 材料匹配决策 | 画像快照、候选集、约束、排序理由、最终选择和用户反馈 | 解释选材与离线回放 |

## 4. 学习证据强度

由强到弱：

1. 未见材料、限时、独立完成的迁移任务；
2. 跨时间独立完成的同技能任务；
3. 独立完成的当前任务；
4. 低级提示后完成；
5. 高级提示或看解析后完成；
6. 第三方逐词记录或用户导入的学习报告；
7. 用户自评、其他 App 使用时长或仅浏览内容。

低强度证据可以决定“需要验证”，不能单独决定“稳定掌握”。

## 5. 阶段内进步判定

### 必须同时考虑

- 同难度表现变化；
- 新材料迁移；
- 7 天/21 天延迟保持；
- 独立程度；
- 耗时变化；
- 文字作品质量；
- 错误假设频率变化；
- 训练量和材料熟悉度。

### 示例

不合格结论：

> 本周做了 300 道题，阅读能力提升 20%。

合格结论：

> 在未见、同难度的 12 道推断题中，独立正确率由 50% 提升到 67%；“无原文证据过度推断”由 5 次降至 2 次。样本仍较少，暂标为发展中。

## 6. 训练量指标

系统区分：

- 用户设定量；
- 系统建议量；
- 实际完成量；
- 有效训练量；
- 同质重复量；
- 到期复习负荷；
- 文字输出量；
- 超负荷风险。

有效训练量必须满足当前路径相关、难度匹配并产生有效证据。不能以无限供给鼓励无目的刷题。

## 7. 产品指标

### 北极星

阶段有效进步单元（SVPU）。

即时进步事件（IPE）是领先信号，不是北极星：它记录同一训练内用户在平行任务、修订或提示减少后的具体改善；只有通过未见、延迟、少提示或无提示验证，才能转化为 SVPU。

### 领先指标

- 首次背景设置完成率和中位耗时；
- 从注册到开始第一篇匹配阅读的时间；
- 前期画像校准完成率；
- 首篇匹配阅读的开始率、完成率和中途退出率；
- 用户“太难/合适/太简单”反馈分布及下一篇纠偏成功率；
- 首周 3–5 次任务后的画像稳定度和用户纠正率；
- 校准后次日、3 日和 7 日回访率，并按首篇难度反馈分层观察；
- 用户查看/纠正学习者模型的比例；
- 完整任务完成率；
- 即时进步事件发生率及其转化为 SVPU 的比例；
- 认知产出率：材料和 AI 反馈产生可验证判断、解释、重构或表达的比例；
- 标记质量：最小证据跨度、语义标签和观点—证据关系准确率；
- 写作启动时间、完全空白率和独立观点/理由/例子数量；
- 帮助撤除率：同类任务达到成功所需提示等级的下降；
- 用户原创、用户改写、AI 建议和参考内容的占比；
- 自定训练量与实际有效完成量的匹配度；
- 外部词汇计划提交率和验证覆盖率；
- 最近词汇进入匹配阅读的覆盖率与等待时间；
- 最近词汇从语境识别到翻译/适当表达的迁移率；
- 语法节点从识别到迁移的转化率；
- 每周文字表达完成率；
- 错误假设验证率和推翻率；
- 阶段切换后两周的任务适配率。

首次体验的漏斗事实源、事件字段、隐私边界和控制舱看板定义见[首次体验页面状态与埋点清单](17-首次体验页面状态与埋点清单.md)。

### 学习效果指标

- 同难度独立正确率；
- 新材料迁移率；
- 7 天/21 天保持率；
- 各题型正确率和耗时；
- 翻译/写作盲评变化；
- 模板换题迁移成功率；
- 真题重复表现与未见材料表现的差距；
- 整套模拟分数、用时和稳定性。

### 护栏指标

- 任务过难/过易率；
- 同质重复比例；
- 错误假设误判率；
- 英一/英二配置错误率；
- 自动解析/评分错误报告率；
- 过度标记率和标记后不解释率；
- AI 内容占成品比例与一键接受整段修改比例；
- 感知进步提高但延迟迁移不变或下降的比例；
- 交互耗时增加但没有学习收益的比例；
- 用户设定过载后的放弃率；
- 数据更正、删除和导出成功率；
- 单个有效进步单元的成本和延迟。

## 8. 不作为核心成果的指标

- App 在线时长；
- 连续打卡天数；
- 单纯题量、外部已学词量或复习量；
- 知识地图点亮总数；
- 看完内容数量；
- 高亮、批注和收藏数量；
- AI 修改后但未经用户独立复现的作文分数；
- 记住原题答案后的重做正确率；
- 未经校准的“预计提分”。

## 9. 离线评测集

| 评测集 | 样例 | 指标 |
|---|---|---|
| 水平匹配 | 同一用户不同局部能力 | 难度适配率 |
| 冷启动匹配 | 无成绩、成绩过时、自评矛盾、猜对、熟题、低置信度 | 场景约束满足率、过难/过易风险率、纠偏步数 |
| 任务连贯 | 训练量增减后的任务路径 | 目标一致性、碎片化率 |
| 知识状态 | 提示后正确、重复材料、迁移失败 | 更新正确率 |
| 错误假设 | 同错异因、同因跨题型、反证 | 教师一致性、误判率 |
| 语法地图 | 看懂、能拆、能改、能迁移 | 点亮状态准确率 |
| 外部词汇证据 | API、导入文件、截图、手填、使用时长 | 解析准确率、证据误用率 |
| 最近词汇队列 | 新词过多、词义未知、材料不自然、等待过久 | 归一准确率、覆盖率、自然度、等待时间 |
| 词汇验证 | 能认不能读、能读不能译、能译不能写、熟词僻义、词组错误 | 状态准确率、保持率、跨语境迁移率、主动使用率 |
| 文字反馈 | 可接受变体、偏题、语法错、模板僵化 | 教师一致性、误报率 |
| 阶段切换 | 基础不足、临近考试、目标变化 | 规则正确率 |
| 英一/英二 | 翻译和写作配置 | 配置准确率 |
| 真题多轮 | 记忆答案但迁移失败 | 防虚假进步率 |

关键样本由教研、产品和数据共同审查，不能只由另一个模型评分。

## 10. 资源受限条件下的验证路线

当前无法开展人工验证实验，因此不能把画像准确性、学习效果或留存改善写成已验证结论。首版按以下顺序降低工程和产品风险：

### A. 上线前离线回放

用合成学习者、旧项目可追溯的工程样例和自建边界案例覆盖无成绩、成绩过时、自评矛盾、猜对、熟题、低置信度及局部能力不均衡。验证匹配规则可解释、可重放，并在证据不足时选择保守材料或继续校准。

### B. 上线后的自然使用校准

在不额外招募实验样本的前提下，观察首次设置耗时、校准完成率、首篇开始/完成/退出、难度反馈、下一篇纠偏和前 3–5 次任务的画像变化。该数据只能验证产品流程和匹配稳定性，不能证明教学效果。

### C. 留存与匹配联合观察

记录次日、3 日和 7 日回访，但必须按首篇“太难/合适/太简单”、完成状态和提示依赖分层。留存是产品健康指标，不替代新材料迁移、延迟保持或独立表现等学习证据。

### D. 后续验证门槛

未来条件允许时，再引入教研盲审、未见材料、盲评作文和英一/英二分层验证。在此之前不发布精确等级、预计提分、能力诊断准确率或因果效果声明。

## 11. 数据与审计

任何自动状态更新至少记录：

- 使用的原始证据；
- 规则/模型/内容版本；
- 更新前后状态；
- 是否来自见过的材料；
- 系统置信度；
- 用户或教师是否纠正；
- 对后续任务和阶段的影响。

用户应能查看并纠正目标、时间、语言背景、偏好和错误假设；删除后不得继续影响任务。

## 12. IRT/CAT 测量视图

IRT/CAT 是学习者模型之外的一套版本化测量视图，只接收已标定题目、可比较施测条件和合格评分产生的证据。

至少保存：

- 量尺、模型和题目参数版本；
- θ 后验及标准误/可信区间；
- 题目/testlet、选择约束和内容蓝图覆盖；
- 作答有效性、是否见过、提示和评分器版本；
- 停止规则、停止原因和题库信息缺口；
- DIF、局部依赖、参数漂移和曝光状态；
- Agent 的后续决策如何引用该测量结果。

普通练习、外部已学记录、看过解析后的作答和真题记忆不能直接进入 IRT 能力更新。CAT 结果也不能覆盖词汇、语法、阅读错因、翻译和写作等细粒度状态。完整设计见[后台 Agent 与 IRT/CAT 技术路线](13-Agent与IRT-CAT技术路线.md)。

## 13. 已知数据问题

- 用户可能猜对、抄答案、忘记计时或已见真题；
- 外部工具显示“已学”或“已复习”不等于阅读和写作会用；
- 第三方 App 使用时长不能证明用户在背词，也不能提供逐词掌握状态；
- 截图、导出文件和手填记录可能遗漏、过时或被修改；
- 导入词形不等于确定用户学习了哪一个词义；
- 为覆盖最近词而生成不自然的文章，会同时污染阅读难度和词汇证据；
- 并非所有阅读词都值得进入作文，强制使用可能降低表达自然度；
- 高训练量可能造成短期熟练而非长期保持；
- 外部导入文件的解析会引入题目和文字错误；
- 不同年份真题难度不能直接比较；
- 语言背景与能力相关但不是因果；
- 临考自然进步会混淆产品效果。
- 阅读题共享文章会产生 testlet 局部依赖，若按独立题使用普通 IRT，可能高估测量精度；
- CAT 选择造成非随机缺失和不同用户题目路径不同，分析时不能按普通固定测验处理；
- 自动写作/翻译评分误差若不进入测量模型，会污染开放题能力估计；
- 题目曝光、真题泄露、参数漂移和不同人群 DIF 会破坏量尺可比性。

所有进步报告都应披露证据量、材料熟悉度和不确定性。
