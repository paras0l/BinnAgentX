# 技术 Spike 章程与开工门槛

> 状态：Sprint 0 基线 v0.1
> 日期：2026-07-15
> 适用范围：首轮两周技术 Spike；不是正式 MVP、封闭试点或学习效果实验。

## 1. 授权结论

项目当前授权启动的是一个**范围受限、结果可推翻、只使用合成身份和合法开发样例的技术 Spike**。

本轮只回答：

- 桌面长文读写工作区能否稳定实现；
- 用户、AI、系统与参考内容的归属能否完整还原；
- 工作流能否暂停、恢复、幂等、降级和审计；
- 用户端与开发者控制舱能否真正隔离；
- 模型、内容、规则和数据结构能否版本化与替换；
- 成本、延迟、日志和数据删除路径能否被观测与验证。

本轮不能回答：

- 用户是否愿意持续使用或付费；
- 内容匹配是否真实改善考研英语能力；
- 自动反馈是否达到教师一致性；
- IPE 是否能够转化为稳定的 SVPU；
- IRT/CAT 是否有效、公平或可以用于用户决策；
- 预计提分、稳定掌握或最终考试成绩。

## 2. 首轮范围

### 2.1 必须实现的能力边界

1. 两个独立桌面 Web 构建：用户学习端与开发者控制舱；
2. 一个贯通语境实验室和微型表达任务的纵向切片；
3. 不可变作答、标记、AI 介入、修订和迁移占位记录；
4. 框架无关的确定性业务状态机；
5. 模型网关及至少两个可替换适配器，允许其中一个为确定性假实现；
6. 暂停恢复、幂等、超时、拒判、人工复核和审计；
7. 首次体验 v1 分析事件契约；
8. 合法开发内容包、权利门和版本门；
9. 数据导出、纠正和删除的技术演练；
10. 桌面浏览器自动化、属性测试、故障注入和固定回归场景。

### 2.2 明确排除

- 正式注册、手机号、微信登录、支付和商业套餐；
- 手机、平板、原生客户端、语音、手写、相机、截图/OCR 和跨终端接续；
- 真实考生个人数据和真实第三方词汇账号；
- 未授权真题、解析、范文、词典内容或外刊全文；
- 自动作文总分、预计提分和高影响阶段切换；
- 用户可见 IRT/CAT、实时 CAT、MST 正式路由；
- 微服务、Kubernetes、独立向量数据库、图数据库和多 Agent 自由协商；
- 推送、离线优先、社区、教师商城和公开分享。

## 3. 暂定业务与工程输入

以下结论只服务技术 Spike，可以在 ADR 评审后替换，不代表生产承诺。

| 项目 | Spike 暂定值 | 约束 |
|---|---|---|
| 目标市场 | 中国大陆考研英语一/英语二考生 | 只影响术语、时区与合规风险登记；Spike 不接收真人数据 |
| 运行环境 | 本地开发与隔离 CI | 不进行公开互联网发布 |
| 身份 | 合成学习者；本地开发身份适配器 | 不实现正式登录，不保存手机号/微信标识 |
| 控制舱角色 | `developer_reviewer` 单角色 | 默认只读；写操作仍需理由、版本和审计 |
| 用户浏览器 | Chrome、Edge、Safari 当前及上一稳定大版本；Firefox 当前版做兼容冒烟 | 最终矩阵以后续使用数据调整 |
| 桌面视口 | 最低工程假设 `1280×720`；推荐 `1440×900` 及以上 | 必须验证 200% 浏览器缩放；不压缩为移动布局 |
| 时区 | 数据存 UTC，界面默认 `Asia/Shanghai` | 首次体验事件仍使用 UTC |
| 内容 | 项目自写或确认可用于开发的内容包 | 每个版本必须通过权利门；不得冒充真题 |
| 模型数据 | 仅使用合成输入和项目自有内容 | 日志不得保存完整正文、完整提示或模型思维过程 |
| 模型调用 | 单个纵向切片最多 3 次远程模型调用 | 超出必须触发预算拒绝或确定性降级 |
| 延迟假设 | 确定性操作 p95 小于 500ms；远程模型节点 p95 小于 20s | 只是 Spike 容量假设，必须记录真实分布 |
| 成本假设 | 单次完整纵向切片远程模型成本上限暂设 0.20 美元 | 必须可配置、可观测；不是商业定价依据 |
| 数据留存 | 合成运行数据默认 30 天；可立即删除 | 生产留存策略另行完成影响评估 |

## 4. 首轮纵向切片

第一条切片固定为：

`最小背景 → 两段阅读校准 → 自然语言画像摘要 → 第一篇匹配阅读 → 语义标记与解释 → 最小提示 → 用户重新输出 → 微型表达 → 保存/恢复 → 难度反馈 → 审计回放`

详细页面、状态、桌面布局与验收见[首个纵向切片与桌面交互规格](20-首个纵向切片与桌面交互规格.md)。

## 5. 证据与产品文案上限

技术原型中只能使用以下表述：

- “系统成功保存并恢复了任务”；
- “该合成场景满足契约”；
- “本次操作所需提示减少”；
- “这次结果仍需要新材料或延迟验证”；
- “自动判断被拒判/转为复核”。

禁止使用：

- “你已经掌握”；
- “能力提升了 X%”；
- “预计提高 X 分”；
- “系统比教师更准确”；
- “已证明有效”；
- “IRT/CAT 精确测出你的水平”。

## 6. 开工前 Definition of Ready

只有以下项目全部满足，才开始纵向切片业务代码：

- [x] 本章程与纵向切片范围已进入同一个可追溯 Git 基线；
- [x] 用户端与控制舱边界、开发身份和控制角色已经冻结；
- [x] 纵向切片状态、成功条件、降级条件和公开错误码已有契约；
- [x] 最小领域 JSON Schema 通过样例校验；
- [x] 至少 4 篇阅读、2 个微型表达任务进入合法开发内容包；
- [x] 每个内容版本都有来源、权利状态、难度假设和答案证据；
- [x] 桌面低保真布局、最低视口和键盘路径已有验收稿；
- [x] 合成正常、边界、恶意输入和故障场景已经列入回归矩阵；
- [x] 依赖版本、目录、环境变量、测试命令和 CI 门槛已经书面冻结；
- [x] 日志禁记字段、删除演练和模型数据政策已经明确；
- [x] 未完成人类证据被记录为发布前风险，而不是被合成测试代替。

截至 2026-07-15，机器契约、内容完整性和 25 个固定场景已完成本地校验；本章程、纵向切片规格、ADR、契约与固定样例随本轮 Sprint 0 准备提交进入同一 Git 基线。上述清单已关闭，可以开始工程骨架和技术 Spike，但不代表已经满足正式 MVP 或真人试点门槛。

## 7. Spike 退出门槛

- 任意业务命令重复执行不产生重复证据、重复事件或重复扣费；
- 进程重启后可以从最近已确认节点恢复；
- 用户原文、AI 建议、用户修订与参考内容归属可还原；
- 用户端、控制舱的会话、API client、构建产物和权限真正隔离；
- 模型超时、格式错误、预算耗尽和供应商失败都有确定性降级；
- 编辑器旧版本内容可以迁移或明确拒绝，不能静默损坏；
- 内容权利不合格时无法呈现材料；
- 日志、事件和追踪抽查不含禁记内容；
- 合成场景、契约、属性测试和桌面 E2E 全部通过；
- ADR 明确哪些方案保留、修改或废弃；
- 最终报告不包含真人需求、教学效果和评分效度结论。

## 8. 立即停止条件

出现以下任一情况时停止扩展功能，先修复基础边界：

- 用户正文或凭据进入普通日志；
- 模型可以直接写入稳定掌握或阶段切换；
- 重试会重复扣费或生成重复证据；
- 控制舱能绕过领域服务直接改库；
- 用户端包含控制 API 或内部技术字段；
- 无权利状态的内容可以被发布；
- AI 结果覆盖用户原始作品；
- 为赶进度把自动评分、IRT/CAT 或真实真题提前放入用户路径。
