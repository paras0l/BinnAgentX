# 微型表达单项反馈与 V2 闭环实现记录

> 状态：表达实验室确定性反馈纵向切片 v0.1
>
> 日期：2026-07-16
>
> 证据边界：证明表达 V1、单项规则反馈、学习者 V2 和修订引用可审计运行；不证明反馈判断正确、V2 已改善或模型 Agent 已接入。

## 1. 本次完成的闭环

微型表达任务现在支持两条路径：

`独立 V1 → 保持 H0 → 直接完成`

`独立 V1 → 主动查看一项优先检查 → 学习者修改 V2 → 保存 V1/反馈/V2 引用 → 完成`

反馈之前必须存在学习者独立 V1。反馈发放后，任务不能直接完成；用户必须修改原作品、保存 `hinted_low` V2，并建立引用该反馈的 `RevisionEvent`。V1 不被覆盖，界面只说明“是否改善仍待验证”。

## 2. 规则型反馈边界

本增量没有调用语言模型，也不对作文打分。每个版本化微型表达内容包含：

- 按优先顺序排列的表面信号检查；
- 每项检查对应的一条人工预审反馈；
- 所有检查均命中时使用的预审兜底检查；
- 原有输出长度、机械照搬和自动反馈拒判边界。

适配器只检查用户 V1 中是否出现版本化信号词，选择第一个未出现的检查方向。它不能证明观点、逻辑或表达真的缺失，因此用户端明确称为“规则检查”，不称为评分或智能批改。文本不足以支持最小判断时拒绝发放，不生成替代句或完整段落。

## 3. API 与不可变证据

学习端新增：

`POST /learner/v1/tasks/{task_id}/feedback/priority`

该命令：

- 仅接受 `micro_expression`；
- 必须引用当前材料的最新 V1；
- 每个当前材料最多发放一次；
- 固定登记为 H2 `priority_feedback`；
- 保存内容版本、输入版本、原因码、反馈文本哈希和实际发放文本；
- 不向学习端暴露适配器名和内部提示版本；
- 相同幂等键只产生一个介入事实。

控制舱仍可通过任务回放查看适配器、提示版本、内容哈希和引用链，但默认不返回完整学习者作品。

## 4. 桌面表达工作区

表达工作区现在把一次提交拆成：

1. 用户亲自完成 35–90 词 V1；
2. 选择保持 H0，或查看一次单项反馈；
3. 反馈卡只展示一个优先检查方向，并明确不提供替代段落；
4. 编辑器恢复用户 V1，由用户亲自形成 V2；
5. 并列保留 V1 和 V2，展示反馈引用状态；
6. 用户确认版本后完成任务。

沿用阅读工作区的本地草稿、幂等保存、刷新恢复、IME 保护和 Console 错误门禁。依据 React/Next 性能基线，没有增加客户端数据请求瀑布或新的重型依赖。

## 5. 自动化证据

自动化覆盖：

- 内容契约要求每个表达材料提供检查序列和兜底反馈；
- 第一个缺失表面信号决定唯一反馈；
- V1 太短时拒绝发放；
- 学习端可以发放反馈，重复命令不会产生第二条介入；
- 学习端响应不暴露模型适配器；
- 控制回放保留适配器、提示版本和哈希证据，不返回完整 V1；
- 浏览器完成表达 V1 → 单项反馈 → 用户 V2 → 修订引用；
- 浏览器 Console、页面异常、失败网络请求和 axe 无障碍扫描进入失败门禁。

## 6. 明确未完成

- 基于语义而非表面词项判断最高优先问题；
- 远程模型适配器、Worker/Outbox 预约、重试与缓存；
- 提示注入对抗评测与真实模型输出效度验证；
- 用户对反馈的采用、拒绝或“说明不准确”动作；
- 自动判断 V2 是否改善；
- 平行新题和延迟无提示迁移；
- 完整考研作文、翻译题和英一/英二评分量表；
- 真人考生可用性与教师反馈一致性验证。

该受约束工具接口已在后续增量实现：输入只引用已保存版本，输出必须符合结构化 Schema，失败时回落到本增量的确定性规则反馈，并把预算、超时和拒判写入安全审计记录。详见[32-受约束模型网关与确定性降级实现记录](32-受约束模型网关与确定性降级实现记录.md)。
