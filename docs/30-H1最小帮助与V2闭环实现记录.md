# H1 最小帮助与 V2 闭环实现记录

> 状态：阅读任务 H1 确定性纵向切片 v0.1
>
> 日期：2026-07-16
>
> 证据边界：证明 V1、预审 H1、学习者 V2 和修订引用可审计运行；不证明提示教学有效、V2 一定改善或模型 Agent 已接入。

## 1. 本次完成的闭环

阅读任务现在支持两条明确路径：

`独立 V1 → 不领取帮助 → H0 完成`

`独立 V1 → 主动领取 H1 → 学习者修改 V2 → 保存 V1/H1/V2 引用 → 完成`

H1 之前必须存在独立 V1。H1 发放后，任务不能直接完成；V2 必须改变输出，并以 `hinted_low` 保存。随后必须建立修订引用，且界面只写“是否改善仍待验证”，不把一次修改伪装成掌握。

## 2. H1 内容边界

本增量没有调用语言模型。H1 来自已通过 Manifest、内容哈希和权利门的版本化开发内容包，只允许重述任务或指出检查维度：

- 不返回正确选项；
- 不返回答案句坐标；
- 不返回全文翻译或完整解析；
- 每个当前材料版本最多发放一次 H1；
- 微型表达不复用阅读 H1 接口。

因此 `approved_content_fixture` 是确定性内容适配器，不应被描述为 AI 个性化反馈。模型预算、超时、降级和引用校验留给 H2 或微型表达反馈阶段。

## 3. 不可变证据

`AiIntervention` 与数据库迁移 `0004_intervention_content` 新增：

- `reason_code`；
- `delivered_content`；
- `content_hash`。

领域事件只记录内容哈希和版本元数据，学习者任务读模型在提示实际发放后返回当前材料的介入内容。未领取提示时，工作区响应仍不会暴露内容包中的 H1–H4、正确答案或完整解析。

完成门新增 `learner_revision_after_intervention`：任何已发放介入在产生后续输出后，都必须有指向介入、V1 和 V2 的 `RevisionEvent`。部分失败时，前端可以从已保存 V2 继续补齐引用，不重复生成 V3。

## 4. 桌面工作区

学习端把原来的“一次提交并推进”拆成：

1. 保存 V1，但不结束任务；
2. 选择保持 H0 完成，或领取 H1；
3. H1 后重新开放选择和解释编辑区；
4. 保存用户 V2 并建立引用；
5. 并列回看 V1、H1 和 V2，再确认完成。

界面始终显示最高帮助等级、输出独立性和引用状态。H1 使用描边按钮，不把请求帮助设计成默认主路径。

## 5. 验证覆盖

自动化验证：

- V1 之前拒绝 H1；
- H1 内容来自当前权利合格内容版本；
- 同一幂等键不会重复发放；
- 介入内容及哈希可回放；
- H1 后拒绝原样保存；
- V2 之前以及 V2 未建立修订引用时拒绝完成；
- H1 后 V2 归因为 `hinted_low`；
- 浏览器完成 V1 → H1 → V2 → 引用路径并通过 axe 扫描。

## 6. 明确未完成

- 根据错误类型决定是否需要 H1，而不是完全由用户主动领取；
- H2 局部结构、逻辑或词义范围提示；
- H3 候选解释比较和 H4 最小示范；
- 微型表达单项最高优先级反馈；
- 模型适配器、预算、超时、拒答和确定性降级；
- 自动判断 V2 是否改善；
- 平行新题和延迟无提示验证；
- 真人考生可用性与教师提示审查。

下一阶段应先用当前 H1 路径开展小规模任务走查，验证用户是否理解“保存 V1”“领取 H1”“亲自写 V2”三个状态，再进入更强提示或模型调用。
