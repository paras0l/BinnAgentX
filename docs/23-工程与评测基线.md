# 工程与评测基线

> 状态：Sprint 0 工程规范 v0.1
> 日期：2026-07-15
> 本文定义开始业务代码前必须建立的工程骨架、验证命令和证据边界；截至 2026-07-16，基础命令、锁文件和全量质量门已经通过，结果见[工程骨架验收记录](26-工程骨架验收记录.md)。

## 1. 运行时与依赖基线

| 项目 | 基线 | 规则 |
|---|---|---|
| Node.js | 24 LTS | `.nvmrc`/工具版本文件与 CI 一致 |
| 包管理 | pnpm | 必须提交锁文件；CI 使用 frozen lockfile |
| Python | 3.13 | 不在依赖未验证前直接追最新解释器 |
| Python 管理 | uv | 提交 `uv.lock`，使用 workspace |
| 数据库 | PostgreSQL 18；托管环境不支持时可用 17 | 开发、CI 与生产目标主版本必须记录差异 |
| 前端 | Next.js + React | 用户端与控制舱独立应用 |
| 后端 | FastAPI + Pydantic + SQLAlchemy + Alembic | 领域包不依赖具体 Web 路由 |
| 测试 | pytest、Vitest、Testing Library、Playwright | 机器契约和固定场景是共同事实源 |
| 可观测 | OpenTelemetry | 日志、指标、追踪遵守禁记字段 |

版本升级必须由依赖更新 PR/提交完成，不能使用无锁定的 `latest` 进入主分支。

## 3. 开工时必须存在的基础文件

- 根 `package.json`、`pnpm-workspace.yaml`、锁文件；
- `pyproject.toml`、`uv.lock`；
- Node/Python 版本文件；
- `.editorconfig`；
- `.env.example`，只含变量名和安全说明；
- 本地容器编排文件；
- 数据库迁移入口；
- lint、typecheck、unit、contract、e2e 命令；
- CI 工作流；
- ADR 模板；
- 安全报告与依赖更新策略；
- 开发数据清理和重新初始化命令。

禁止把真实 API Key、OAuth token、真实学习记录或第三方内容放进示例环境文件和 fixture。

## 4. 环境变量契约

变量至少分组为：

- 应用环境与公开 URL；
- 数据库与对象存储；
- 用户端身份适配器；
- 控制舱身份与 MFA 策略；
- 模型网关适配器、预算和超时；
- 日志、指标和追踪；
- 内容包路径与权利门；
- 功能开关；
- 数据留存和删除作业。

每个变量必须标记：是否秘密、适用环境、默认值、失败行为和是否允许出现在浏览器构建中。控制 API、模型凭据和数据库凭据不得进入任何前端公开变量。

## 5. CI 质量门

每个提交至少执行：

1. 格式化与静态检查；
2. TypeScript/Python 类型检查；
3. JSON Schema、OpenAPI 和示例契约验证；
4. 单元与属性测试；
5. 数据库迁移前进/回退测试；
6. 用户端与控制舱 API client 漂移检测；
7. 内容权利、版本、哈希和证据跨度校验；
8. 用户端技术字段与控制能力泄漏扫描；
9. 日志禁记字段测试；
10. 桌面主路径 E2E 与无障碍冒烟；
11. 依赖漏洞和秘密扫描；
12. 固定金标/合成场景回归。

没有真人证据的情况下，CI 只能证明实现满足契约，不能将绿色检查解释为教学有效。

## 6. 测试金字塔

| 层级 | 重点 |
|---|---|
| 纯领域单元 | 状态转换、证据等级、权利门、预算、拒判、阶段保护 |
| 属性测试 | 幂等、不可变、版本单调、无权利不发布、AI 不直接写稳定状态 |
| 契约测试 | JSON Schema、OpenAPI、事件来源端、前后端 client |
| 集成测试 | PostgreSQL 事务、Outbox、Worker、对象存储和删除作业 |
| 模型节点回归 | 结构化输出、引用跨度、越界、拒判、供应商切换 |
| 桌面 E2E | 首次体验、标记、微型表达、保存恢复、错误降级 |
| 控制舱 E2E | RBAC、脱敏、复核、审计和版本冲突 |
| 故障注入 | 超时、重复消息、进程退出、乱序、预算耗尽和供应商失败 |

## 7. 首批固定场景

### 正常场景

1. 无最近成绩完成首次体验；
2. 英一用户完成标准校准和匹配；
3. 英二用户完成标准校准和匹配；
4. 用户不使用提示完成解释和微型表达；
5. 用户使用 H2 后重新输出并保存 V2。

### 边界场景

6. 用户报告材料见过并获得替换材料；
7. 画像证据冲突，系统使用保守匹配；
8. 用户连续重复提交同一幂等命令；
9. 客户端事件乱序和重复到达；
10. 中文输入法组合态遇到自动保存和快捷键；
11. 编辑器存在重叠标记、撤销重做和旧 Schema 文档；
12. 用户在 200% 缩放下只使用键盘完成任务。

### 故障场景

13. 模型超时；
14. 模型返回非法 JSON；
15. 模型反馈没有有效证据引用；
16. 模型预算耗尽；
17. Worker 在副作用前后分别退出；
18. 内容在会话之间被撤回；
19. 控制舱命令缺少理由、权限或预期版本；
20. 用户发起导出和删除后第三方回调延迟。

### 恶意与安全场景

21. 材料正文包含提示注入指令；
22. 用户正文包含脚本、SQL 和日志伪造内容；
23. 用户端尝试调用控制 API；
24. 控制舱尝试直接修改数据库；
25. 事件属性尝试写入作文正文、自由文本或设备指纹。

每个场景必须记录前置状态、输入、预期领域事实、预期事件、公开错误、审计结果和禁止副作用。

## 8. 核心不变量

- 原始作答和作品版本不可覆盖；
- AI 输出不能直接成为用户能力证据；
- 稳定掌握必须引用独立、未见或延迟证据；
- 无合格权利状态的内容不可呈现；
- 同一幂等键只产生一次业务事实；
- 重试不得重复远程扣费；
- checkpoint 不是长期学习者模型；
- 用户端不得获得控制 API 能力；
- 控制舱写操作必须包含操作者、理由和版本；
- 模型、规则、提示、量表、内容和工作流都引用不可变版本；
- 日志不得包含完整正文、完整提示、凭据或模型思维过程；
- IRT/CAT、自动总分和预计提分在功能开关上默认禁用。

## 9. 模型评测基线

模型节点分别评测，不能只看端到端“感觉正确”：

- Schema 合规率；
- 正确引用跨度率；
- 无依据推断率；
- 越权建议率；
- 该拒判时拒判、可回答时不过度拒判；
- 不同供应商/版本的差异；
- 延迟、token、成本和失败分布；
- 对不同英语基础合成切片的明显偏差；
- 提示是否泄露完整答案；
- 反馈后是否仍要求用户亲自输出。

合成回归用于防退化，不得命名为教师金标或学习效果评测。

## 10. 日志与隐私基线

允许记录：

- ID、版本、状态、耗时分桶、token、成本和公开错误码；
- 结构化拒判原因；
- 脱敏后的输入输出摘要；
- 状态差异和审计引用。

禁止普通日志记录：

- 作文、翻译、解释和完整原文；
- 完整提示和原始模型响应；
- OAuth token、API Key、Cookie、数据库连接串；
- 最近真实成绩、目标院校和自由文本困难；
- 第三方导入原文件；
- 模型思维过程和内部安全规则。

## 11. Definition of Done

一个纵向切片只有同时满足以下条件才算完成：

- 行为符合产品与教学不变量；
- 数据、事件、API 和内容契约全部通过；
- 正常、边界、故障和恶意场景均有自动化覆盖；
- 桌面浏览器、键盘、缩放、中文输入法和恢复路径通过；
- 日志、追踪和审计可解释每个业务事实；
- 权利门、预算门和拒判门实际阻断；
- 数据导出删除演练通过；
- ADR、运行手册和已知限制同步；
- 没有用合成结果声称真人可用性或学习效果。

## 12. 正式 MVP 前追加门槛

- 真实目标用户可用性研究；
- 教师量表与金标一致性；
- 内容真实难度标定；
- 未见材料、延迟保持和限时输出协议；
- 正式内容权利、隐私、协议和备案评估；
- 生产认证、MFA、备份、恢复、告警和应急响应；
- 8–12 周试点与审慎效果表述。
