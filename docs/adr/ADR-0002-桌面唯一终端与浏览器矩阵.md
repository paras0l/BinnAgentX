# ADR-0002：桌面唯一终端与浏览器矩阵

> 状态：accepted
> 日期：2026-07-15
> 决策所有者：项目负责人
> 适用范围：用户学习端与开发者控制舱

## 背景

产品的核心动作是长文阅读、精确语义标记、键盘写作、版本对比和并排证据查看。移动布局会改变这些动作，也会引入触控、软键盘、相机、跨端同步和额外无障碍范围。

## 决策

产品只支持电脑浏览器桌面 Web，不开发或适配手机、平板、原生桌面客户端及其他终端。首轮工程矩阵为：Chrome、Edge、Safari 当前及上一稳定大版本，Firefox 当前版兼容冒烟；最低视口假设为 `1280×720`，推荐 `1440×900` 及以上，并验证 200% 浏览器缩放、键盘全路径和中文输入法组合态。

小于最低视口时给出清楚的环境提示，不自动降级成手机式核心流程。纸笔只用于未来考场迁移验证，不是软件入口。

## 替代方案

- 响应式覆盖手机和平板：扩大交互和测试面，且不符合当前产品决定；
- 原生桌面应用：首版没有需要原生能力的证据；
- 只支持 Chromium：会放弃 Safari 兼容风险的早期发现。

## 后果

- 交互可以围绕可调双栏、精确选区和键盘效率优化；
- 禁止加入相机、语音、手写、截图/OCR 和跨终端接续；
- 浏览器矩阵是 Spike 基线，正式支持版本仍需后续真实使用数据；
- 200% 缩放不得以隐藏核心内容换取布局完整。

## 验证证据

- 主路径桌面 E2E 浏览器矩阵；
- 200% 缩放下的键盘完成测试；
- 中文输入法、选区、重叠标记、撤销重做和自动保存测试；
- 5,000 词编辑器性能实验。

## 回滚方式

若某一浏览器在 Spike 中存在无法合理规避的底层缺陷，可暂时降为“已知限制”，但必须记录复现证据。增加新终端属于重新立项，不能以本 ADR 的普通修订完成。
