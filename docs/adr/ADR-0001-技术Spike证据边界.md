# ADR-0001：技术 Spike 证据边界

> 状态：accepted-for-spike
> 日期：2026-07-15
> 决策所有者：项目负责人
> 适用范围：首轮两周技术 Spike

## 背景

项目当前无法开展考生访谈、教师共创或真人学习效果实验，但仍需验证桌面长文交互、状态恢复、模型边界和数据可追溯性。若不预先限定证据含义，合成测试结果容易被误写成产品需求或教学效果已经成立。

## 决策

首轮只使用合成身份、项目自写内容和固定场景，回答工程可行性问题。技术报告只能陈述契约、可靠性、延迟、成本、安全和恢复结果。用户意愿、持续使用、内容真实难度、教师一致性、学习迁移、提分、IRT/CAT 效度和付费意愿均不在证据范围内。

任何用户可见进步表述必须区分“本次完成”“需要新材料验证”和“延迟稳定证据”。技术原型不得展示能力提升百分比、预计提分或已掌握结论。

## 替代方案

- 直接按完整 MVP 开发：缺少需求、内容和评分证据，返工面过大；
- 完全等待真人资源：无法提前消除工作流、编辑器和权限等工程风险；
- 用模型互评代替真人：只能做回归辅助，不能证明学习效果。

## 后果

- 可以开始一个可逆、隔离的技术 Spike；
- 正式产品和封闭试点仍被真人、教师、权利和合规门槛阻断；
- 所有分析、界面文案和报告必须携带证据等级；
- 不实现真实登录、支付、真实词汇账号或未授权真题。

## 验证证据

- `docs/19-技术Spike章程与开工门槛.md`；
- `fixtures/content/v1/` 六个合法开发样例；
- `fixtures/scenarios/v1/scenarios.json` 固定工程场景；
- Spike 退出报告对禁止结论的文本扫描与人工复核。

## 回滚方式

若仍无法隔离真实数据、内容权利或高影响结论，停止 Spike 的业务扩展，仅保留无内容的基础设施实验。新的真人证据不能反向改变本 ADR 的历史范围，应通过新的产品阶段 ADR 接续。
