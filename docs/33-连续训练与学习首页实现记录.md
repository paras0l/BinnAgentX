# 连续训练与学习首页实现记录

> 状态：桌面 Web 可连续体验增量 v0.1
>
> 日期：2026-07-16
>
> 证据边界：证明首次体验完成后可以进入新材料的下一次读写闭环，并保留可理解的本机体验记录；不证明长期留存、选材教学有效性或稳定能力提升。

## 1. 本次完成的体验闭环

原有纵向切片在完成后只能返回首次背景，下一任务也只是不可执行的占位事实。本增量把主路径扩展为：

`首次背景 → 校准 A/B → 校准摘要 → 匹配阅读 → 微型表达 → 难度反馈 → 完成页 → 学习首页 → 新材料 practice 运行 → 匹配阅读 → 微型表达 → 收尾`

首次运行继续保留两段校准。后续运行不再重复 60 秒背景和两段校准，而是直接从匹配阅读开始；阅读工作区、语义标记、V1/H1/V2、表达单项反馈、草稿恢复、暂停和幂等命令仍沿用已有实现。

### 难度评价、当前适配水平与下一篇材料

阅读实验室沿用收尾页原有的“偏简单 / 刚刚好 / 偏困难”入口，不另建评价流程。评价和运行状态在同一事务中保存，并排入 `learner_level_assessments`；常驻 Worker 随后调用水平评测 Agent，汇总该账号已有的任务完成、提示依赖、作答独立性、修订、标注、语法挑战以及历次主观难度反馈。

评测结果称为“当前适配水平”，不是考试分数或正式 CEFR 结论。结果包含综合档位 `foundation / developing / independent / advanced`，阅读理解、词汇、语法、书面表达四个维度，以及 `low / medium / high` 置信度、证据数量和可审计原因码。

学习画像页展示综合档位和四维结果。下一次个性化阅读生成会读取最新已完成评测，同时调节词汇负荷、句法负荷、篇章关系和支架强度；低置信度时只允许增加一个主要挑战维度。标准题库的题目档位和后续材料匹配也优先使用当前适配水平，并在没有新评测时兼容原画像快照。用户跳过主观评价时仍可根据学习行为更新，但不会伪造难度评价信号。

该实现复用原有难度反馈、Worker、画像和个性化材料生成链路，不存在第二套阅读器、反馈状态机或材料工作区。

## 2. 运行类型与连续创建

`VerticalSliceRun` 新增两个显式字段：

- `run_kind=first_experience | practice`；
- `predecessor_run_id`。

首次运行必须从 `calibration_a` 和校准阅读创建；`practice` 必须引用一个已完成前序运行，从 `matched_reading` 和一条已登记的匹配决策创建。领域层会拒绝混合状态，完成门只对首次运行要求两段校准。

学习 API 新增：

`POST /learner/v1/runs/{workflow_run_id}/continue`

该接口要求前序运行已完成且版本一致。数据库对 `predecessor_run_id` 建立唯一约束；相同前序即使使用新的幂等键再次请求，也只返回已有后续运行，不会并行产生两条体验链。

## 3. 新材料匹配

连续创建会复制前序学习者快照中的显式背景，更新证据数量并生成新的不可变快照。匹配器先排除前一运行使用过的正式阅读材料；若合格池只剩一项则使用该项，池耗尽时才重新开放完整候选集。

匹配决策使用 `continuous_practice_match_v1`，并保存 `recent_material_excluded` 原因码。它仍是确定性保守规则，不是能力估计或远程模型决策；当前开发内容包只有两组正式阅读与配对表达，因此只能验证轮换机制，不能证明长期内容供给已经完成。

## 4. 首次校准摘要

首次运行完成两段校准、进入匹配阅读时，学习端先呈现一次校准摘要：

- 本次较稳的部分；
- 接下来优先观察的动作；
- 为什么采用当前匹配负荷；
- 证据仍少、不是最终等级的边界说明。

摘要不展示内部参数、预计分数或精确能力等级。用户确认后进入已分配的匹配阅读；确认状态只保存在当前电脑，存储不可用时安全地再次显示摘要。

## 5. 学习首页与完成页

完成页现在展示本次任务数、帮助使用情况、难度反馈对应的下一步建议，并允许直接开始新材料训练或返回学习首页。

学习首页提供：

- 已保存的英一/英二方向、单次时长和反馈方式；
- 近 7 天本机完成分钟与用户计划的对照；
- 完整闭环次数和全程 H0 次数；
- 最近 5 次的运行类型、难度反馈、任务数和帮助使用情况；
- 更新训练设置并重新校准的入口。

这些数据使用版本化 `localStorage` 投影，最多保留 12 次。界面明确写明“仅保存在此电脑”，且不把时长、次数或 H0 数量解释为能力提升。服务端领域事实仍是任务与运行是否完成的权威来源。

## 6. 自动化证据

新增或扩展验证包括：

- `practice` 从匹配阅读开始且完成门不再要求校准；
- 连续接口拒绝未完成或版本不一致的前序运行；
- 首次匹配到材料 01 后，下一次自动选择材料 02；
- 同一前序运行不能生成第二个后续运行；
- 本机完成记录重复写入保持幂等；
- 返回用户从学习首页进入 3 步 practice 工作区；
- 新增页面通过 Chromium 主路径、Console/网络错误门禁和 axe 扫描；
- Python/TypeScript 类型检查、单元、契约、集成、Lint 和生产构建通过。

## 7. 明确未完成

- 账号、登录和服务端用户级训练历史；
- 跨浏览器或跨电脑同步；
- 超过两组正式材料后的长期内容调度与重复间隔；
- 难度反馈驱动的多维升降级策略；
- 延迟任务、无提示迁移任务和阶段切换；
- 用户主动纠正画像、熟题替换和内容问题上报；
- 返回用户的完整偏好编辑、删除数据和隐私控制；
- 真人考生留存、可用性、教师审查与学习效果验证。

因此本增量把项目从“一次性 60 秒入口和单轮技术切片”推进到了可重复运行的完整桌面体验，但仍属于范围受限的工程验证版本，不是生产学习系统。
