# 训练系统与教学规范

> 本文定义后台 Agent 如何建模、选材、编排任务、更新知识状态和切换阶段。

工程实现采用“受约束工作流编排器 + 专用工具”的方向，不把每项能力拆成自由对话的多 Agent。IRT/CAT 只负责前期定位和阶段检查等测量任务，不直接决定日常训练内容。技术边界见[后台 Agent 与 IRT/CAT 技术路线](13-Agent与IRT-CAT技术路线.md)。

## 1. 后台 Agent 的职责

1. 建立并维护学习者模型；
2. 把考试目标分解为三阶段目标；
3. 根据水平、时间和训练量选择内容；
4. 将零散知识组织成完整任务；
5. 从作答和文字作品中生成待验证错误假设；
6. 协调外部词汇复习、语法补强、迁移和延迟复习；
7. 评估阶段内进步与下一阶段准备度；
8. 后期按英一/英二及题型切换训练策略。

系统不是官方阅卷人，不应代替用户完成翻译、作文或真题。

## 2. 教学原则

结构化 AI 教学的有效性依赖主动学习、认知负荷管理、脚手架、准确反馈、自定步调和专家内容约束；自由生成并不足以稳定完成复杂教学。[Scientific Reports 研究](https://www.nature.com/articles/s41598-025-97652-6)

据此采用：

- 训练流程由阶段规则、知识图谱和审核内容约束；
- 用户必须先处理材料或输出文字；
- 反馈后必须有补强和迁移；
- 同一知识在识别、理解、使用和新语境中分别验证；
- 内容匹配局部状态，不用统一“初级/高级”覆盖全部能力；
- 增加训练量时保持路径一致性，不退化为随机刷题。

## 3. 任务编排输入

每次生成训练周期时至少读取：

- 当前阶段和阶段剩余时间；
- 英一/英二、目标分和考试日期；
- 用户设置的时间、题量和任务长度；
- 语言背景和显式偏好；
- 词汇、词组、语法和技能状态；
- 当前错误假设及优先级；
- 外部词汇工具最近学习的具体词表、目标词义、时间和证据等级；
- 到期复习；
- 最近任务的正确率、耗时、提示和文字质量；
- 内容难度、前置知识和权利状态。

缺少关键证据时，应安排评估或验证任务，而不是假装了解用户。

## 4. 内容水平匹配

### 4.1 匹配目标

建议每个训练周期包含：

- 20% 巩固：用户大概率独立完成；
- 60% 主训练：需要努力、少量脚手架即可完成；
- 20% 挑战/诊断：暴露下一层问题。

比例是初始假设，应通过试点校准。基础薄弱或临考用户可调整。

### 4.2 难度因素

- 词汇覆盖率与熟词僻义密度；
- 句子长度、从句层级和省略；
- 篇章抽象度、论证结构和推理跨度；
- 题型复杂度与干扰项强度；
- 输出长度与评分要求；
- 限时压力；
- 用户是否见过材料。

### 4.3 难度调整

- 连续独立高正确且耗时低：增加迁移或难度；
- 正确但高度依赖提示：保持难度，减少脚手架；
- 多次失败且错误集中在前置知识：回退到对应节点；
- 多次失败但前置知识稳定：更换解释或任务形式；
- 用户提高训练量：先增加语境和迁移，再增加同质重复；
- 用户降低时间：保持主线，删除低优先扩展，不把任务切成随机碎片。

## 5. 完整任务模板

### 任务头

- 阶段目标；
- 本次目标；
- 关联知识节点；
- 选择原因；
- 预计时长与强度；
- 完成标准。

### 学习过程

1. 激活前置知识；
2. 处理完整材料；
3. 独立作答或输出；
4. 查看分层反馈；
5. 确认知识缺口或错误假设；
6. 完成外部词汇复习建议、语法或方法补强；
7. 用新句、新段或新题迁移；
8. 生成复习计划。

### 任务尾

- 本次新增证据；
- 更新的知识状态；
- 新建/确认/推翻的错误假设；
- 下次复习时间；
- 对阶段目标的贡献。

## 6. 三阶段编排规范

### 6.1 前期熟悉阶段

训练目标不是提分最大化，而是减少模型不确定性。

编排要求：

- 评估覆盖广度优先于大量重复；
- 选择用户可能会也可能不会的边界材料；
- 同一结论至少用不同形式校准；
- 文字表达必须进入评估，不能只测选择题；
- 明确区分语言背景事实、用户自评和系统推断；
- 形成中期任务阶段的初始路径和训练量建议。

### 6.2 中期任务阶段

训练目标是补足局部知识并建立读写迁移。

一个周周期应包含：

- 主路径任务：围绕 1–3 个高影响知识/技能目标；
- 最近词汇的阅读再遇、翻译/表达迁移和到期语法复习；
- 至少一次文字表达；
- 至少一次新材料迁移；
- 至少一个高优先错误假设验证；
- 周末阶段内进步检查。

不能按“每天随机换主题”的方式追求新鲜感。一个目标可持续数天，直到获得足够证据。

### 6.3 后期考试冲刺阶段

训练目标是将能力组织成题型表现和考场流程。

编排要求：

- 先做题型基线，再按预期失分和边际收益分配训练量；
- 每个题型有方法学习、分项练习、限时练习和真题复盘；
- 真题重复训练必须区分每轮目标；
- 写作模板必须从记忆走到换题应用；
- 每周保留整套或组合限时训练；
- 临考不无限新增知识，优先稳定高频错误和时间策略。

## 7. 外部词汇积累与本系统验证规范

本模块不是另一个闪卡系统，而是外部高效记忆之后的“语境巩固与主动使用层”。系统需要记住用户最近学过哪些词，并在完整任务中让这些词反复被读懂、译出或恰当使用。

### 状态维度

- 目标考义识别；
- 主动回忆；
- 拼写/词形；
- 固定搭配；
- 真题同义替换；
- 翻译使用；
- 写作使用；
- 延迟保持。

### 工具分工

- 有道词典、不背单词等外部工具负责词书、记忆调度、日常复习和基础统计；
- 本系统负责决定词汇学习如何服务当前阶段，接收用户授权的数据，并在考研语境中验证掌握；
- 外部工具给出的“已学”“已复习”或熟悉度不能直接覆盖本系统知识状态；
- 本系统发现弱词、熟词僻义或词组问题时，生成可复制、可导出的外部复习清单，不再自行提供卡片；
- 用户不提交外部记录时，系统仍从阅读、翻译和写作表现更新词汇状态。

### 最近词汇记忆

每个最近词条至少包含：

- 规范词形与词形变体；
- 用户最近学习的目标词义，未知时标记待确认；
- 来源工具与原始记录；
- 首次接收和最近复习时间；
- 适用目标：阅读识别、翻译处理、主动表达；
- 已出现过的主题、句法和材料；
- 当前迁移阶段与下一次验证窗口。

系统不要求把所有导入词同时塞进下一篇文章，而应按新近程度、考研价值、用户水平、材料自然度和反复错误优先级选择。

### 迁移梯度

建议采用以下梯度，具体间隔由表现调整：

1. **语境确认**：在短句或原主题附近确认目标词义；
2. **篇章再遇**：在符合水平的完整阅读材料中再次读到；
3. **跨语境理解**：更换主题、搭配或句法后仍能理解；
4. **翻译处理**：对需要精确处理的词义或词组完成英译汉；
5. **受控表达**：对适合主动使用的词组完成改写或限定表达；
6. **自由表达**：在相关作文任务中自然使用，不提示具体词；
7. **延迟迁移**：隔一段时间在未见材料中再次成功。

阅读识别型词汇可在第 3 或第 4 层形成稳定，不必强制进入作文；只有常用、语域合适且能提升表达的词组进入主动表达路径。

### 阅读材料匹配

- 先满足用户水平、阶段目标和材料完整性，再考虑目标词覆盖；
- 一篇材料只承载合理数量的目标词，不能为了覆盖率造成词汇堆砌；
- 优先检索自然包含目标词的合法材料，必要时使用经过审核的适配材料；
- 同一个词的再遇应尽量改变主题、搭配或句法，避免只记住原句；
- 目标词过多时分散到多次完整任务，不拆成随机词句流；
- 用户理解文章但目标词判断错误时，单独更新该词状态，不错误降低全部阅读能力。

### 输出使用规则

- 作文训练优先复用高频词组、逻辑表达、功能句和可迁移句式；
- 生僻词、强语域词和用户尚未掌握搭配的词不进入强制作文目标；
- “用过”不等于“会用”，还需检查词义、搭配、语法、语域和是否自然；
- 有提示使用与无提示主动使用分别记录；
- 翻译正确但作文不自然时，保留翻译处理状态，不提升主动表达状态。

### 外部证据等级

1. 官方合作 API 返回的逐词行为；
2. 用户主动导入的结构化导出文件；
3. 用户分享、截图或批量粘贴的最近具体词表，经解析后由用户确认；
4. 用户分享的汇总学习报告；
5. 用户手动填写的时长、词量和完成情况；
6. 仅有其他 App 的前台使用时长。

前三类可以建立最近词汇迁移队列，但仍不能证明掌握；后三类只能证明不同强度的学习投入。词汇掌握必须通过目标考义、词组、阅读语境、翻译或写作输出验证。

## 8. 语法地图规范

### 节点层级

1. 词法与形态；
2. 基本句型与主干；
3. 名词性/定语/状语从句；
4. 非谓语；
5. 并列、省略、倒装、强调、比较；
6. 指代、连接和篇章逻辑；
7. 综合长难句。

### 点亮规则

| 状态 | 最低证据 |
|---|---|
| 接触 | 完成一次讲解或示例 |
| 识别 | 在句中正确指出结构 |
| 拆解 | 独立找出主干和关系 |
| 改写 | 保持原意完成结构变换 |
| 迁移 | 在未见句子或表达中正确使用 |
| 稳定 | 跨时间、跨材料多次迁移成功 |

不能因看完内容自动点亮高阶状态。

## 9. 错误假设规范

### 创建条件

- 同类错误重复；
- 单次错误但影响范围大；
- 阅读、翻译、写作出现同源问题；
- 用户自述与实际表现明显冲突。

### 假设优先级

`影响题型数量 × 出现频率 × 可修复性 × 临考权重 × 置信度`

### 验证设计

- 使用不同题材和表面形式；
- 同时安排能支持和推翻假设的任务；
- 控制前置知识，避免把词汇问题误判为语法问题；
- 记录独立程度和耗时；
- 改善后做延迟和跨题型验证。

禁止使用“粗心”“基础差”“语感不好”作为不可操作的最终结论。

## 10. 阅读与表达联动

每个主阅读任务至少选择一种输出：

- 用中文复述论证结构；
- 翻译目标长难句；
- 用目标词组改写句子；
- 用英文写 1–3 句摘要；
- 复用篇章结构完成段落提纲。

输出难度必须匹配用户水平。基础薄弱者从结构化句子开始，高水平者进入摘要、论证和限时写作。

## 11. 后期题型规范

### 完形

区分词义、搭配、语法和篇章连贯；训练全篇意识与速度，不只逐空讲词。

### 阅读 A

按细节、推断、主旨、词义、例证、态度等建立证据链；记录定位、理解和选项辨析分别在哪里失败。

### 阅读 B

按当年英一/英二题型配置训练段落功能、衔接、排序和匹配。

### 翻译

- 英一：复杂语篇中的句子理解与准确表达；
- 英二：段落整体理解、完整性和连贯中文；
- 保存初稿、修改稿和限时稿；
- 参考译文最后出现。

### 小作文

训练任务识别、格式、功能句、语域和信息完整；模板必须按具体任务替换。

### 大作文

训练审题、提纲、段落功能、论证、语言和时间；英一/英二分值与要求分别配置。

## 12. 作文模板学习规范

模板不是整篇直接套用，而是分为：

- 题型功能框架；
- 段落骨架；
- 可复用句型；
- 主题变量；
- 风险表达与禁用搭配。

学习序列：理解 → 分块记忆 → 遮挡回忆 → 默写 → 纠错 → 变量替换 → 换题迁移 → 限时整篇。

只有完成换题迁移的模板才能标为“会用”。

## 13. 反馈规范

- 每次只优先处理最影响当前目标的 1–3 个问题；
- 引用用户真实作答或文字；
- 区分正确性错误、方法问题和表达升级；
- 先让用户修改，再展示完整参考；
- 自动评分使用区间和置信度；
- 用户和教师可纠正反馈，纠正结果进入评测集。

## 14. 质量检查

每个任务发布前检查：

- 是否属于当前阶段；
- 是否匹配局部水平；
- 是否满足用户设置的训练量；
- 是否有完整输入—处理—输出—反馈—迁移；
- 是否能更新知识状态或错误假设；
- 是否使用审核内容；
- 是否把真题记忆误当迁移；
- 是否包含必要文字表达；
- 冲刺阶段是否使用正确英一/英二配置；
- 训练量增加后是否仍然连贯。
