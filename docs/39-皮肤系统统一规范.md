# 39｜皮肤系统统一规范

> 状态：实现约束（长期有效）
> 适用范围：学习端所有现有页面、未来新增页面、公共组件与主题资产
> 关联文档：`38-皮肤系统优化.md` 负责设计方向与体验目标；本文负责可执行的工程契约。两者冲突时，皮肤系统实现以本文为准。

## 1. 目标

皮肤不是单页配色开关，而是一套可继承、可降级、可验证的表现层协议。新增页面和组件在不认识具体皮肤名称的前提下，也必须自动获得正确的色板、文字对比度、边界、图标、焦点态和状态反馈。

皮肤可以改变视觉与布局，但不得改变学习流程、业务语义、数据结构、权限判断和可访问名称。

## 2. 权威来源

实现由以下文件共同组成：

- `theme/contracts.ts`：等级、能力、资产槽的类型契约。
- `theme/registry.ts`：唯一皮肤注册表、等级说明和资产清单。
- `theme/runtime.ts`：把注册项投射到根节点属性和 CSS 变量。
- `theme/tokens.css`：所有皮肤都必须提供的语义令牌与安全默认值。
- `theme/anchors.css`：跨页面、跨组件的稳定换肤锚点。
- `theme/compatibility.css`：旧组件兼容层，只允许逐步缩减。
- `theme/themes/<theme-id>.css`：单个皮肤的增量实现。
- `public/themes/<theme-id>/README.md`：有复杂资产时必须提供的来源、用途和切图说明。

禁止在页面组件里通过 `skin === "某皮肤"` 分支决定纯视觉样式。运行时分支只允许处理确实无法由 CSS 表达的可访问内容或资源加载行为，并且优先抽象为能力。

## 3. 皮肤等级

等级表示表现层覆盖范围，严格向下继承。

| 等级 | 标识 | 必须包含 | 可增加 |
| --- | --- | --- | --- |
| 基础 | `standard` | 语义色板、表面、边界、控件、焦点态 | 简单明暗变化 |
| 史诗 | `epic` | 基础全部能力 | 环境材质、标题语言、图标底座 |
| 传说 | `legendary` | 史诗全部能力 | 专属布局、角色安全区、插画、装饰、彩蛋、增强转场 |
| 典藏 | `collector` | 传说全部能力 | 组件级替换、多场景陪伴、皮肤图鉴、粒子标题、主题指针轨迹、完整素材叙事 |

规则：

1. 高等级不等于更多饱和色，而是更完整、一致且有辨识度的覆盖。
2. 高等级必须包含低等级的可读性、状态完整性和无障碍能力，不能用装饰抵消基础质量。
3. 注册表中的等级说明必须能在偏好设置里展开查看。
4. 新增等级必须同时更新类型、等级表、排序、说明、测试和本文档，不允许只添加标签。

## 4. 能力模型

当前能力及职责：

| 能力 | 作用 |
| --- | --- |
| `palette` | 核心语义色板 |
| `surfaces` | 页面、卡片、浮层、控件表面 |
| `typography` | 标题和强调文字的字体语言 |
| `icon-frames` | 图标颜色、底座、选中与悬停外观 |
| `ambient` | 页面级低干扰环境材质 |
| `decorations` | 非必要装饰槽和彩蛋 |
| `hero-art` | 首页主插画 |
| `layout-variant` | 专属布局和角色安全区 |
| `component-skins` | 按钮、标签、空状态、计数器等完整组件外观 |
| `companion-scenes` | 角色在多个页面场景中的一致陪伴 |
| `artbook` | 可访问的皮肤图鉴入口与素材展示 |
| `particle-headings` | 使用 Canvas 将低频 `<h1>` 重组为可交互文字粒子 |
| `pointer-trail` | 页面级主题化滚动轨迹；在鼠标或触摸当前位置随滚轮/页面滚动触发，不得拦截输入 |
| `enhanced-motion` | 可降级的增强动效 |

组件不得查询等级决定行为，应查询根节点的 `data-theme-features` 能力。等级用于商品化展示、排序和能力下限；能力用于实际渲染。

## 5. 运行时协议

`ThemeProvider` 必须把规范化后的配置写入 `document.documentElement`：

- `data-theme="<theme-id>"`
- `data-theme-tier="<tier>"`
- `data-theme-features="<space-separated capabilities>"`
- `data-density="comfortable|compact"`
- `data-motion="full|reduced"`
- `color-scheme`

每次切换皮肤必须先清理旧资产 CSS 变量，再写入新皮肤变量，避免猫爪、背景图或局部切图残留。未知或已下架的皮肤必须回退到 `paper`，不得让页面进入半换肤状态。

首屏内联启动逻辑与客户端运行时必须使用同一注册表数据语义，避免刷新时先闪现另一套皮肤。

## 6. 语义令牌

新增组件必须优先使用语义令牌，不得复制某个皮肤的十六进制颜色。

最低覆盖范围：

- 页面：`--theme-canvas`、`--theme-canvas-muted`
- 表面：`--theme-surface`、`--theme-surface-raised`、`--theme-surface-soft`
- 文字：`--theme-ink`、`--theme-ink-muted`
- 强调：`--theme-accent`、`--theme-accent-strong`、`--theme-accent-soft`
- 边界与焦点：`--theme-line`、`--theme-line-strong`、`--theme-focus`
- 控件与浮层：`--theme-control-*`、`--theme-popover-background`
- 状态：`--theme-danger-*`、`--theme-success-*`、`--theme-warning-*`、`--theme-info-*`
- 图标：`--theme-icon-background`、`--theme-icon-ink`
- 典藏粒子：`--theme-particle-heading-color`、`--theme-pointer-trail-color`、`--theme-pointer-trail-highlight`
- 首页重点区：`--theme-hero-*`
- 阴影、圆角、字体、滚动条和动效令牌。

正文与背景的目标对比度不低于 4.5:1；大字号文字不低于 3:1；边界不是唯一状态信号。主题色不能直接用作小字号正文，除非实测达标。

## 7. 组件接入规范

所有新增组件至少完成以下事项：

1. 外层使用语义令牌；不因皮肤切换丢失文字、边界或焦点态。
2. 可复用语义位置添加稳定的 `data-ui-anchor`，名称描述功能而不是皮肤，例如 `empty-state`、`toolbar`、`status-area`。
3. 纯装饰资源使用 `data-theme-slot`，并设置 `aria-hidden="true"`、`pointer-events: none`。
4. 图标颜色和底座必须跟随令牌；选中、悬停、禁用和强制色模式都可辨认。
5. 空白不等于必须加边界。只有存在分组、层级或交互范围时才增加边界；不得在品牌标题前制造无语义的垂直分界线。
6. 标题艺术变换只用于低频大标题，不用于正文、表单标签、数据或高频变化文本。

锚点属于跨皮肤公共契约。修改或删除锚点前必须检索所有主题 CSS 和测试。

## 8. 专属布局与角色安全区

传说和典藏皮肤可声明 `layout-variant`，但必须满足：

- 先标注插画的主体、脸部、标志和文案安全区，再安排真实内容。
- 桌面、窄桌面、平板、手机分别定义安全布局，不能只改变 `background-position`。
- 文字卡不得遮住角色脸部和关键姿态；裁切不得切掉头部、眼睛或主题标志。
- 窄屏优先改成“插画区在上、内容区在下”，而不是让内容继续覆盖插画。
- 动态文案变长、出现缓存任务、校准提醒和错误状态时仍需验证。
- 如果素材无法适配某个比例，允许隐藏非关键装饰，但必须保留功能和可读性。

## 9. 资产槽与文件规范

当前资产槽：

- `ambient`
- `hero`
- `sidebar-ornament`
- `hero-accent`
- `history-easter-egg`
- `companion-atlas`
- `ui-library`
- `concept-sheet`
- `usage-guide`
- `empty-state-illustration`
- `profile-companion`
- `preferences-companion`
- `workspace-companion`
- `milestone-companion`
- `card-corner-decor`

每个注册资产必须声明 `src`、`width`、`height`、`mimeType`；有特殊裁切时声明 `fit` 与 `position`。只有真实首屏关键图才允许 `preload: true`。

文件要求：

1. 路径使用 `/themes/<theme-id>/`，运行时文件名使用小写 ASCII 与连字符。
2. 原始设计稿保留在文档素材目录，Web 目录使用压缩副本，不直接覆盖源文件。
3. 复杂图集必须提供 README，记录来源、用途、切图位置和更新联动点。
4. 主图优先 WebP/AVIF；需要兼容或保真时可使用压缩 JPEG/PNG。
5. 装饰图不得阻塞功能首屏。图鉴、概念稿和素材表默认不预加载。
6. 所有网络失败场景必须回退到主题色或通用组件，不能显示破图图标。
7. 外部素材必须记录授权和来源；AI 生成资产必须记录输入素材、目的和生成约束。
8. 运行时界面不得从概念大图、角色图集或 UI 总表中缩放裁切角色；它们只用于图鉴和设计追溯。界面角色必须使用完整轮廓的独立透明资产，并按页面语义注册。

## 10. 典藏皮肤附加要求

典藏不是“传说加一个徽章”。至少满足：

- 注册 `component-skins`、`companion-scenes` 和 `artbook`；需要粒子体验时显式注册 `particle-headings` 与 `pointer-trail`。
- 首页拥有经过验证的专属安全布局。
- 角色或主题语言出现在至少三个不同功能场景，且不影响操作。
- 按钮、标签、图标、空状态、数据卡或状态组件中至少四类形成统一组件语言。
- 图鉴入口只在声明 `artbook` 时出现；对话框支持键盘关闭、清晰标题和可访问图片名称。
- 所有原始素材都应有明确用途；未在主界面直接展示的概念稿进入图鉴或资产说明。
- 粒子标题必须保留原始 `<h1>` 的语义内容；Canvas 仅作为 `aria-hidden` 表现层。文本换行使用 Pretext 计算，禁止逐帧触发布局测量。
- 粒子标题必须先按可用宽度建立独立展示区并放大字形，再采样为 ASCII Hero；不得直接在原始小字号上叠加粒子。中日韩与长标题允许通过 Pretext 换行，不能为了统一动效牺牲可读性。
- 复杂的中日韩字形应采用参考实现移动端的 `01` 小点阵密度，并在同一 Canvas、同一坐标提供低透明度字形骨架；拉丁展示标题使用完整 ASCII 字符集。
- 指针轨迹只能在声明能力后加载；触发逻辑与参考实现一致：`mousemove`/`touchmove` 只记录位置，`wheel` 或实际 `scroll` 才在该位置发射粒子，并执行 30ms 节流与页面上下边界判断。不得拦截滚动、点击或上下文菜单。

## 11. 动效与降级

- 动效只用于建立层级、反馈状态或增强陪伴感。
- `data-motion="reduced"` 和 `prefers-reduced-motion` 任一命中时，循环漂浮、视差和大幅转场必须停止。
- 粒子标题和指针轨迹在减少动态、打印及强制色模式下必须恢复原生标题并彻底隐藏 Canvas。
- 禁止用动效表达唯一信息。
- `forced-colors: active` 下隐藏纯装饰图和背景图片，保留系统颜色、结构与焦点。
- 资源未加载、CSS 变量为空和浏览器不支持 `backdrop-filter` 时，文本仍需可读。

## 12. 性能预算

- 每个皮肤仅预加载一张真正影响 LCP 的主视觉。
- 图集优先复用，避免同一角色姿势生成多个大文件。
- 页面切换不得重复插入相同预加载标签。
- 背景纹理需压缩并保持低对比度；避免多个全屏透明层同时做昂贵滤镜。
- 新增依赖前先判断 CSS、现有图标库和已有资产能否完成。
- React 组件读取注册表后应复用派生定义，不在渲染列表中重复做昂贵处理。
- 典藏动效运行时代码必须按能力动态加载；所有可见标题共享一个动画帧循环，并通过可见性观察器暂停离屏标题。

## 13. 新增皮肤流程

1. 明确等级、视觉目标、可访问性风险和是否需要专属布局。
2. 整理源素材，制作运行时副本和资产 README。
3. 在 `contracts.ts` 增加必要的通用能力或资产槽；不得添加只服务一个皮肤的同义能力。
4. 在 `registry.ts` 注册皮肤、等级、能力、预览色和完整资产元数据。
5. 只覆盖与该皮肤不同的 CSS；通用缺陷修复应放在令牌或锚点层。
6. 在偏好设置验证标签、等级说明、选中态、保存与刷新恢复。
7. 运行契约检查、单元测试、构建与浏览器视觉检查。
8. 在桌面、窄屏、手机、减少动态和强制色模式下完成验收。

## 14. 必测矩阵

每个皮肤至少覆盖：

- 首页、学习资产、学习画像、偏好设置、训练工作区。
- 初始空状态、有历史数据、缓存任务、错误提示和禁用状态。
- 皮肤 A → 皮肤 B → 皮肤 A，确认无资产和图标残留。
- 刷新后注册皮肤与偏好设置选中项一致。
- 1280px 以上桌面、约 1024px 窄桌面、760px 以下移动端。
- 键盘导航、焦点可见、弹窗 Escape 关闭、减少动态、强制色模式。

自动验证至少包括：

- 注册表 ID、等级排序和未知值回退。
- 高等级能力继承。
- 根节点属性与资产变量写入、切换清理。
- 关键页面锚点存在。
- 构建与主题契约脚本通过。

## 15. 完成定义

满足以下全部条件才算皮肤完成：

- 功能、语义和数据不因皮肤改变。
- 所有页面使用一致色板、字体、图标与组件语言。
- 没有遗留资产、低对比度文字、无意义分界线和被遮挡角色。
- 专属布局在目标尺寸下验证过，窄屏有明确降级。
- 等级与实际能力相符，等级说明可展开。
- 资产来源、加载策略和切图约束有文档。
- 测试、主题契约检查、构建和浏览器视觉验收全部通过。
