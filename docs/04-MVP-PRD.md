# MVP 产品需求文档（PRD）

> 状态：Draft 0.3，待按双实验室定位重写  
> 日期：2026-07-15  
> 产品形态：固定三阶段的读写迁移 Agent。

> **范围警告：** 本 PRD 形成于产品定位收敛之前，不能直接作为开发基线。当前有效的前台范围只有“语境实验室 + 表达实验室 + 阶段与进步”；知识地图、语法、词汇迁移、错误假设和真题均应作为后台能力或阶段配置重新编排。定位与第一版边界见[双实验室产品定位与竞争力分析](14-双实验室产品定位与竞争力分析.md)。

## 1. MVP 目标

验证系统能否在完整备考周期中：

1. 形成可解释、可纠正的学习者模型；
2. 按多维知识负荷和自定训练量提供连贯的语境任务；
3. 承接外部工具最近学习的词汇，通过阅读吸收、短表达和延迟迁移补足局部知识；
4. 识别并验证反复出现的错误假设；
5. 用同难度、延迟和迁移证据展示阶段内进步；
6. 通过逐级撤除的脚手架提高写作启动、独立成文和新题迁移；
7. 在冲刺阶段按英语一/英语二阅读与写作题型特化。

## 2. 非目标

- 不做全科、择校、调剂和复试；
- 不建立直播课、名师商城或开放社区；
- 不以随机生成大量题目代替教研内容；
- 不提供通用 AI 聊天、默认全文代写或一键润色；
- 不自建单词闪卡、词书和单词间隔重复；
- 不把分级阅读生成或作文自动评分本身当作核心产品；
- 不承诺预测真实分数或保证上岸；
- 不以连续打卡、页面停留时长或内容浏览量衡量价值。

## 3. 北极星指标

**阶段有效进步单元（Stage-Verified Progress Unit, SVPU）**

一次有效进步单元至少包含：

1. 完成一个属于阶段目标的完整任务；
2. 产生独立作答或文字输出；
3. 更新一个知识状态或错误假设；
4. 完成即时补强；
5. 在延迟或新材料验证中表现改善。

仅浏览解析、完成外部背词量或重复记住原题答案不单独计为阶段进步。

## 4. 信息架构

| 一级模块 | 核心内容 |
|---|---|
| 语境实验室 | 近期积累、匹配材料、独立首读、证据处理、讲解、重构与延迟验证 |
| 表达实验室 | 表达 Skill、想法脚手架、独立写作、分层反馈、修订与新题迁移 |
| 阶段与进步 | 当前阶段、训练量、学习者模型摘要、迁移证据和下一阶段条件 |

## 5. P0 功能需求

### 5.1 学习者档案

用户故事：作为考生，我希望系统先理解我的目标、时间和语言背景，再安排训练。

需求：

- 采集英一/英二、考试日期、目标分；
- 采集每日/每周学习时间、可用时段、单次连续学习上限；
- 采集四六级/既往考研、英语学习经历和用户自述基础；
- 采集显式偏好：任务长度、反馈密度、纸屏习惯、计时、题量；
- 支持随时查看、更正和删除；
- 系统推断与用户明确输入分开展示。

验收：

- 5 分钟内可完成基础档案；
- 不能仅凭学硕/专硕自动决定英一/英二；
- 缺少字段时系统能继续评估，但显示不确定性；
- 用户修改时间和目标后，下个训练周期重新编排。

### 5.2 前期多维评估与校准

用户故事：我希望系统用合理的任务了解我，而不是只用一套选择题给我贴标签。

需求：

- 评估词汇考义、词组、核心语法、句法、阅读和文字表达；
- 采用分段评估，单次 20–40 分钟，可在数日内完成；
- 记录答案、耗时、提示、信心和文字版本；
- 输出初始知识地图和待验证错误假设；
- 之后安排 3–5 个校准任务；
- 用户可标记评估异常、熟悉题目或误操作。

验收：

- 每个画像结论能跳回证据；
- 单一题目不直接形成“已掌握/未掌握”；
- 评估材料按英一/英二目标和用户基础选择；
- 评估结束明确说明中期训练范围和建议训练量。

### 5.3 三阶段状态与切换

需求：

- 使用周期固定为熟悉、任务、冲刺三阶段；
- 阶段顺序不可随意打乱；
- 时长由考试日期和证据在可配置范围内调整；
- 每阶段显示目标、必做内容、可选训练量、完成条件；
- 阶段切换由规则和证据触发，用户可申请提前/延后；
- 临近考试的新用户采用压缩路线，但保留评估与基础补强。

验收：

- 用户始终能看到当前阶段；
- 阶段切换说明使用了哪些证据；
- 手动调整会显示风险和影响；
- 历史阶段数据不被覆盖。

### 5.4 训练量定制器

用户故事：我希望根据自己的时间自由决定训练量，而不是受固定每日题量或内容额度限制。

需求：

- 支持按时间、篇数、题数、词数、写作次数设置周训练量；
- 提供标准、加强、冲刺预设，也允许自定义；
- 训练内容供给不设置人为额度上限；
- 计算预计完成压力、复习负荷和表达任务占比；
- 增量任务从当前学习路径扩展，不随机堆题；
- 支持随时增减，下个训练周期生效；
- 系统可建议减量，但用户保留最终选择权。

验收：

- 任务总预计时长与用户设置误差在可接受范围；
- 提高题量不会挤掉全部复习和表达训练；
- 过高负荷有清晰风险说明和替代配置；
- 不因订阅档位限制基础训练量，商业限制如存在必须与训练质量解耦。

### 5.5 完整任务单元

每个任务必须包含：阶段目标、知识节点、选择原因、预计时长、输入材料、主动处理、反馈、补强、迁移、完成标准和复习安排。

任务类型示例：

- 阅读—语法—翻译联动任务；
- 词组—句式—段落表达任务；
- 语法节点掌握任务；
- 错误假设验证任务；
- 真题题型方法任务；
- 作文模板记忆与迁移任务。

验收：

- 每个任务至少关联一个阶段目标和知识节点；
- 随机词、随机句、随机小题不能单独成为主任务；
- 任务结束必须产生状态更新或明确“证据不足”；
- 任务量增加时优先扩展同一目标的不同语境和迁移。

### 5.6 词汇迁移记忆与掌握验证

需求：

- 不自建单词卡、词书或间隔重复调度；
- 用户可选择有道词典、不背单词或其他背词工具，并设置计划词书、每日时间或词量；
- 支持用户授权的数据接口、文件导入、分享内容、学习报告截图和手动确认；不同来源明确标注证据等级；
- 若第三方没有开放接口，不承诺自动读取逐词记录，不通过破解、无障碍服务或抓取私有接口绕过限制；
- 个性化迁移必须尽量取得最近学习的具体词表；只有学习时长或总词量时，仅记录投入，不能生成逐词迁移任务；
- 对导入词汇进行词形归一、目标词义识别、来源和学习时间记录，无法确定词义时让用户确认或等待语境证据；
- 维护最近词汇窗口和迁移队列，优先处理新近、考研高价值、反复失败且适合当前水平的词；
- 为每个词记录考义识别、主动回忆、词形、搭配和语境迁移；
- 区分“完全不认识、熟词僻义、词组不熟、能认不能用”；
- 外部完成量、学习时长和第三方熟悉度只作为投入证据，不直接写成“已掌握”；
- 将目标词自然安排进符合用户水平的完整阅读材料，避免为了覆盖词表堆砌生硬文章；
- 在不同主题、句法和篇章中安排多次再遇，并通过阅读理解判断用户是否真正读懂；
- 对翻译处理型词汇安排英译汉，对主动表达型词组安排改写、造句、段落或作文使用；
- 阅读识别型词汇不强制进入作文，低频、生僻或语域不合适的词不得为了“激活”而硬塞进表达；
- 通过延迟的新材料验证保持和迁移，而不是重复原句；
- 将任务中发现的弱词、熟词僻义和词组整理为“待外部复习清单”，允许用户复制、导出或分享到所选工具；
- 中期任务中要求将部分词组用于翻译或句子表达。

验收：

- 用户不连接第三方工具也能继续完整训练；
- 每条外部记录显示来源、时间、采集方式和可信等级；
- 每个最近词能查看当前目标：阅读识别、翻译处理或主动表达；
- 每个状态变化能追溯到具体句子、篇章或文字输出；
- 一篇材料的目标词密度不能破坏自然度或超出用户可学习区间；
- 外部工具不可用或格式变化时，不影响阅读、语法和表达主流程；
- 同一词在不同语境表现可分别记录；
- 外部复习后有延迟和新语境验证；
- 系统不得把第三方 App 使用时长等同于有效背词时长或掌握度。

### 5.7 语法地图

需求：

- 地图覆盖词法、句法、从句、非谓语、并列/省略、指代与逻辑结构；
- 节点状态：未知、接触、识别、能拆解、能改写、能迁移、稳定；
- 节点“点亮”必须由证据驱动；
- 每个节点关联真题句、解释任务、拆句、翻译和改写；
- 允许多个节点构成前置依赖；
- 错误假设可指向具体语法节点。

验收：

- 仅看讲解不能点亮“能迁移”；
- 用户能查看点亮依据；
- 新材料失败会降低稳定性或触发复查；
- 地图不以一次性全部点亮为营销目标。

### 5.8 阅读与文字表达联动

需求：

- 阅读任务要求定位证据、解释结构和判断选项；
- 从阅读中抽取目标词组/结构进入翻译、改写或摘要；
- 前期：句子主干复述与短句翻译；
- 中期：长难句翻译、句式改写、段落摘要/展开；
- 后期：限时翻译、小作文和大作文；
- 保存所有文字版本，展示具体改进；
- 参考答案在用户完成独立输出后出现。

验收：

- 每周至少有符合用户水平的文字表达任务；
- 反馈区分理解错误、语言错误和表达升级；
- 第二稿与第一稿可比较；
- 同一目标结构至少在新语境使用一次。

### 5.9 反复错误假设

需求：

- 同类错误出现时创建候选假设，而非立即定论；
- 假设关联证据、知识节点、题型、置信度和影响范围；
- 系统生成验证任务，包含支持和反证机会；
- 用户可以确认、质疑或标记情境原因；
- 假设状态：待验证、已确认、已改善、已推翻、需复查；
- 已改善假设仍需延迟和迁移验证。

验收：

- 每个已确认假设至少有多条证据；
- 用户能看到系统为何提出该假设；
- 被推翻假设不继续影响任务；
- 高影响反复错误优先于低影响偶发错误。

### 5.10 后期英一/英二题型特化

| 模块 | 特化训练 |
|---|---|
| 完形 | 篇章连贯、搭配、逻辑和速度 |
| 阅读 A | 证据定位、题型方法、干扰项、时间分配 |
| 阅读 B | 按英一/英二题型集合训练篇章组织 |
| 翻译 | 英一复杂句与英二段落任务分别评分 |
| 小作文 | 功能、格式、语域、常见任务模板 |
| 大作文 | 英一/英二分值和表达要求分别配置 |

真题多轮训练：

1. 第一轮：独立作答和真实基线；
2. 第二轮：证据、语言与方法复盘；
3. 第三轮：限时重做与步骤自动化；
4. 迁移轮：使用未见材料验证，不把记住答案当进步。

作文模板训练：理解结构 → 分块记忆 → 默写 → 替换变量 → 换题迁移 → 限时使用。

验收：

- 英一/英二不能混用翻译与写作评分；
- 每个题型有独立正确率、速度和稳定性；
- 真题重复正确与新材料迁移分开展示；
- 考试技巧必须解释适用条件，不能替代语言基础。

### 5.11 阶段报告

报告必须包含：

- 阶段起点与当前表现；
- 同难度、延迟和新材料对比；
- 新点亮语法节点；
- 稳定词汇/词组及仍不稳定项；
- 反复错误假设状态；
- 文字作品变化；
- 训练量完成情况与负荷；
- 下一阶段准备度和依据。

验收：每个结论可追溯；证据不足时不输出伪精确值；不把完成量等同进步。

## 6. 核心流程

### 首次使用

`确认目标与时间 → 填写语言背景与偏好 → 分段评估 → 查看初始画像 → 完成校准任务 → 进入中期任务阶段`

### 中期一周

`设置训练量 → 外部闪卡积累 → 最近词汇进入迁移队列 → 匹配阅读 → 翻译/适当表达 → 延迟验证 → 周进步报告`

### 冲刺周期

`确认英一/英二 → 题型基线 → 按失分分配训练量 → 真题多轮 → 模板迁移 → 整套模拟 → 风险项补强`

## 7. 关键业务规则

- 阶段顺序固定，时长和训练量可调；
- 内容难度必须依据局部知识状态，不使用统一难度标签代替匹配；
- 独立作答权重高于提示后完成；
- 记住原题答案不等于掌握；
- 外部背词记录和语法训练必须回到句子、阅读或表达验证；
- 训练量没有人为上限，但系统必须显示负荷风险；
- 错误结论以假设形式存在，直到多证据验证；
- 冲刺阶段按实际试卷配置，英一/英二不混用；
- 所有自动评分标明版本、区间和非官方性质。

## 8. 非功能要求

- 长文阅读、词汇记录导入、地图和写作均支持手机与平板；
- 重要信息不能只依赖颜色；
- 支持动态字体与基础屏幕阅读器语义；
- 弱网时不丢失作答与文字版本；
- 用户能导出、纠正和删除学习者模型数据；
- 内容、评分、任务规则和知识地图均需版本化；
- 模型或服务失败时，确定性训练流程仍可继续。

## 9. 发布门槛

- 评估结果与教师判断达到约定一致性；
- 语法地图节点定义和点亮规则完成教研审核；
- 错误假设误判率在试点可接受范围；
- 内容匹配测试覆盖不同基础与英一/英二；
- 真题答案和解析完成双人审核；
- 作文/翻译反馈达到教师一致性门槛；
- 隐私、版权、生成内容标识、注销和删除流程可用；
- 14 天试点显示阶段任务可执行且没有明显碎片化回退。

## 10. 未决策问题

1. 各阶段默认时长和最短时长；
2. 语法地图第一版节点粒度；
3. 训练量的商业边界与模型成本处理；
4. 首批合法真题年份；
5. 写作与翻译人工金标来源；
6. 首发移动端、Web 或平板优先级；
7. 是否首发支持纸质作答 OCR；
8. 各基础层的难度标定方法。
9. 首批支持哪些外部词汇工具及各自允许的数据接入方式。
