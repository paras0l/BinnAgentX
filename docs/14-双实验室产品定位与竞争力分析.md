# 双实验室产品定位与竞争力分析

> 状态：定位草案 v0.4，待用户访谈与人工训练实验验证  
> 日期：2026-07-15  
> 范围：只讨论考研英语的“阅读吸收”与“自主表达”，暂不进入编码。

## 1. 定位结论

产品不再定位为覆盖所有环节的“考研英语全能 Agent”，而是定位为：

> **面向英语一、英语二考生的读写迁移 Agent。它不替用户背单词，也不替用户写作文，而是把用户已经接触过的词汇、词组、句法和写作思路，转化为读得懂、记得住、写得出的能力。**

前台只保留两个核心学习场域：

1. **语境实验室：读懂与吸收**——把用户近期学过但尚未稳定的知识，放入知识负荷匹配、自然且连贯的材料中，使其在阅读、解释、重构和延迟新语境中反复出现。
2. **表达实验室：想清楚并写出来**——把优秀文章中的可复用思路提炼为“表达技能”，用逐级撤除的脚手架帮助用户从想法、提纲、句子、段落走到独立成文，并在新题中验证迁移。

两个实验室不是并列工具，而是一个闭环：

`近期积累 → 语境中读懂 → 提炼可表达成分 → 用户独立写作 → 暴露表达缺口 → 反向安排阅读与吸收 → 延迟迁移`

一句话价值主张：

> **不替你背词，不替你写作文；让你学过的，真正读得懂、写得出。**

## 2. 命名建议

### 2.1 推荐命名

总结构可称为 **“考研英语读写迁移双实验室”**，两个核心模块为：

| 名称 | 面向用户的副标题 | 选择理由 |
|---|---|---|
| 语境实验室 | 把学过的知识放进读得懂的文章里，反复遇见并真正吸收 | “语境”比“阅读”更能体现词义、搭配、句法和篇章中的再遇，也与普通阅读器拉开距离 |
| 表达实验室 | 把读懂的内容转成自己的思路和句子，逐步写出完整作文 | 保留用户原命名；重点是形成表达能力，不是得到一篇被 AI 美化的作文 |

### 2.2 不优先使用“积累实验室”的原因

“积累”容易让用户联想到收藏、记笔记、词库和数量增长，不能直接表达“语境吸收与延迟迁移”。如果调研发现“语境”理解成本较高，可采用更直白的备选：

- **读懂实验室 / 表达实验室**：最清楚，但差异化略弱；
- **吸收实验室 / 表达实验室**：强调内化，但“吸收”难以直接对应产品界面；
- **输入迁移实验室 / 输出迁移实验室**：技术含义准确，但不适合作为大众产品名。

最终命名应通过无解释命名测试决定：让 8–12 名目标用户只看名称和一句副标题，复述“它帮助我完成什么”。

## 3. 为谁解决什么问题

### 3.1 建议首先服务的用户

首个目标群体不是所有考研人，而是满足以下特征的考生：

- 已经在使用不背单词、有道词典、墨墨等词汇工具；
- 认识一批词，却仍会在长句、熟词僻义、搭配和篇章推理中卡住；
- 知道作文重要，但存在空白页焦虑、依赖模板、无法展开或不敢下笔；
- 愿意每天完成一段连续训练，而不只领取答案；
- 有明确的英一/英二、考试日期、目标分和可投入时间。

“剩余 3–9 个月、基础薄弱至中等”可作为首轮招募假设，而不是已经确认的市场事实。零基础用户可能需要系统课程，高分用户可能只需要冲刺专项，二者都可能稀释第一版验证。

### 3.2 两个核心问题

#### 问题一：学过，但没有成为可调用的阅读知识

表面表现：

- 单词卡上认识，放进文章不认识；
- 只记住常见词义，遇到考义或搭配失败；
- 单句能拆，篇章中的指代、转折、论证关系读不稳；
- AI 讲解一看就懂，换篇文章再次失败；
- 阅读材料过难导致频繁查词，过易又没有学习价值。

深层问题不是“缺少更多文章”，而是系统不知道：用户当前掌握了哪个词义、在哪种句法和篇章条件下会失败、下一篇材料应增加多少新负荷，以及何时在新语境中重新验证。

#### 问题二：有知识，但不能形成自己的文字表达

表面表现：

- 面对题目没有思路，迟迟不开始；
- 背了模板，却无法针对新题替换和展开；
- 能看懂范文，但说不出范文如何组织观点；
- AI 改写后文章更漂亮，用户本人仍写不出来；
- 只修语法和词汇，不会改内容、结构、语域和论证。

深层问题不是“缺少一次作文批改”，而是用户没有经历可成功完成的微型表达、范例解构、受控模仿、逐步撤架、修订和新题迁移。

## 4. 是否有竞争力

### 4.1 结论

**问题选择有竞争力，单个功能没有天然竞争力；只有把两个问题做成同一个可测量的迁移系统，才可能形成壁垒。**

现阶段应给出“有条件 Go”，不能仅凭概念判断已经形成产品优势。

### 4.2 已经被市场做成熟或正在快速商品化的能力

| 能力 | 当前代表 | 对本项目的含义 |
|---|---|---|
| 按已知词比例生成或推荐分级材料 | NextWord 公开声称生成 90% 以上已知词的故事；Hend 根据已知词推荐、导入或生成材料 | “知道词表 + 生成文章 + 点词解释”不是差异点 |
| 互动阅读、查词和词汇进度 | Hend、LingQ、Readlang 等 | 不应把阅读器功能当作核心壁垒 |
| 自动作文评价、提示和反复修订 | Cambridge Write & Improve 已提供大量任务、自动反馈和进度；通用写作产品也提供量表反馈 | “评分 + 纠错 + 润色”已是成熟赛道 |
| 考研题库、真题、课程、长难句和 AI 作文 | 扇贝考研、新东方在线等综合产品 | 不能靠增加一个考研专区与其竞争 |

更重要的反证来自 Storyfier：该研究把目标词放入生成故事，再安排完形和写作；参与者主观喜欢生成故事和写作辅助，但 28 人被试实验中，使用这些 AI 功能时的目标词回忆与使用反而低于基线。它提示“流畅、有趣、被喜欢”不能替代学习效果验证。

### 4.3 可以形成优势的系统能力

真正值得投入的不是某个按钮，而是以下互相依赖的能力：

1. **细粒度学习者证据**：区分词形、具体词义、搭配、句法条件、篇章功能、引导使用、独立使用和延迟迁移，而不是只有“认识/不认识”。
2. **多维知识负荷匹配**：同时控制词汇覆盖、目标词义、句法、篇章关系、推理、主题熟悉度、长度、时间和帮助强度，而不是只控制 CEFR 或已知词比例。
3. **语境再遇调度**：根据水平决定同语境巩固、跨语境变化和间隔；低基础用户不能过早追求高语境多样性。
4. **读写共享技能图谱**：阅读中识别的词组、句式和修辞动作，可以进入写作；写作暴露的缺口，又能反向触发阅读任务。
5. **逐级撤除的表达脚手架**：系统默认促进用户第一稿，不默认代写；帮助强度应随能力提高而下降。
6. **纵向作品与迁移证据**：保存用户从第一稿到修订稿、再到新题独立写作的过程，衡量“本人是否会了”。
7. **考研英语专家金标与评测集**：围绕英一/英二的篇章、翻译和写作建立难度标定、错误类型、表达技能和人类评分一致性。
8. **内容质量与权利流程**：优先使用合法来源的自然材料并做可控适配；生成内容必须经过事实、自然度、难度、考研相关性和权利检查。

这些能力必须联合存在。竞争者可以快速复制“生成一篇文章”或“给作文打分”，但较难复制长期学习证据、调度策略、读写迁移数据和经过教研验证的评测集。

### 4.4 必须验证的竞争成立条件

- 用户愿意低摩擦提交近期词汇或其他学习记录；
- 材料匹配不仅让用户感觉顺畅，还能改善未见材料表现；
- 语境再遇能提高目标知识的延迟理解和恰当使用；
- 表达脚手架能提高用户独立开始、完成和修订的概率；
- 帮助减少后，用户仍能在新题写出，而不是依赖系统；
- 用户愿意持续做需要思考的任务，而非只消费讲解；
- 人工编排原型有效后，自动化成本与质量能够接受。

任何一项长期不成立，都可能要求缩小目标用户、改变任务形态或停止立项。

## 5. 两个实验室共同的技术底座

### 5.1 共享学习者模型

学习者模型必须包含用户已经明确要求的字段：

- 目标：英一/英二、考试日期、目标分、当前阶段；
- 学习时间：每周总量、可用时段、连续学习上限；
- 语言背景：四六级、既往考研、学习经历、母语或其他语言影响；
- 显式偏好：材料长度、题材、反馈密度、纸屏习惯、是否计时；
- 局部知识状态：词义、搭配、句法、篇章、阅读技能和表达技能；
- 反复错误假设：触发证据、反证、置信度、影响范围和复查时间；
- 帮助依赖：用户在何种提示级别下才能完成；
- 情感与行为证据：写作启动时间、放弃点、自我效能和任务负荷感。

学习状态不应只有一条“掌握度”，至少区分：

`接触 → 语境识别 → 独立理解 → 引导使用 → 独立使用 → 延迟迁移`

### 5.2 共享知识与技能图谱

图谱至少包含四类节点：

| 节点 | 示例 | 可观察证据 |
|---|---|---|
| 词汇/词组 | `address` 的“处理问题”义；`be subject to` | 新篇章理解、翻译处理、恰当表达 |
| 句法结构 | 让步、同位、倒装、指代链 | 独立拆解、解释、改写、新句迁移 |
| 篇章关系 | 转折、因果、例证、观点与证据 | 证据标注、摘要、推理、段落重构 |
| 表达技能 | 主张—理由—例证；让步—反驳；问题—原因—方案 | 提纲、句群、段落、换题独立使用 |

“优秀写作思路作为 Skill”在产品中应定义为**可复用的表达技能**，而不是复制范文句子或模仿某位作者风格。每个 Skill 应包含：适用题型、交际目的、组成步骤、正反例、常见失败、可用语言资源、练习和迁移测试。

### 5.3 受约束的 Agent

Agent 位于后台，是有状态、可审计的训练编排器，不是聊天机器人。它负责：

1. 读取目标、时间、知识状态和最近证据；
2. 选择本轮唯一或少量高价值学习目标；
3. 在两个实验室之间编排材料、任务、反馈和复查；
4. 更新知识状态与错误假设；
5. 在证据不足、内容质量不合格或评分不确定时降级或拒绝自动决策。

第一版采用明确工作流和有限工具调用，不追求多个 Agent 自由协作。模型输出都是候选，必须通过规则、检索证据、难度检测、量表或人工抽检后才能成为学习任务。

IRT/CAT 只适合前期校准和阶段测量，用于提高评估效率；不应把每日阅读和写作训练变成连续考试。日常调度使用知识状态、任务表现、帮助等级和延迟迁移证据。

## 6. 语境实验室需要解决的问题

### 6.1 不是简单生成，而是选择、适配与验证

材料供给优先级建议为：

1. 合法且自然的原生材料；
2. 对合法材料做受控简化、缩写或注释；
3. 在缺少合适材料时受约束生成；
4. 生成后经过自动质量门和人工抽检。

原因是考研阅读需要自然论证、作者立场、指代和干扰空间。纯生成材料容易语言平滑却缺少真实篇章结构，甚至把目标词生硬塞入文本。

### 6.2 知识量“匹配”必须可计算

每篇材料都应有知识负荷向量，而非单一难度等级：

- 已知词形覆盖率与已知目标词义覆盖率；
- 新词、近期词、熟词僻义和目标搭配密度；
- 句法层级、依存距离、嵌套和省略；
- 篇章关系、指代链和推理步数；
- 主题先验与文化背景负荷；
- 字数、预计时长和限时要求；
- 用户可获得的提示等级。

匹配算法的目标不是让文章“尽可能简单”，而是让大多数任务落在用户可独立完成或少量提示可完成的区间，同时保留少量巩固与挑战。

### 6.3 近期知识怎样反复出现

- 不把全部近期词塞进一篇文章；先按词义、语域、主题与用户水平筛选；
- 为每个目标安排不同层级：原语境复现、相近语境、跨主题语境和延迟新语境；
- 根据学习者水平调整语境多样性，避免基础较弱时过早增加干扰；
- 阅读识别型词只要求读懂；翻译处理型词增加英译汉；主动表达型词组才进入写作；
- 再遇成功不能只由“用户看完了”推断，必须来自无提示理解、证据定位、重构或延迟任务。

### 6.4 讲解怎样不替代学习

建议顺序：

`独立首读 → 标记证据与不确定处 → 最小必要提示 → 用户重新解释 → 针对性讲解 → 微型重构 → 新语境验证`

系统应避免默认给全文翻译和完整总结。研究提示，AI 总结可能改善即时任务表现，却不等同于形成长期学习；主动记录、提取和解释过程不能被跳过。

### 6.5 语境实验室的成功指标

- 匹配率：用户在目标帮助等级下完成，且主观负荷处于目标区间；
- 无提示理解率和证据定位率；
- 目标词义/结构在延迟新篇章中的成功率；
- 同一知识在翻译或表达中的迁移率；
- 讲解前后与若干天后改善，而非只看即时正确；
- 材料自然度、事实性、篇章质量和人工审核通过率。

## 7. 表达实验室需要解决的问题

### 7.1 第一目标是“用户写出来”

系统不以生成高分范文为默认动作，而以用户完成越来越独立的写作为目标。需要记录：

- 看到题目到首次输入的启动时间；
- 无帮助、提问、提纲、句框、局部建议等帮助等级；
- 用户独立产生的词数和有效内容；
- 完稿率、修订次数和修订成功率；
- 新题中的独立迁移；
- 写前/写后自我效能和对作品的所有感。

### 7.2 优秀范文怎样成为 Skill

不应把范文直接切成“高级句型库”。建议流程：

1. 教师或合法来源提供高质量范例；
2. 标注交际目的、篇章功能和修辞动作；
3. 提炼成可复用 Skill，如“描述现象—解释含义—提出立场”；
4. 展示多个水平、多个题目的实现方式，避免把一种表述当唯一答案；
5. 让用户识别、排序、补全、受控模仿，再独立换题；
6. 用新题验证用户是否掌握思路，而非是否记住原句。

英一、英二应共享底层表达技能，但在冲刺阶段切换题型配置，例如：

- 英一图画：画面描述—象征解释—观点论证；
- 英二图表：数据概括—显著变化—原因/意义—立场；
- 小作文：目的—必要信息—行动请求—合适语域。

### 7.3 脚手架必须逐级撤除

建议帮助阶梯从低到高为：

1. 只提供启动问题；
2. 提供表达 Skill 的空白结构；
3. 提供可选择的观点材料；
4. 提供句子开头或局部语言资源；
5. 指出问题并给修订方向；
6. 在用户尝试后给局部示例；
7. 最后才展示完整范例。

系统应优先给“问题与方向”，而不是直接替换全文。现有研究中，直接纠错和 AI 改写对深加工与新作迁移的影响并不相同；完整改写可能提高眼前文本质量，却不能证明用户形成了能力。

### 7.4 反馈必须超过语法纠错

反馈分层：

- 任务完成：是否回应题目、完成交际目的；
- 内容：是否有明确观点、理由、例证和必要信息；
- 结构：修辞动作是否齐全、顺序是否有效、段落是否连贯；
- 语言：准确性、搭配、句法、语域和可读性；
- 迁移：是否恰当使用近期吸收的语言资源；
- 学习行为：是否过度依赖提示、是否完成有效修订。

篇章级自动评价仍是模型弱项。分数应附带不确定性；低置信度、异常题目、疑似跑题或高风险建议要拒判或进入人工复核。

### 7.5 怎样让用户“敢写”

可落地的机制包括：

- 从 3–5 分钟微型段落开始，先制造真实成功经验；
- 允许先写关键词、中文要点或混合语言草稿，再逐步英文化；
- 一次只处理少数高影响问题，避免满屏红色错误；
- 先评价“意思是否成立”，再升级语言；
- 显示用户自己保留下来的好句、好结构和成功迁移；
- 对比第一稿与第二稿时，明确哪些改进由用户完成；
- 随能力提高自动减少提示，使用户看到“我不靠它也能写”。

写作脚手架提高自我效能和范例反馈提高表现已有积极研究信号，但样本与场景不能直接外推到考研英语，因此必须做本地对照实验。

## 8. 两个实验室如何真正联动

### 8.1 语境到表达

每个语境任务最多输出少量“可迁移成分”：

- 一个可主动使用的词组；
- 一个句法结构；
- 一个篇章连接方式；
- 一个表达 Skill 的范例。

用户随后用这些成分完成微型表达。系统评价的是恰当性和独立性，不鼓励为了使用新词而生硬堆砌。

### 8.2 表达到语境

写作暴露的问题转成新的阅读目标：

- 没有理由：阅读两段不同的因果展开；
- 只会空泛喊口号：阅读“主张—证据—解释”范例；
- 转折生硬：比较多种让步—反驳实现；
- 语域不合适：阅读同一意图在正式与非正式文本中的差异；
- 某结构反复失败：在新文章中识别、解释，再回到写作。

### 8.3 周期结构

一个建议的人工原型周期：

`本周表达 Skill → 2–3 个匹配语境中的识别与解构 → 一次微型表达 → 针对性反馈 → 一次修订 → 延迟换题独立表达`

训练量可以增加，但新增内容必须沿同一目标扩展，不能变回随机文章和随机作文题。

## 9. 三阶段仍然保留，但不再形成模块堆砌

三阶段是使用周期，不是三个产品专区。两个实验室贯穿全程：

| 阶段 | 语境实验室 | 表达实验室 |
|---|---|---|
| 前期熟悉 | 通过少量材料校准词义、句法、篇章和帮助强度 | 通过低门槛写作了解启动困难、表达水平和帮助依赖 |
| 中期任务 | 近期知识的匹配阅读、语境再遇、解释、重构与延迟验证 | 表达 Skill、微型写作、逐级撤架、修订和读写迁移 |
| 后期冲刺 | 英一/英二真题篇章、证据定位、题型方法和限时训练 | 英一/英二小作文与大作文的题型 Skill、模板迁移和限时独立写作 |

语法地图、词汇迁移、错误假设、真题和报告都作为两个实验室的后台能力或阶段配置，不再成为并列首页入口。

## 10. 第一版边界

### 10.1 MVP 只验证一个最小闭环

建议人工原型先验证：

1. 用户提交一小批近期学习的词或词组；
2. 系统选择其中少量适合当前水平与主题的目标；
3. 用户完成一篇匹配材料的独立首读、证据处理和必要讲解；
4. 系统提炼一个表达 Skill 和少量可主动使用的语言资源；
5. 用户完成一个短段落而非整篇大作文；
6. 系统给分层提示，用户修订；
7. 数日后用新材料和新题验证理解与表达迁移。

### 10.2 第一版明确不做

- 自建单词闪卡、词书和单词间隔重复；
- 通用 AI 对话入口；
- 全科、择校、社区、直播课和名师商城；
- 独立语法课程或以“点亮整张语法地图”为卖点；
- 海量无审核生成阅读和模拟题；
- 默认一键生成作文、全文润色或代写；
- 仅凭 AI 分数预测真实考研分数；
- 在缺少标定数据时上线每日 CAT；
- 一开始覆盖所有水平、所有题型和整套模考。

## 11. 研究与实验优先级

### 11.1 先验证需求，不先验证界面

首轮访谈应围绕真实学习物，而不是询问“你想不想要 AI”：

- 带来最近一周的背词记录、阅读材料和作文；
- 找出卡片上认识但文章中失败的具体例子；
- 复盘一次迟迟不动笔或依赖范文的写作；
- 观察用户愿意提交哪些外部学习数据；
- 测试两个实验室名称能否被正确理解。

### 11.2 人工 Wizard-of-Oz 实验

先由教师和团队人工完成材料匹配、Skill 提炼和任务编排，避免把模型能力与产品命题混在一起。建议至少比较：

- 普通难度分级 vs 多维知识负荷匹配；
- 只看讲解 vs 独立加工后再讲解；
- AI 全文改写 vs 分层提示与用户修订；
- 同题改好 vs 新题独立迁移；
- 只测即时表现 vs 加入数日后的延迟测试。

### 11.3 建议的 Go / Pivot / Stop 证据

| 决策 | 证据信号 |
|---|---|
| Go | 未见材料理解、延迟保持、写作启动、独立成文和新题迁移均有改善信号，且用户愿意持续完成任务 |
| Pivot | 阅读有效但用户不写，或写作有效但外部词汇接入成本过高；缩为单实验室或调整首发人群 |
| Stop | 只能提高满意度和眼前文本质量，无法改善独立表现；或人工有效但自动化质量/成本不可接受 |

## 12. 主要风险

| 风险 | 影响 | 缓解方向 |
|---|---|---|
| 生成材料看似自然但学习无效 | 核心价值不成立 | 自然材料优先；延迟与迁移评测；保留无 AI 基线 |
| 外部词汇数据拿不到 | 无法知道用户近期积累 | 支持导入、分享、截图与低摩擦确认；产品不可依赖未开放私有接口 |
| AI 帮助过强 | 用户作品变好但本人没学会 | 独立首答、帮助阶梯、提示撤除、作品所有感和新题迁移 |
| 写作任务太费力 | 完成率与留存低 | 从短表达开始；任务与用户时间匹配；每次只解决少量问题 |
| 自动评分在篇章层误判 | 错误教学与信任损失 | 分项评价、置信度、拒判、人工金标和抽检 |
| 范文与真题版权 | 无法商用或造成侵权 | 权利台账、授权/公版/自有内容、只提炼抽象表达技能并保留来源 |
| 只针对考研导致季节性与市场有限 | 商业规模受限 | 先以垂直质量验证价值；是否扩展四六级/雅思另行决策，不污染首版 |
| 功能被综合 App 快速复制 | 获客与定价受压 | 把壁垒放在纵向数据、调度、金标、迁移效果和教研流程 |

## 13. 当前决策与待决项

### 已收敛

- 前台核心只有语境实验室与表达实验室；
- 不做单词闪卡，但尊重并承接成熟词汇工具；
- 不做聊天机器人；Agent 是后台训练编排与状态更新系统；
- 不以生成文章、全文讲解、作文评分或润色作为定位；
- 三阶段保留为固定使用周期，英一/英二题型特化发生在后期；
- 学习效果以独立、延迟和新材料迁移证明。

### 尚需用户研究决定

- 首发用户的基础、剩余备考时间与付费意愿；
- “语境实验室”是否比“读懂实验室”更易理解；
- 近期词汇数据的最低可接受提交成本；
- 用户每天愿意投入的连续阅读与短写作时长；
- 先从阅读带写作，还是先从写作缺口反推阅读；
- 第一批表达 Skill 选择图画、图表还是通用论证；
- 人工教师在内容审核和低置信度反馈中的必要比例。

## 14. 研究依据与外推边界

- [二语附带词汇学习元分析（2023）](https://www.cambridge.org/core/journals/language-teaching/article/how-effective-is-second-language-incidental-vocabulary-learning-a-metaanalysis/E38E3468FD2090B1FA3051051DE8E70C)：意义导向输入能够产生词汇学习，但延迟收益有限且受水平、材料、间隔和测试方式调节；不能把“读过”当作“学会”。
- [语境多样性与附带词汇学习（2026）](https://www.cambridge.org/core/journals/studies-in-second-language-acquisition/article/effects-of-contextual-diversity-on-incidental-vocabulary-learning/F83A1EB55404E2E07FDC1695B0BC2BAE)：语境多样性的延迟识别收益受既有词汇水平调节，支持按水平安排语境变化。
- [Storyfier（2023）](https://arxiv.org/abs/2308.03864)：目标词生成故事、完形与写作的组合获得主观偏好，但小样本实验学习结果低于基线，直接支持“生成体验不等于学习效果”的警戒。
- [LLM、记笔记与阅读理解随机实验（2026）](https://doi.org/10.1016/j.compedu.2025.105514)：主动记录和信息处理不能简单被 LLM 总结替代；研究人群不是考研考生。
- [写作脚手架与自我效能（2024）](https://www.frontiersin.org/journals/psychology/articles/10.3389/fpsyg.2024.1454104/full)：50 名大学生的准实验显示，分解任务并配合反馈后，部分写作自我效能显著提高；样本单一且不是考研场景。
- [范例反馈与二语写作表现（2025）](https://doi.org/10.1080/09658416.2025.2467113)：为范例解构与比较提供积极证据信号，但仍需在考研题型中复验。
- [直接纠错与 AI 改写比较（2025）](https://doi.org/10.1016/j.system.2025.103765)：提示不同反馈方式可能产生不同深加工与新作迁移，反对默认全文改写。
- [EssayJudge（2025）](https://aclanthology.org/2025.findings-acl.329/)与[不确定性校准作文评分（2025）](https://aclanthology.org/2025.emnlp-main.992/)：支持篇章级评价、分项量表、不确定性与拒判机制。

以上证据用于形成产品假设，不构成“必然提分”承诺。正式立项需要目标人群访谈、教师共创、人工训练实验和未见材料迁移测试。
