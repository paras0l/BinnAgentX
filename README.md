# BinnAgent 考研英语版

面向全国硕士研究生招生考试英语（一）与英语（二）的读写迁移 Agent。

当前阶段：**资源受限的技术立项准备**。截至 2026-07-15，本仓库只有调研、产品和研发前置文档，尚未进入编码阶段；当前无法开展考生/教师人工验证，后续只能先验证工程可行性，不能宣称产品需求或学习效果已经得到真人验证。

交付形态已经冻结为：**仅支持电脑浏览器的桌面 Web 产品**。不开发或适配手机、平板、原生桌面客户端及其他终端；纸笔仅用于考场迁移训练与效果验证，不是软件入口。

## 产品定义

Agent 位于后台，负责建立学习者模型、编排训练、更新知识状态、验证错误假设和切换备考阶段；它不是聊天机器人。用户前台只围绕两个核心学习场域训练：**语境实验室**与**表达实验室**。

- **语境实验室：读懂与吸收**——把用户近期学过但尚未稳定的词义、词组、句法和篇章知识，放进知识负荷匹配的自然材料中反复遇见并延迟验证。
- **表达实验室：想清楚并写出来**——把优秀文章中的可复用思路提炼为表达技能，用逐步撤除的脚手架帮助用户独立完成句子、段落和作文。

> 不替你背词，不替你写作文；让你学过的，真正读得懂、写得出。

产品核心目标：

> 在固定的备考周期内，把外部积累转化为阅读中的独立理解和写作中的独立表达，并用延迟、新材料和新题迁移证明阶段内进步。

核心循环：

`用户先尝试 → 语义标记/自主输出 → 最小必要提示 → 用户重新输出 → 平行新任务 → 延迟迁移 → 模型更新`

## 固定使用周期

1. **前期熟悉阶段**：通过少量语境与表达任务、学习背景和显式偏好形成初步画像，校准阅读负荷、写作起点和帮助强度。
   首次体验采用“60–90 秒最小背景设置 → 8–12 分钟轻量阅读校准 → 第一篇匹配阅读”，并在前 3–5 个真实任务中持续纠偏；时间与数量是待上线数据校准的首版假设。
2. **中期任务阶段**：词汇积累继续使用用户选择的成熟工具；本系统把近期知识安排进匹配语境，再通过表达 Skill、短写作、修订和延迟任务形成读写迁移。
3. **后期考试冲刺阶段**：两个实验室按英语一/英语二切换题型配置，进行真题篇章、阅读方法、作文模板迁移和限时独立写作。

阶段顺序固定；每阶段时长、训练量和难度根据考试日期、学习时间、目标和进步证据调整。

## 当前产品约束

- 前台核心只有语境实验室与表达实验室；语法、词汇迁移、错误假设、真题和报告作为后台能力或阶段配置，不堆成独立产品入口。
- 用户学习端和开发者控制舱都只交付桌面电脑浏览器版本；不为手机、平板、原生客户端或触控笔建立兼容范围。
- 前端固定拆为两个独立入口：用户端只呈现学习体验和可理解的进步证据，不出现技术解释；开发者监控控制舱用于运行观测、人工复核和受审计控制。
- AI 不替用户完成认知动作；用户先尝试，AI 只给最小必要帮助，之后必须由用户重新输出并在新材料中验证。
- 前端标记必须表达观点、证据、逻辑、不确定点或可迁移表达等认知意图；不以高亮和收藏数量表示进步。
- 输出工作区允许用户通过电脑键盘从中文、关键词、中英混合或不完整句起步，但最终必须由用户完成英文组织、修订和撤架迁移。
- 即时进步事件只表达“本次已做到”，不得冒充延迟、无提示的稳定掌握。
- 同时支持英语一与英语二，采用“共同读写能力底座 + 后期试卷/题型配置”。
- 学习内容必须落在用户的可学习区间，不能长期过难或过易。
- 训练量由用户按时间与目标自由定制，系统给出负荷建议，但不以固定题量或套餐额度限制。
- 每个任务必须属于明确阶段目标和知识路径，避免把每日一句、随机单词、零散小题拼成学习计划。
- 学习者模型必须包含目标、学习时间、语言背景、显式偏好、局部知识状态和反复错误假设。
- 不自建背单词或间隔重复功能；外部工具的完成记录只能证明投入，词汇掌握仍由本系统在阅读、翻译和写作中验证。
- 充分利用成熟闪卡的高效率；本系统不重复测试词义卡片，而是负责让最近词汇在多个语境中被读懂、译出、恰当使用并延迟保持。
- 文字表达贯穿全周期，从低门槛短表达逐步过渡到段落、作文模板迁移和限时独立写作。
- 真题、解析、词典、例句和范文商用前必须完成权利审查或授权。
