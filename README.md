# BinnAgentX 考研英语读写训练

面向全国硕士研究生招生考试英语（一）与英语（二）的读写迁移 Agent。

当前阶段：**范围受限的可连续体验版本，邮箱验证登录、账号级任务隔离、核心 Agent 辅助、异步材料生成与跨次训练已运行**。截至 2026-07-21，仓库已打通“邮箱验证 → 选择/注册学习账号 → 首次背景 → 校准 A/B → 校准摘要 → 匹配阅读 → 精确语义标记与独立 V1 → 按需 H1–H4 → 用户 V2 与修订引用 → 按阅读来源选择表达材料 → 微型表达 V1 → 模型单项反馈 → 用户表达 V2 → 难度反馈 → 完成页/学习首页 → 新材料连续训练”的真实 API 主路径。Obsidian 学习上下文会按目标相关性、最新学习、到期复习、证据冲突与近期使用记录统一选材；训练完成后写入不可变学习证据，投影知识状态和下次复习时间，并由常驻 PostgreSQL Worker 异步生成到期材料。模型网关默认自动选择已配置的 DeepSeek、LongCat 或本地 Ollama，并继续执行结构 Schema、原文证据引用、超时和预算约束；简单单项反馈关闭 LongCat thinking，文章生成、长选区分析、表达复盘和内容审核保留推理并只解析最终结构化内容。阅读生成一次形成 4–6 道基础到进阶的题库，运行时按学习者证据选择语境词义、语法填空、细节理解、推断、写作目的、句子插入或篇章逻辑等任务。候选材料主要由独立审核 Agent 评分、退回修订和批准，人工负责抽检、高风险升级、权利与发布审核。确定性适配器只用于显式离线/测试或调用失败后的内部降级，不再作为正常体验文案。除已落地的资产导出消费者外，其他通用 Outbox 消费、服务端跨浏览器学习历史及真人效度验证仍未完成。当前只能验证工程与交互流程可行性，不能宣称材料匹配、反馈判断或学习效果得到真人验证。

交付形态已经冻结为：**仅支持电脑浏览器的桌面 Web 产品**。不开发或适配手机、平板、原生桌面客户端及其他终端；纸笔仅用于考场迁移训练与效果验证，不是软件入口。

## 产品定义

Agent 位于后台，负责建立学习者模型、编排训练、更新知识状态、验证错误假设和切换备考阶段；它不是聊天机器人。用户前台只围绕两个核心学习场域训练：**语境实验室**与**表达实验室**。

- **语境实验室：读懂与吸收**——把用户近期学过但尚未稳定的词义、词组、句法和篇章知识，放进知识负荷匹配的自然材料中反复遇见并延迟验证。
- **表达实验室：想清楚并写出来**——把优秀文章中的可复用思路提炼为表达技能，用逐步撤除的脚手架帮助用户独立完成句子、段落和作文。

> 不替你背词，不替你写作文；让你学过的，真正读得懂、写得出。

产品核心目标：

> 在固定的备考周期内，把外部积累转化为阅读中的独立理解和写作中的独立表达，并用延迟、新材料和新题迁移证明阶段内进步。

核心循环：

`用户先尝试 → 语义标记/自主输出 → 最小必要提示 → 用户重新输出 → 平行新任务 → 延迟迁移 → 模型更新`

## 项目结构

```text
apps/                       # 学习端与开发者控制舱
services/                   # API、Worker 与数据库迁移
python/
  binnagent_agent/          # Agent 角色、工作流、工具、记忆、提示、网关和策略
  binnagent_domain/         # 与框架无关的领域模型和不变量
  binnagent_evaluation/     # 离线评测与内容完整性检查
packages/                   # 前端共享领域逻辑与 UI 基础
contracts/                  # Analytics、内容、领域、模型和场景契约
fixtures/                   # 可复现的开发内容与固定场景
tests/                      # 单元、契约、集成和端到端测试
scripts/                    # 可重复执行的开发与运维脚本
docs/                       # 产品、架构、研发记录和 ADR
```

本地一键启动：

- `./scripts/deploy.sh`：默认以独立的 `binnagentx` Compose 项目构建并后台启动 PostgreSQL、迁移、内容 Worker、Obsidian 资产导出器、API、学习端和控制舱；只有 API、所选前端和内容 Worker 均健康后才报告成功。入口包括控制舱 `http://127.0.0.1:3001` 和复用旧 BinnAgent 的 Langfuse `http://127.0.0.1:3100`。
- `./scripts/deploy.sh --restart`：运行代码未变化时跳过镜像构建并强制重建容器；如果代码指纹与上次成功构建不同，会自动升级为重新构建，避免启动旧代码。默认构建过程每 10 秒显示一次心跳，详细输出仍保存在 `logs/compose.log`。
- 不需要某个前端时使用 `--no-learner` 或 `--no-control`；排障时追加 `--verbose`。使用 `docker compose -p binnagentx down` 停止容器；业务任务、进度、重试与发布状态统一保存在 PostgreSQL，生成内容保存在独立持久卷中。
- 需要逐个调试宿主机进程时使用 `./scripts/deploy.sh --host-services`，此模式保留原来的 `uv` / `pnpm` 启动与 `Ctrl-C` 清理行为。

控制舱的“用户管理”展示当前数据库中的注册账号与体验账号、最近登录、有效会话、完整训练次数、资产数量和 Obsidian 配对状态。开发审核角色可以撤销某个用户的全部有效会话，使其所有设备重新登录；控制舱不会展示验证码、会话 token 或 Obsidian Sync Secret。

控制舱同时提供 BinnAgentX 独立的 **Tools** 与 **Prompts** 管理：Tools 页面治理代码注册能力、风险、权限 Schema 和持久化启停策略；Obsidian 已内化为所有学习 Agent 共用、学习者隔离的长期记忆 Provider，显式读写 Tool 只是同一记忆模块的受治理入口。Prompts 页面管理草稿、激活、归档、变量、模型策略和渲染预览；`python/binnagent_agent/prompts/` 保存代码默认目录和变量契约，远程模型运行时会解析数据库活动版本，控制舱修改不再是孤立配置。所有记录固定使用 `project_key=binnagentx`，不读取或覆盖旧 BinnAgent 的注册表与 Prompt。详细边界见[Agent 配置控制面](docs/44-Agent配置控制面.md)与[Obsidian Agent 记忆模块](docs/45-Obsidian作为Agent记忆模块.md)。

### 本机 Obsidian 学习资产

学习资产页只保存索引与训练证据；笔记正文保留在用户自己的 Obsidian Vault。普通用户在资产页的“配置 Obsidian”中选择 Vault、下载 `BinnAgentX-Learning-Sync-v0.1.6.zip`，再把页面生成的 Connection ID 和 Sync Secret 填入插件。插件首次加载会初始化 `BinnAgentX/00-Inbox` 至 `06-Attachments`，创建使用指南、MOC / Dataview Dashboard、模板和不参与同步的入门示例，并把 Obsidian 模板与附件目录自动指向对应文件夹；总、词汇、语法地图统一命名为 `00-Dashboard.md`，在文件名升序模式下保持为目录中的第一篇笔记。重复初始化不覆盖已有内容。连接按学习账号持久化在 PostgreSQL，插件凭据保存在 Vault 的插件设置中；刷新、退出登录、容器重启或再次登录都不会要求重复配置。只有连接尚未完成首次同步、已撤销或同步失效时，学习首页才把“从 Obsidian 笔记生成新材料”切换为“去配置 Obsidian”。插件启动后及每 60 秒自动同步，也可用命令面板中的 **Sync approved learning context** 立即重试；完成提示会分别报告接收、上传、Inbox 检出、Agent 分类、实际移动和目标目录，不再把“同步成功”等同于“整理完成”。

双向闭环为：学习端新增资产或保存阅读标注 → 待导出队列 → 插件在 `BinnAgentX/00-Inbox/` 创建待整理笔记并回执 → 下次登录或资产页手动入口排队同一个整理 Agent，插件在同步时把 Inbox 笔记安全移动到词汇、语法、阅读或写作白名单目录 → Obsidian 模板笔记上传有限摘录并登记为元数据资产 → BinnAgent 按用户指定目标和笔记类型生成新的个性化英语阅读，并用自管短期记忆避开近期重复上下文 → 材料进入学习首页的标准训练任务队列 → 用户选择材料后进入既有阅读实验室，继续复用选项作答、语义标注、语法挑战、V1/V2、H1–H4、表达迁移、难度反馈和完成记录 → 新标注再次进入同一收集箱链路。整理 Agent 不删除或改写笔记，也不能输出任意路径。训练材料状态由标准训练运行驱动并持久化；队列不另设阅读器或手工完成按钮。同一时间只允许一个标准训练运行。资产页不负责生成或展示阅读正文，资产 API 响应也不包含笔记正文或摘录。

`binnagent-obsidian-bridge` 是开发和受管部署场景的可选增强：它只允许创建和打开 `BinnAgentX/` 目录中的笔记，监听 `127.0.0.1:8787` 并要求 token。它用于后台异步投递，不是普通学习者开始使用 Obsidian 的前置条件。

Agent 核心代码统一从 `binnagent_agent` 导入；模型提供方、预算、安全策略和未来的持久记忆都必须留在该包定义的边界内。可再生依赖、缓存、测试报告与视觉验收截图不进入版本库。

## 固定使用周期

1. **前期熟悉阶段**：通过少量语境与表达任务、学习背景和显式偏好形成初步画像，校准阅读负荷、写作起点和帮助强度。
   首次体验采用“60–90 秒最小背景设置 → 8–12 分钟轻量阅读校准 → 第一篇匹配阅读”，并在前 3–5 个真实任务中持续纠偏；时间与数量是待上线数据校准的首版假设。
2. **中期任务阶段**：词汇积累继续使用用户选择的成熟工具；本系统把近期知识安排进匹配语境，再通过表达 Skill、短写作、修订和延迟任务形成读写迁移。
3. **后期考试冲刺阶段**：两个实验室按英语一/英语二切换题型配置，进行真题篇章、阅读方法、作文模板迁移和限时独立写作。

阶段顺序固定；每阶段时长、训练量和难度根据考试日期、学习时间、目标和进步证据调整。

## 当前产品约束

- 前台核心只有语境实验室与表达实验室；语法、词汇迁移、错误假设、真题和报告作为后台能力或阶段配置，不堆成独立产品入口。
- 用户学习端和开发者控制舱都只交付桌面电脑浏览器版本；不为手机、平板、原生客户端或触控笔建立兼容范围。
- 前端固定拆为两个独立入口：用户端只呈现学习体验和可理解的进步证据，不出现技术解释；开发者监控控制舱用于运行观测、人工复核和受审计控制。
- AI 不替用户完成认知动作；用户先尝试，AI 只给最小必要帮助，之后必须由用户重新输出并在新材料中验证。
- 前端标记必须表达观点、证据、逻辑、不确定点或可迁移表达等认知意图；不以高亮和收藏数量表示进步。
- 输出工作区允许用户通过电脑键盘从中文、关键词、中英混合或不完整句起步，但最终必须由用户完成英文组织、修订和撤架迁移。
- 即时进步事件只表达“本次已做到”，不得冒充延迟、无提示的稳定掌握。
- 同时支持英语一与英语二，采用“共同读写能力底座 + 后期试卷/题型配置”。
- 学习内容必须落在用户的可学习区间，不能长期过难或过易。
- 训练量由用户按时间与目标自由定制，系统给出负荷建议，但不以固定题量或套餐额度限制。
- 每个任务必须属于明确阶段目标和知识路径，避免把每日一句、随机单词、零散小题拼成学习计划。
- 学习者模型必须包含目标、学习时间、语言背景、显式偏好、局部知识状态和反复错误假设。
- 不自建背单词或间隔重复功能；外部工具的完成记录只能证明投入，词汇掌握仍由本系统在阅读、翻译和写作中验证。
- 充分利用成熟闪卡的高效率；本系统不重复测试词义卡片，而是负责让最近词汇在多个语境中被读懂、译出、恰当使用并延迟保持。
- 文字表达贯穿全周期，从低门槛短表达逐步过渡到段落、作文模板迁移和限时独立写作。
- 真题、解析、词典、例句和范文商用前必须完成权利审查或授权。

## 下一决策点

当前先冻结双前端、数据契约、工作流可靠性、安全、成本与可观测性，构建范围受限的技术原型。用户访谈、教师共创、人工训练实验和长期效果验证保留为未关闭证据门槛；条件改善前，不把合成测试、模型互评或开发者自测写成真人学习效果。详见[产品发现与研发路线图](docs/08-产品发现与研发路线图.md)与[技术栈调研与架构选型](docs/16-技术栈调研与架构选型.md)。
